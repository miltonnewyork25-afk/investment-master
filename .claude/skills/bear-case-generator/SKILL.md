# 看空对抗分析技能 v1.0

> 来源: COST v21 P4-B 独立看空Agent + `docs/bear_case_methodology.md`
> PG 反例: Phase 4 看空是补充性质，受 Phase 1-3 看多偏见严重影响

## 触发条件

- Tier 3 Phase 4 启动时（必须触发）
- Tier 2 风险分析时（建议触发，简化版）

## 核心原则

```
1. 独立性: 看空Agent不读Phase 1-3的看多结论
2. 对抗性: 任务是"最大化看空论点的说服力"，不是"平衡"
3. 等权重: 每个Bear Case不论概率大小，都给予等量分析篇幅
4. 反证: 每个看多论点必须有对应的看空反驳
```

## 输入限制 (关键设计)

看空Agent**只能读取**以下文件：
- `reports/{TICKER}/data/shared_context.md` (事实数据)
- `reports/{TICKER}/data/data_anchors.md` (已验证数据点)
- `reports/{TICKER}/data/core_questions.md` (CQ列表)

**禁止读取**：
- Phase 1-3 的分析报告 (避免确认偏差)
- Phase 3 的护城河评分 (避免锚定效应)
- Phase 3 的估值结论 (避免路径依赖)

## 执行流程

### Step 1: 构建看空论点库

从数据文件中**主动寻找负面信号**，不是被动回应看多。

最低要求: **≥8 个独立 Bear Case**

Bear Case来源清单:
| 来源 | 寻找方向 | 示例 |
|------|---------|------|
| **估值** | 历史PE/PB分位数, 均值回归风险 | PE 53x vs 5年均值 42x |
| **增长** | 增速放缓信号, 天花板接近 | 同店增速从8.6%降至6.4% |
| **竞争** | 竞对反攻, 新进入者, 替代品 | Sam's Club SSS超过COST |
| **宏观** | 利率/通胀/关税/衰退 | IEEPA关税$0.7-1.5B年化成本 |
| **技术** | 被颠覆风险, 技术落后 | Agentic Commerce侵蚀冲动消费 |
| **治理** | 管理层交接, 激励错位 | 新CEO未经考验 |
| **监管** | 新法规, 税务变化 | 最低工资上涨→成本压力 |
| **财务** | 会计质量, 一次性项目, 资本配置 | 会员费增速不可持续 |

### Step 2: 等权重深度分析

每个 Bear Case 用统一模板分析：

```markdown
### Bear Case {N}: {标题}

**核心逻辑**: {1-2句话}

**量化影响**:
- 触发概率: {X}%
- 估值影响: {-$X/股 或 -X%}
- 时间框架: {N个月/年}

**支撑证据** (≥3条):
1. [硬数据: {来源}] — {证据}
2. [硬数据: {来源}] — {证据}
3. [合理推断: {推理链}] — {证据}

**看多方可能的反驳**:
- {预期的反驳}
- **再反驳**: {为什么反驳不成立或不够充分}

**Kill Switch建议**:
- 触发条件: {具体量化条件}
- 动作: {减仓/清仓/重新评估}
```

### Step 3: 概率加权熊市估值

构建等权重看空矩阵:

```markdown
| Bear Case | 概率 | 估值影响 | 概率加权影响 |
|-----------|:----:|---------|------------|
| BC-1: PE压缩 | 35% | -$280/股 | -$98 |
| BC-2: 增长停滞 | 20% | -$150/股 | -$30 |
| ... | ... | ... | ... |
| **总概率加权下行** | — | — | **-${总计}** |
```

### Step 4: "如果全错了"极端情景

必须包含一个"所有看多论点同时失效"的极端情景:

```markdown
### 极端看空情景: 如果我们全错了

假设条件:
- {看多论点1}不成立
- {看多论点2}不成立
- {看多论点3}不成立

极端估值: ${极端低价} (当前价格的{X}%)
PE: {X}x (历史底部)
触发路径: {具体事件链}
概率: {X}% (主观判断)
```

### Step 5: 输出要求

```markdown
# {TICKER} 看空等权重分析

## 看空论点总览 (≥8个)
{表格汇总}

## 详细分析
{每个Bear Case的深度分析}

## 概率加权熊市估值
{矩阵}

## 极端情景
{全错分析}

## 元数据
- Agent: Bear Case Generator
- 独立性声明: 本分析未读取Phase 1-3看多结论
- Bear Case数: {N}
- 平均篇幅: {N}字符/Case
- 标注密度: {N}/万字符
```

## 质量检查

- [ ] ≥8 个独立 Bear Case
- [ ] 每个 Case ≥3条硬数据支撑
- [ ] 每个 Case 有量化影响($/股)
- [ ] 包含"再反驳"(不能只列反驳就结束)
- [ ] 概率加权矩阵完整
- [ ] 极端情景存在
- [ ] 看空内容 ≥ Phase 4 总篇幅的 50%
- [ ] 未引用 Phase 1-3 的看多结论
