# 投资分析框架编排器 v20.0-半导体

## 触发条件

当检测到以下关键词时自动执行：深度分析、研究、调研、analyze、半导体公司股票代码

## v20.0升级说明

- 基础字数: 120,000 × 行业系数（半导体×2.0 = 240,000）
- 五引擎协同: 并行Agent执行，节省70%时间
- 预测市场集成: PPDA + PMSI + Kill Switch概率驱动
- 七条铁律: 含规则6（概率必背离分析）和规则7（情绪必市场验证）
- 证据标准: A+B级≥95%
- TSM验证基准: 120,847字符 | 87.4/100 | 91.3%背离

## 执行流程

### Step 1: 识别半导体子行业

根据公司名/代码判断子行业类型：

| 子行业 | 代表公司 | 特殊分析维度 |
|--------|---------|-------------|
| 代工厂 | TSM, UMC, SMIC, GFS | 产能+ASP+地缘直接冲击 |
| 设计公司 | NVDA, AMD, QCOM, AVGO, MRVL | AI曝光度+设计胜出率 |
| 设备厂商 | ASML, LRCX, AMAT, KLA | 订单积压+周期先行 |
| 存储芯片 | MU, SK Hynix | 价格弹性+库存周转 |
| IDM | INTC, TXN, ADI, ON | 制造+设计双重分析 |
| IP/EDA | ARM, SNPS, CDNS | 生态锁定+授权模式 |

如检测到非半导体公司，提醒用户切换到对应行业worktree。

### Step 2: 加载模块

1. **必须读取** `modules/universal.md` — 通用模块（23个）
2. **必须读取** `modules/semiconductor.md` — 半导体适配模块（10个）
3. **必须激活** v20.0五引擎系统
4. **必须激活** 预测市场集成（PPDA + PMSI）
5. 合并模块清单 = 通用23 + 半导体10 + v20.0增强5 = 38模块

### Step 3: 生成执行计划

读取 `templates/execution_plan.md` 模板，填充后写入 `task_plan.md`：

```

## 行业：半导体 - {子行业}
## 行业复杂度系数：2.0
## 最低字数：120,000 × 2.0 = 240,000
## 框架版本：v20.0-半导体

### Phase 1: 技术+生态定位 (通用 + 半导体适配)
- [ ] 通用模块: 公司画像/价值链/生态系统...
- [ ] 半导体S1: AI 7层定位
- [ ] 半导体S2: 技术代差分析
- [ ] v20.0: 预测市场概率矩阵 (≥12事件)

### Phase 2: 周期+数据雷达 (通用 + 半导体适配)
```

### Step 4: 执行并追踪

- 每完成一个模块，更新 `progress.md`
- 每完成一个 Phase，输出检查点（阻断式门控）
- 检查点通过后才进入下一 Phase
- 使用 `/parallel-analysis` 加速Phase 2

### Step 5: 完成后复盘

写入 `reflection.md`：
- 哪些模块做得好，哪些需改进
- v20.0引擎输出质量评估
- 与TSM验证基准对比
- PPDA/PMSI算法校准建议
