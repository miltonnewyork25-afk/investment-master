# 第一部分：第一性原理层（WHY — 底层规律）

## 原理1：人类决策非理性（Kahneman）
所有营销的底层都是利用人类认知偏误：
- **损失厌恶**：强调"错过"而非"获得" → 2倍心理效力
- **锚定效应**：设置参照物（先贵后便宜）
- **框架效应**：同一事实不同表述产生不同行为
- **社会认同**：不确定时模仿他人
- **稀缺性**：越少越想要
- **确认偏误**：人只接受符合已有信念的信息

**决策规则**：每个内容的 Hook 必须触发至少2个认知偏误。

## 原理2：用户购买的是"工作完成"（Christensen JTBD）
用户不买产品，他们"雇佣"产品完成一个工作：
- **功能性工作**：在正确时机做出买卖决策
- **情感性工作**：在市场恐慌时获得信心和确定感
- **社会性工作**：在社交场合展示投资智慧

**决策规则**：所有传播信息必须描述"工作完成后的状态"，而非产品功能。

## 原理3：心智定位决定一切（Ries & Trout）
用户心智空间有限，每个品类只记住1-3个品牌：
- 第一原则：成为品类第一（或创造新品类成为第一）
- 第二原则：用一个词/概念占据心智
- 第三原则：简单>复杂，尖锐>全面

**投资大师的心智定位**："100%准确率的AI周期投资框架"
**占据的概念**："100% Accuracy"

## 原理4：竞争在于结构性优势（Porter + Buffett）
持续竞争优势来自结构性护城河：
- 数据积累（10年回测+持续评分历史）
- 方法论壁垒（多维度+行业特定指标）
- 网络效应（社区验证→更多信任→更多用户）
- 转换成本（用户配置+历史数据+习惯）

**决策规则**：每个营销动作同时服务两个目的——短期增长+长期护城河。

## 原理5：创造无竞争空间优于竞争（Blue Ocean）
不在红海中竞争，而是创造蓝海：
- **消除**：主观判断、高成本
- **减少**：覆盖广度、操作复杂度
- **增加**：透明度、准确性验证
- **创造**：供应链图谱、AI原生交互、周期评分系统

## 原理6：采用遵循S曲线（Rogers + Moore）
新产品扩散遵循可预测的曲线：
- 当前目标：Innovators (2.5%) + Early Adopters (13.5%)
- 消息策略：新颖性+技术优势+数据证明
- 跨越鸿沟条件：10-16%采用率+社会证明积累
- 鸿沟后策略：案例+易用性+从众心理

## 原理7：增长有复利效应但边际递减（Marginal Returns）
- 每个渠道遵循S曲线：初期高回报→中期线性→后期饱和
- 投入的最优分配：追逐边际回报最高的渠道
- 当一个渠道的边际CAC超过LTV → 立即切换
- 70/20/10法则：已验证/实验/探索

## 原理8：网络效应是最强增长引擎（Platform Economics）
- 直接网络效应：更多用户→更有价值
- 间接网络效应：更多内容→更多用户→更多内容
- 飞轮>漏斗：追求自增长循环而非线性转化

# 第二部分：策略层（WHAT — 基于原理的决策框架）

## 决策框架：MECE-MARGINAL 方法

每次面对营销决策时：
```
1. MECE分解：穷尽所有选项（互斥+完全穷尽）
2. 边际分析：每个选项的边际回报是多少？
3. 杠杆识别：哪个选项有乘数效应？
4. 约束检查：当前瓶颈是什么？（资源/知识/渠道/时间）
5. 优先排序：边际回报 × 杠杆倍数 ÷ 投入成本 = 优先级
6. 执行：从优先级最高的选项开始
7. 反馈：度量结果，更新对边际回报的估计
```

# 第三部分：战术层（HOW — 执行方法论）

## VIRAL-SYSTEM 执行循环（11步）

```
V - Value（价值锚定）     → 定位理论 + JTBD 确定核心价值
I - Insight（洞察挖掘）   → 行为经济学 + 用户研究识别杠杆点
R - Reach（触达策略）     → 扩散理论 + 边际分析选择渠道
A - Amplify（放大效应）   → 网络效应 + STEPPS 设计传播机制
L - Loop（闭环优化）      → AARRR + 边际递减检测

S - Story（故事包装）     → 框架效应 + 叙事经济学包装信息
Y - Yield（产出度量）     → AARRR 漏斗指标度量
S - Scale（规模扩张）     → 飞轮效应 + S曲线识别扩张时机
T - Test（测试迭代）      → 边际分析 + A/B 测试验证假设
E - Evolve（进化升级）    → 淘汰饱和方法，引入新杠杆
M - Multiply（倍增分发）  → 网络效应 + K-Factor > 1 设计
```

## 内容工程（基于认知偏误设计）

### Hook 矩阵（每个Hook触发≥2个偏误）
| Hook | 偏误1 | 偏误2 | 示例 |
|------|-------|-------|------|
| 数据冲击 | 锚定 | 权威 | "100%准确率，164个数据点" |
| 反直觉 | 确认偏误打破 | 好奇 | "最好的投资感觉最糟糕" |
| 损失框架 | 损失厌恶 | 紧迫 | "错过这个信号=错过160%涨幅" |
| 对比冲突 | 锚定 | 社会认同 | "所有人买NVDA，框架说买LRCX" |
| 身份标签 | 社会认同 | 归属 | "如果你是逆向投资者..." |
| 权威证据 | 权威 | 确认偏误 | "基于Buffett+Damodaran理论" |
| 稀缺 | 稀缺性 | 损失厌恶 | "窗口期只有3个月" |

### CTA 阶梯（基于承诺一致性）
```
微承诺：点赞/保存 → 中承诺：关注/评论 → 大承诺：订阅 → 终极：付费
```
每一步建立的承诺使下一步更自然（Cialdini 承诺一致性原理）。

### 内容原子化策略
一个核心洞察的衍生路径（MECE分解）：
```
核心洞察："半导体设备处于周期底部"
├── 按角度
│   ├── 数据证明（"100%准确率怎么做到的"）
│   ├── 个股分析（"LRCX为什么是Top 1"）
│   ├── 反面案例（"NVDA为什么不推荐"）
│   ├── 教育价值（"10条周期投资规则"）
│   ├── 透明度（"完整回测方法公开"）
│   ├── 供应链（"跟着供应链找信号"）
│   └── 对比分析（"INTC vs LRCX 同行业不同命运"）
├── 按格式
│   ├── Thread (280字×N)
│   ├── 长文 (2000-5000字)
│   ├── 视频 (60s / 15min)
│   ├── Newsletter
│   ├── 信息图
│   ├── 社区帖子
│   └── 播客脚本
└── 按语言
    ├── English
    └── 中文
```
**总组合**：7角度 × 7格式 × 2语言 = 98个潜在内容单元

### 饱和监测规则
当某渠道出现以下信号 → 降低投入，转向下一渠道：
- 粉丝增速放缓至<前期50%
- 互动率持续3周下降
- 内容Score连续低于平均
- 边际获客成本(时间)上升>50%

```
K = 每用户邀请数(i) × 转化率(c)
目标：K > 1 → 指数增长
```

# 第四部分：度量层（MEASURE — 数据驱动决策）

## 北极星指标
**传播效力 = 内容触达人数 × 转化率 × 留存率**

## AARRR 漏斗追踪
| 阶段 | 核心指标 | 目标 | 优化原理 |
|------|---------|------|---------|
| Acquisition | 新增触达/日 | 持续增长 | 扩散理论Phase匹配 |
| Activation | 首次"AHA"率 | >60% | JTBD工作完成感 |
| Retention | 周活跃率 | >50% | 习惯循环+承诺一致 |
| Referral | K-Factor | >0.5→>1 | STEPPS+社交货币 |
| Revenue | 付费转化率 | >3% | 损失厌恶+锚定 |

## 边际分析仪表盘
```
每渠道追踪：
  - 边际获客成本(时间) = 本周新增时间投入 / 本周新增用户
  - 边际内容效率 = 新增内容数 / 新增互动数
  - 饱和度指数 = 当前周增长率 / 首周增长率
```

## 反思循环协议（Ralph Loop 每次迭代）
```
1. MECE分解：当前可优化空间的穷尽列举
2. 边际分析：哪个优化方向回报最高？
3. 约束识别：瓶颈是什么？（知识/内容/渠道/数据）
4. 杠杆行动：执行回报最高的1-3个动作
5. 记录度量：产出了什么？效果如何？
```

## 方法论淘汰规则
- 理论层面过时（新研究推翻）→ 替换底层原理
- 战术层面饱和（边际递减）→ 降级+探索新战术
- 效果<基线30% 连续3次 → 移除
- 新方法论效果>现有30% → 立即升级

## 自主任务触发条件
Agent 应在以下情况自主发起新任务：
1. 产品有新数据/报告产出 → 自动生成营销内容
2. 检测到渠道边际收益下降 → 自动探索新渠道
3. 竞品有重大动作 → 自动制定应对策略
4. 方法论有新发现 → 自动整合并更新技能
5. 市场环境变化 → 自动调整定位和消息
