## 已尝试但失败的方法 (避免踩坑)

### 字符计量
- **错误**: 用 `wc -c` 计算字符数 → 中文3字节/字符，结果虚高3倍
- **正确**: 用 `wc -m` (Unicode字符计数)
- **首次踩坑**: TSM v1.0, 2026-02-04

### 内容膨胀
- **错误**: 为凑字符数添加空洞的总结段落
- **正确**: 每次扩展必须包含新数据/新洞察/新分析，通过 G5 禁止词检查
- **首次踩坑**: AMD Phase 3 字符不足时

### SubAgent数据质量
- **错误**: SubAgent返回的数据不加验证直接引用
- **正确**: Phase 4 事实核查必须对SubAgent数据逐项溯源
- **经验**: AMD Phase 4 的12项核查全部通过，说明SubAgent数据可靠度尚可

### Edit工具匹配失败
- **错误**: old_string不够精确，匹配到多处或零处
- **正确**: 每次Edit前必须先Read确认，old_string包含足够上下文
- **频率**: 高 (每个会话平均2-3次失败)

### 并行Agent prompt质量
- **错误**: 给SubAgent简短prompt → 返回泛泛数据
- **正确**: prompt必须包含具体数据需求列表、Phase前序发现、CQ关联、字符目标
- **经验**: AMD Phase 0 的9个Agent效果好，因为prompt详细度高
