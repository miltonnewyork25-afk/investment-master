## 框架概述

本框架是一套跨行业通用、对"AI叙事噪音"免疫的评估体系，用于系统性评估美股上市公司的AI实施状态和投资价值。

### 核心设计理念

```
"你的任务不是写'AI很重要'的散文，而是把每家公司关于AI的真实进展
压缩成可审计、可比较、可证伪的结论。"
```

### 框架解决的核心问题

1. **公司AI做到什么程度？** → L轴（实施级别）评估
2. **AI对财务的影响？** → S轴（商业兑现阶段）评估
3. **是受益者还是被颠覆者？** → 颠覆分析
4. **是短期炒作还是长期范式？** → 范式评估
5. **有哪些领先信号？** → 领先信号体系
6. **传统护城河如何变化？** → 护城河协同分析
7. **新进入者威胁多大？** → 新进入威胁评估
8. **投资含义是什么？** → 投资转化

## 核心评估维度

### L轴：AI实施级别 (L0-L4)

| 级别 | 名称 | 描述 |
|------|------|------|
| L0 | 观察级 | AI仅用于报表/预测，不影响动作 |
| L1 | 决策支持级 | AI提供建议，人类确认执行 |
| L2 | 受控自动化级 | AI局部自动执行，有规则拦截 |
| L3 | 闭环控制级 | AI形成反馈→决策→执行闭环 |
| L4 | 自主运行级 | AI跨场景自主调度与自恢复 |

### S轴：商业兑现阶段 (S0-S5)

| 阶段 | 名称 | 描述 |
|------|------|------|
| S0 | 叙事期权期 | 宣传与试验多，生产化证据弱 |
| S1 | 试点扩散期 | 多个POC，缺乏统一治理 |
| S2 | 生产嵌入期 | AI进入生产，影响有限 |
| S3 | 系统重构期 | 工作流被AI改写，治理成体系 |
| S4 | 财务兑现期 | AI影响在财报可见 |
| S5 | 边界重写期 | 平台化收税或被去中介化 |

### 五不变量

| 不变量 | 问题 |
|--------|------|
| 可验证性 | AI系统输出是否可验证、可审计？ |
| 治理与责任 | 有没有清晰的权限、回滚、责任边界？ |
| 互补资产 | 是否拥有AI发挥价值所需的关键资产？ |
| 单位经济 | AI规模扩大是更赚钱还是更亏钱？ |
| 边界议价权 | AI让公司收税还是交税？ |

## 投资评级体系

| 评级 | 分数 | 含义 | 建议 |
|------|------|------|------|
| Strong_Beneficiary | 3.5-4.0 | AI强受益者 | 积极配置，可超配 |
| Beneficiary | 2.8-3.5 | AI受益者 | 适度配置 |
| Neutral | 2.2-2.8 | AI中性 | 观望 |
| At_Risk | 1.5-2.2 | AI风险敞口 | 谨慎/减仓 |
| Strong_At_Risk | 1.0-1.5 | AI强风险敞口 | 回避/做空 |

## 抗幻觉机制

### 核心原则

1. **证据优先，叙事后置**
2. **双锚点规则**：业务锚+系统锚同时满足才能升级
3. **Unknown显式化**：不能证明就写unknown
4. **可证伪输出**：next_gate + kill_switches

### 证据分级

| 等级 | 来源 | 可信度 |
|------|------|--------|
| A级 | 监管/正式披露（10-K/10-Q/8-K） | 95%+ |
| B级 | 业绩会/官方材料 | 75-90% |
| C级 | 媒体/二手材料（仅作线索） | 40-70% |

## 质量门控（7条规则）

1. 影响向量≤2
2. 每个分数都有证据等级
3. 无证据高分检查
4. L≥L3需要系统锚点
5. 金融行业必须覆盖模型验证
6. next_gate是可观测事实
7. kill_switches真实有效
