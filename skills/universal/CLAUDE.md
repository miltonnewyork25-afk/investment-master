## ⚠️ 强制执行顺序

本框架体系必须按固定顺序执行，不得跳跃：

```
Phase 1: business_decomposition_protocol_v1.yaml (业务拆解)
   ↓ 质量门控 (财务拆分完整性检验)
Phase 2: option_business_valuation_v1.yaml (期权型业务估值)
   ↓ 质量门控 (估值方法合理性检验)
Phase 3: business_synergy_analysis_v1.yaml (协同分析)
   ↓ 质量门控 (负协同未被忽视)
Phase 4: compound_moat_assessment_v1.yaml (护城河复合评估)
   ↓ 质量门控 (交互机制有因果解释)
Phase 5: conglomerate_risk_framework_v1.yaml (风险与折价)
   ↓ 质量门控 (5类风险全部评估)
Phase 6: weighted_valuation_engine_v1.yaml (综合估值)
   ↓ 质量门控 (估值桥梁逻辑清晰)
```

**铁律**: 不通过当前Phase质量门控，禁止进入下一Phase

## 📊 数据Level强制标注

所有数据必须标注可信度Level，格式严格遵循：

| Level | 标注格式 | 示例 |
|-------|---------|------|
| **A** | `[API: 源名称]` | `ROIC 28.7% [API: FMP key-metrics-ttm]` |
| **B** | `[财报: 期间]` | `储能收入$10.5B [财报: 10-K FY2025]` |
| **C** | `[第三方: 源]` | `员工满意度4.1/5 [第三方: Glassdoor]` |
| **D** | `[分析师: 姓名/机构]` | `目标价$265 [分析师: Harlan Sur/JPM]` |
| **E** | `[估算: 基于XX假设]` | `Robotaxi收入$20B [估算: 基于20%渗透率假设]` |

**Level E强制要求**:
- 必须说明估算方法
- 必须列出关键假设
- 必须标注置信度

**示例（正确）**:
```
Tesla Robotaxi业务估值$10.4B [Level E估算: 基于以下假设]
  关键假设:
  - FSD L4监管批准概率: 60%
  - 2030年渗透率: 20%
  - 单次行程价格: $2.5
  - 年运营成本: $15B
  置信度: 15% (极高不确定性)
```

## 🔄 协同分析保守原则

### 正协同 (Positive Synergies)

**规则**: 根据置信度应用权重

| 置信度 | Base Case权重 | 说明 |
|--------|--------------|------|
| ≥60% | 100% | 高证据支持，全额计入 |
| 50-59% | 75% | 中等证据，部分计入 |
| 40-49% | 50% | 弱证据，保守计入 |
| <40% | 0% | 证据不足，Base Case不计入（Bull Case可考虑） |

**示例**:
```
Cross-sell 汽车→能源: NPV +$16.2B, 置信度40%
→ Base Case调整: +$16.2B × 50% = +$8.1B
→ Bull Case调整: +$16.2B × 100% = +$16.2B
```

### 负协同 (Negative Synergies)

**保守原则**: 即使低置信度也应计入

| 置信度 | Base Case权重 | 说明 |
|--------|--------------|------|
| ≥30% | 100% | 负协同风险真实存在 |
| 20-29% | 75% | 不能忽视的风险 |
| 10-19% | 50% | 低概率但高影响 |
| <10% | 25% | 极低概率但需警示 |

**铁律**: Base Case必须至少识别并量化≥2个负协同

### 期权型协同 (Option-like Synergies)

**示例**: 数据飞轮、网络效应加速

**规则**:
- Base Case权重: **0%** (太不确定)
- Bull Case权重: 50-75%
- 单独标注为"期权价值"

## 💰 估值桥梁铁律

**目的**: 连接SOTP和最终目标价，不允许逻辑跳跃

### 触发条件

```
如果 |最终目标价 - SOTP概率加权| / SOTP概率加权 > 20%
→ 必须提供"估值桥梁"详细解释
```

### 估值桥梁必须包含

1. **SOTP基准值**: 明确标注
2. **调整因子清单**: 逐项列出
3. **量化计算**: 每个因子的数值影响
4. **合理性解释**: 为什么这个调整是合理的

### 模板

```markdown
## 估值桥梁分析

### SOTP概率加权基础
- Bull场景 (20%): $530B
- Base场景 (60%): $440B
- Bear场景 (20%): $334B
- **概率加权目标价**: $437B = $137/股

### 调整因子

| 因子 | 调整幅度 | 计算 | 理由 |
|------|---------|------|------|
| 时间价值（12个月） | +15% | $137 × 1.15 = $158 | 业务年增长15% |
| FSD V13催化剂 | +5% | $158 × 1.05 = $166 | Q2 2026发布预期 |
| Optimus Gen 3催化剂 | +2% | $166 × 1.02 = $169 | H2 2026量产 |

### 调整后目标价
**$169/股** (12个月)

### 与SOTP差距验证
- SOTP: $137/股
- 目标: $169/股
- 差距: +23%
- 合理性: 增长+催化剂，差距合理 ✓
```

**禁止示例**:
```
❌ SOTP $137, 目标价$250, 理由"参考分析师共识"
  → 差距+82%，无详细桥梁，拒绝通过
```

## 🚫 禁止事项清单

### 1. 执行流程禁止

- ❌ **跳过Phase**: 必须按1→2→3→4→5→6顺序执行
- ❌ **跳过质量门控**: 每个Phase完成后必须验证通过标准
- ❌ **Phase内模块跳过**: 例如Phase 3中跳过负协同分析

### 2. 协同分析禁止

- ❌ **忽略负协同**: 必须识别≥2个负协同（蚕食/管理注意力/复杂度）
- ❌ **全是正协同**: 如果分析结果全是正向协同，必定有遗漏
- ❌ **协同幻觉**: 管理层声称但无证据的协同，必须标注置信度<30%

### 3. 估值禁止

- ❌ **Conglomerate折价<10%**: 除非充分理由（如Berkshire模式）
- ❌ **目标价与SOTP差距>20%且无估值桥梁**: 绝对禁止
- ❌ **期权业务不标注置信度**: Robotaxi/Optimus必须标注成功概率

### 4. 数据标注禁止

- ❌ **Level E不说明方法**: 所有估算必须解释"如何得出"
- ❌ **凭印象给数据**: "大约"、"估计"必须转化为Level E标注
- ❌ **数据来源不明**: 每个具体数字必须有Level标注

## ✅ 质量门控自检清单

每个Phase完成后，必须自检以下项目：

### Phase 1 - 业务拆解

```
□ 所有业务独立性≥3/5
□ 财务拆分: Σ业务 = 公司总体 (误差<2%)
□ 数据Level标注完整
□ 期权型业务已标记
```

### Phase 2 - 业务估值

```
□ 每个业务估值方法有充分理由
□ 假设合理性已检验（vs 行业/历史）
□ 期权业务使用rNPV或Black-Scholes
□ 置信度诚实标注
```

### Phase 3 - 协同分析

```
□ 负协同已识别≥2个
□ 协同量化方法清晰可复现
□ 置信度保守标注
□ 协同价值有NPV计算
```

### Phase 4 - 护城河评估

```
□ 交互机制有因果解释（每个强交互≥200字）
□ 持久性评估有证据（4维度）
□ 未重复计价（避免DCF中已含）
```

### Phase 5 - 风险与折价

```
□ 5类风险全部评估
□ 折价有对标参考
□ 折价在10-40%合理区间
```

### Phase 6 - 综合估值

```
□ SOTP计算逻辑一致
□ 估值桥梁清晰（如差距>20%）
□ 敏感性分析完整（≥5个关键假设）
□ Bull/Base/Bear三场景
```

## 📈 持续改进机制

### Reflection循环（每个Phase完成后）

```
1. 识别≥2个深度不足部分
   - 哪些分析停留在表面（Level 2）？
   - 哪些数据缺失关键支撑？

2. 识别≥1个逻辑漏洞
   - 哪些推理链条有跳跃？
   - 哪些假设未经验证？

3. 对每个弱点提出改进
   - 补充数据/机制分析/反证条件

4. 改进后深度≥L3
   - 必须达到机制分析层级
```
