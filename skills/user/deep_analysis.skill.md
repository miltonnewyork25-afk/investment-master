# Deep Analysis Skill

## 触发方式

用户说：`/deep {股票代码}` 或 `深度分析 {公司名}`

## 执行协议

当触发时，强制执行以下流程，**不可跳过任何步骤**：

---

### Phase 1: 第一性原理（在任何数据收集之前）

必须先回答这三个问题，每个≥300字：

```
Q1: 这家公司为什么存在？它解决了什么人类根本需求？
    （不能说"生产产品"，要到达需求本质）

Q2: 如果这家公司明天消失，世界会具体损失什么？
    （列出3-5个具体损失，不能泛泛而谈）

Q3: 这个行业的终局是什么？这家公司在终局中的位置？
    （10-20年后的行业形态）
```

### Phase 2: 护城河深度解构

用Hamilton Helmer 7 Powers逐一验证：

| Power | 有/无 | 强度(1-10) | 趋势(加深/变浅) | 具体证据 |
|-------|------|-----------|----------------|---------|
| 规模经济 | | | | |
| 网络效应 | | | | |
| 反向定位 | | | | |
| 转换成本 | | | | |
| 品牌 | | | | |
| 垄断资源 | | | | |
| 流程优势 | | | | |

然后回答：过去5年护城河加深还是变浅？未来5年呢？

### Phase 3: 产品矩阵分析

| 产品 | 生命周期 | 增长驱动力 | 天花板 | 剩余空间 | 与其他产品关系 |
|------|---------|-----------|-------|---------|---------------|

关键问题：如果砍掉一个产品，其他产品受什么影响？

### Phase 4: 竞争格局深度分析

```
Q1: 谁是真正的竞争对手？（可能不是直觉想到的）
Q2: 竞争对手在哪些维度比这家公司强？
Q3: 竞争对手愿意亏损多少来竞争？
Q4: 有什么新进入者可能从下方颠覆？
Q5: 什么技术变化可能让现有优势失效？
```

### Phase 5: 数据收集

现在才开始调用API：
- FMP API（财务数据）
- 100baggers API（杜邦分析、ROIC、领先指标）

### Phase 6: 分析师框架解构与超越

对每位顶级分析师，必须回答4个问题：

```
1. 他的核心框架是什么？（思维方式，不是结论）
2. 框架基于什么假设？
3. 这些假设什么情况下失效？
4. 他的盲区是什么？
```

然后回答：我比顶级分析师多知道什么？我的独特视角是什么？

### Phase 7: 创新分析尝试

必须尝试至少3种非常规方法：

```
选项：
- 反推估值：市值X隐含了什么假设？
- 期权定价：把某业务当期权估值
- 用户价值法：用户数×终身价值=应得市值？
- 边际分析：某变量变化10%，结果如何？
- 情景模拟：竞争对手做到X，市值变多少？
- 供应链反推：从供应商数据推断公司状况
```

即使数据不全或方法失败，也要记录尝试过程。

### Phase 8: 估值与报告

- SOTP多场景估值
- 目标价必须基于SOTP（差距<20%）
- 质量门控执行并附结果

---

## 输出格式

报告必须包含以下章节（按顺序）：

```
1. 第一性原理思考（三问回答）
2. 护城河深度分析（7 Powers表格+趋势）
3. 产品矩阵分析
4. 竞争格局深度分析
5. 顶级分析师框架解构与超越
6. 创新分析尝试（至少3种）
7. 财务分析（API数据）
8. 估值建模（SOTP）
9. 投资建议
10. 质量门控结果
```

---

## 检查点

在输出报告前，必须确认：

```
□ 第一性原理三问已回答（每问≥300字）
□ 7 Powers表格已完成
□ 护城河动态分析已完成
□ 产品矩阵表格已完成
□ 竞争5问已回答
□ ≥3位分析师框架已解构
□ ≥3种创新方法已尝试
□ SOTP估值已完成
□ 目标价与SOTP差距<20%
```

**任一未完成 = 报告不得输出**
