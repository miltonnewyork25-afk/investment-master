## Purpose

系统评估公司面临的监管风险（反垄断、数据隐私、行业监管等），并将其转化为可量化的估值调整。

传统投资分析往往对监管风险处理粗糙（"存在监管风险"），本框架提供：
1. 监管风险分类与识别
2. 概率×影响的量化评估
3. 时间线与催化剂追踪
4. 估值折价建议

## Assessment Workflow

### Step 1: 风险识别

```yaml
checklist:
  反垄断:
    - 市场份额>40%在任一主要市场？
    - 是否有正在进行的反垄断调查/诉讼？
    - 是否有可能被定义为"看门人"（EU DMA）？
    - 近3年是否有大型收购受阻？

  数据隐私:
    - 是否处理大量个人数据？
    - 是否有跨境数据传输？
    - 是否使用AI进行决策？
    - 近3年是否有数据泄露事件？

  行业特定:
    - 是否需要政府牌照运营？
    - 是否有产品审批依赖？
    - 是否有环保/安全强制标准？

  跨境:
    - 收入地理集中度？
    - 是否有高风险国家业务（中国/俄罗斯）？
    - 是否涉及敏感技术？

  税务:
    - 有效税率vs法定税率差异>10pp？
    - 是否有大量海外现金？
    - 是否有激进的转移定价安排？
```

### Step 2: 概率评估

| 风险等级 | 概率区间 | 判断标准 |
|---------|---------|---------|
| **确定** | 90-100% | 已有裁决/已发生 |
| **很可能** | 60-89% | 调查进行中+历史先例 |
| **可能** | 30-59% | 有调查但结果不明 |
| **不太可能** | 10-29% | 有风险因素但无调查 |
| **罕见** | <10% | 理论风险但无触发信号 |

### Step 3: 影响评估

| 影响等级 | 财务影响 | 典型情景 |
|---------|---------|---------|
| **灾难性** | >30%市值 | 强制拆分/主营业务禁止 |
| **严重** | 15-30%市值 | 大额罚款+业务限制 |
| **中等** | 5-15%市值 | 中等罚款+合规成本 |
| **轻微** | 1-5%市值 | 小额罚款 |
| **可忽略** | <1%市值 | 仅合规成本 |

### Step 4: 风险矩阵

```
              影响
         轻微    中等    严重    灾难
       ┌──────┬──────┬──────┬──────┐
  确定 │  中  │  高  │ 极高 │ 极高 │
       ├──────┼──────┼──────┼──────┤
很可能 │  低  │  中  │  高  │ 极高 │
 概    ├──────┼──────┼──────┼──────┤
 率 可能│ 很低 │  低  │  中  │  高  │
       ├──────┼──────┼──────┼──────┤
不太   │ 很低 │ 很低 │  低  │  中  │
可能   └──────┴──────┴──────┴──────┘
```

### Step 5: 量化估值影响

```yaml
估值调整公式:
  期望监管成本 = Σ(概率_i × 影响_i)

  其中影响包括:
    - 直接罚款
    - 业务限制导致的收入损失
    - 合规成本增加
    - 估值倍数压缩

示例（Google反垄断）:
  搜索案:
    概率: 100%（已裁决）
    影响: 年化$3-5B（数据共享+合规）
    NPV: ~$30-50B

  广告案:
    概率: 40%（可能拆分）
    影响: $200B价值损失（若拆分）
    期望损失: 40% × $200B = $80B

  DMA罚款:
    概率: 60%
    影响: $10-30B
    期望损失: 60% × $20B = $12B

  总期望监管成本: ~$120-140B
  占市值比例: ~3.5%
  建议估值折价: 3-5%
```

## Scoring System: Regulatory Risk Score (RR_Score)

### 评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| **当前案件严重性** | 30% | 正在进行的调查/诉讼 |
| **历史处罚记录** | 20% | 过去5年罚款/处罚 |
| **市场地位风险** | 20% | 份额集中度 |
| **地缘风险** | 15% | 跨境业务敏感度 |
| **合规能力** | 15% | 合规投入/文化 |

### 评分公式

```
RR_Score = 100 - Σ(风险得分 × 权重)

解读:
  80-100: 低监管风险，无需折价
  60-79: 中等风险，建议2-5%折价
  40-59: 高风险，建议5-10%折价
  <40: 极高风险，建议>10%折价或规避
```

### 评分示例（Google）

| 维度 | 风险得分(0-10) | 加权 |
|------|---------------|------|
| 当前案件 | 7 (多案进行中) | 2.1 |
| 历史处罚 | 6 (欧盟多次罚款) | 1.2 |
| 市场地位 | 8 (90%搜索份额) | 1.6 |
| 地缘风险 | 4 (主要在西方) | 0.6 |
| 合规能力 | 3 (投入充足) | 0.45 |

**RR_Score = 100 - (2.1+1.2+1.6+0.6+0.45)×10 = 100 - 59.5 = 40.5** → 高风险区间

## Kill Switches

| ID | 条件 | 触发动作 |
|----|------|---------|
| **KS-RR-001** | 强制拆分裁决确认 | 立即重估整个投资论点 |
| **KS-RR-002** | 罚款>年利润50% | 检查偿付能力和现金流 |
| **KS-RR-003** | 被禁止进入主要市场 | 收入预测需重做 |
| **KS-RR-004** | 连续3年DMA违规 | 系统性违规风险需额外折价 |

## Evidence Thresholds

| 判断类型 | 最低证据要求 |
|---------|------------|
| 概率>60% | 需有官方调查启动 |
| 影响>$10B | 需有先例或官方声明 |
| 拆分可能 | 需有监管机构明确提出 |

---

## Red Flags

| 红旗 | 说明 |
|------|------|
| 🚩 低估历史重演 | 同类公司类似案件结果应参考 |
| 🚩 忽视地缘联动 | 一地裁决可能触发全球跟进 |
| 🚩 时间乐观 | 案件往往比预期拖更久 |
| 🚩 管理层淡化 | 管理层总说"我们会赢" |

## Regional Regulatory Landscape

### 主要监管机构对比

| 地区 | 机构 | 风格 | 处罚力度 |
|------|------|------|---------|
| 美国 | DOJ/FTC | 诉讼导向 | 高（可拆分） |
| 欧盟 | DG Comp | 罚款导向 | 极高（10%收入） |
| 中国 | SAMR | 政策导向 | 中高 |
| 英国 | CMA | 独立严格 | 中高 |

### DMA "看门人"框架

| 标准 | 阈值 |
|------|------|
| 市值 | >€75B |
| 用户数 | >45M欧盟用户 |
| 业务用户 | >10K欧盟业务用户 |
| 核心平台 | 搜索/社交/通讯/广告等 |

被指定为"看门人"后：
- 禁止自我优待
- 必须数据可携带
- 违规罚款最高10%全球收入
- 累犯可达20%
