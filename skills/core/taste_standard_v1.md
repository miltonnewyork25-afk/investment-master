# 品味标准 v1.0

> 品味不是主观的。它是对"什么有效、什么无效"的深度认知积累。

**版本**: 1.0  
**创建日期**: 2026-01-29  
**来源**: 台积电报告升级复盘 (91.75 → 96 → 98.5)

---

## 一、"好"与"卓越"的具体对比

### 1.1 开场

| 维度 | 好 (90分) | 卓越 (96+) |
|------|----------|-----------|
| 开头 | 简短摘要 | 30秒速览仪表盘 |
| 第一印象 | "这是一份完整的报告" | "我立刻知道值不值得读" |
| 读者行为 | 需要翻几页才能找到结论 | 第一眼就看到结论 |

**核心差异**: 卓越报告尊重读者的时间，30秒内给出答案。

### 1.2 数据呈现

| 维度 | 好 (90分) | 卓越 (96+) |
|------|----------|-----------|
| 核心数字 | 嵌在段落中 | 突出框单独展示 |
| 判断 | 文字描述 | 信号灯系统 🟢🟡🔴 |
| 数据来源 | 偶尔标注 | 每个数据都标注 |
| 推测内容 | 与事实混在一起 | 明确标注[分析推测] |

**核心差异**: 卓越报告的每个数据都有出处，每个判断都有视觉标识。

### 1.3 结构

| 维度 | 好 (90分) | 卓越 (96+) |
|------|----------|-----------|
| 组织方式 | 按主题分章节 | 按阅读深度分层 |
| 操作建议 | 文字描述 | 决策树可视化 |
| 导航 | 目录 | 目录 + 分层阅读指引 |

**核心差异**: 卓越报告支持"扫一眼→快速读→深入研"三层阅读。

### 1.4 验证

| 维度 | 好 (90分) | 卓越 (96+) |
|------|----------|-----------|
| 观点来源 | 仅自己分析 | 自己分析 + 投资者验证 |
| 可信度 | 依赖分析质量 | 分析质量 + 外部背书 |
| 完整性 | 可能忽略反面观点 | 多空双方观点都呈现 |

**核心差异**: 卓越报告不只说"我觉得"，还要说"看看聪明钱怎么投票"。

## 二、文字密度的判断标准

### 2.1 段落长度

| 等级 | 描述 | 判断 |
|------|------|------|
| ✅ 好 | 平均每段 ≤4 句 | 易于扫读 |
| ⚠️ 警告 | 平均每段 5-7 句 | 稍显沉重 |
| ❌ 差 | 平均每段 >7 句 | 需要拆分 |

### 2.2 信息密度

| 等级 | 描述 | 判断 |
|------|------|------|
| ✅ 好 | 每段有1个核心观点 | 清晰 |
| ⚠️ 警告 | 每段有2-3个观点 | 可接受 |
| ❌ 差 | 每段>3个观点 | 需要拆分 |

### 2.3 重复度

| 等级 | 描述 | 判断 |
|------|------|------|
| ✅ 好 | 每个核心观点只出现1次 | 精炼 |
| ⚠️ 警告 | 同一观点出现2次 | 可接受（强调） |
| ❌ 差 | 同一观点出现3+次 | 需要删减 |

### 2.4 自检方法

```
文字密度自检：
1. 随机选3个段落，数句子数
   - 平均≤4句 → 通过
   - 平均>4句 → 需要拆分

2. 搜索关键结论是否重复
   - 出现≤2次 → 通过
   - 出现>2次 → 需要删减

3. 检查每个段落是否只有1个核心观点
   - 是 → 通过
   - 否 → 需要拆分
```

## 三、视觉平衡的判断标准

### 3.1 密度规则

| 指标 | 标准 | 判断 |
|------|------|------|
| 每1000字可视化 | ≥1个 | 基准 |
| 每章节框线元素 | ≤3个 | 避免疲劳 |
| 全篇可视化总数 | ≤15个 | 适度 |

### 3.2 视觉元素分布

**好的分布**:
```
章节1: 速览仪表盘 + 信号灯
章节2: 因果链图
章节3: 双栏对比 + 关键数字框
章节4: 决策树
章节5: 投资者仪表盘
```

**差的分布**:
```
章节1: 仪表盘 + 信号灯 + 表格 + 图表 (视觉过载)
章节2: 纯文字 (视觉断层)
章节3: 纯文字 (视觉断层)
```

### 3.3 自检方法

```
视觉平衡自检：
1. 数每个章节的框线元素
   - 每章节≤3个 → 通过
   - 某章节>3个 → 需要精简

2. 检查是否有连续2个章节无可视化
   - 无 → 通过
   - 有 → 需要补充

3. 数全篇可视化总数
   - ≤15个 → 通过
   - >15个 → 需要精简
```

## 四、"投资者视角"的检验方法

### 4.1 投资者扫读测试

模拟一个忙碌的投资者，只有30秒时间：

| 问题 | 能否30秒内找到？ |
|------|-----------------|
| 这公司值不值得研究？ | ✅/❌ |
| 买入/持有/观望建议？ | ✅/❌ |
| 最大风险是什么？ | ✅/❌ |
| 估值是否合理？ | ✅/❌ |

**通过标准**: 4个问题都能30秒内找到 → 通过

### 4.2 信息完整性测试

投资者做决策需要的信息：

| 信息 | 报告中是否完整？ |
|------|-----------------|
| 公司护城河 | ✅/❌ |
| 竞争格局 | ✅/❌ |
| 周期定位 | ✅/❌ |
| 估值逻辑 | ✅/❌ |
| 风险因素 | ✅/❌ |
| 操作建议 | ✅/❌ |

**通过标准**: 6项都完整 → 通过

### 4.3 外部验证测试

| 问题 | 是否有回答？ |
|------|------------|
| 顶级机构怎么看？ | ✅/❌ |
| 聪明钱在增持还是减持？ | ✅/❌ |
| 有没有相反观点？ | ✅/❌ |

**通过标准**: 至少2项有回答 → 通过（如有数据）

### 4.4 总体评判

```
投资者视角总评：
- 扫读测试通过 + 完整性测试通过 + 验证测试通过 → 卓越
- 扫读测试通过 + 完整性测试通过 → 优秀
- 完整性测试通过 → 良好
- 完整性测试不通过 → 需要补充
```

## 五、参照物借鉴要点

### 5.1 Bridgewater Daily Observations

**借鉴要点**:
- 开头直接给结论（"我们认为..."）
- 数据可视化简洁有力
- 逻辑链清晰（因为A → 所以B → 因此C）
- 概率思维（"我们认为有70%可能..."）

**应用场景**: 宏观分析、周期判断

### 5.2 ARK Invest Big Ideas

**借鉴要点**:
- 视觉设计现代、易读
- 大胆预测但有数据支撑
- 叙事简洁有力
- 图表丰富但不杂乱

**应用场景**: 趋势分析、新兴技术

### 5.3 《经济学人》

**借鉴要点**:
- 标题精炼传达观点
- 第一段就给结论
- 段落短小精悍
- 复杂问题用简单语言解释

**应用场景**: 整体写作风格

### 5.4 实践对照表

| 参照物 | 最值得学的 | 应用到报告的 |
|--------|-----------|-------------|
| Bridgewater | 结论先行 + 概率思维 | 速览仪表盘 + 情景分析 |
| ARK | 视觉设计 + 大胆预测 | 信号灯系统 + 因果链图 |
| 经济学人 | 简洁表达 + 短段落 | 文字密度控制 |

## 六、品味培养路径

### 6.1 短期（1-2周）

1. **收集好例子**
   - 保存3-5份"卓越"报告作为参照
   - 分析其视觉设计和信息组织

2. **建立检查清单**
   - 使用本文档的自检方法
   - 每份报告完成后自评

### 6.2 中期（1-2月）

3. **建立反例库**
   - 收集"问题报告"作为反面教材
   - 分析其失败原因

4. **迭代框架**
   - 根据实践经验更新标准
   - 增加新的有效模板

### 6.3 长期（持续）

5. **用户反馈**
   - 收集读者反馈
   - 追踪阅读完成率

6. **跨领域学习**
   - 学习信息设计领域的最佳实践
   - 学习数据可视化的专业方法
