# 深度思考协议 v1.0

## 设计原则

这不是"建议"或"最佳实践"。
这是**强制执行的思维流程**。
每个问题必须有≥200字的回答，否则分析不合格。

---

## 第一阶段：第一性原理解构（必须先完成）

在调用任何API或搜索之前，必须回答：

### 1.1 存在性问题
```
问：这家公司为什么存在？它解决了什么根本问题？
要求：不能用"提供产品/服务"这种套话。必须到达人类需求的根本层面。

示例（错误）：Apple存在是因为它生产手机和电脑。
示例（正确）：Apple存在是因为人类需要延伸认知和沟通能力的工具，
而Apple提供的是"无摩擦"的工具体验——用户不需要理解技术就能
获得技术的好处。这种"技术民主化"是Apple存在的根本价值。
```

### 1.2 不可替代性问题
```
问：如果这家公司明天消失，世界会损失什么？
要求：具体描述损失，不能泛泛而谈。

示例（错误）：用户会失去iPhone。
示例（正确）：如果Apple消失：
1. 24亿设备的生态系统断裂，用户的照片/健康数据/工作流程需要重建
2. 隐私计算的标杆消失，行业可能倒退向数据收割模式
3. 硬件设计的审美标准失去锚点
4. 开发者失去最高ARPU的分发渠道
```

### 1.3 终局问题
```
问：这个行业的终局是什么？这家公司在终局中的位置是什么？
要求：描述10-20年后的行业形态，以及公司的可能位置。
```

## 第二阶段：护城河深度解构（必须完成）

### 2.1 Hamilton Helmer 7 Powers逐一验证

对每一个Power，必须回答：有没有？强度如何（1-10）？在加深还是变浅？

| Power | 定义 | 验证问题 |
|-------|------|---------|
| **规模经济** | 单位成本随规模下降 | 公司的规模优势体现在哪里？竞争对手能追上吗？ |
| **网络效应** | 用户越多价值越大 | 新增一个用户如何提升现有用户价值？强度如何量化？ |
| **反向定位** | 新商业模式让在位者无法跟随 | 公司做了什么让竞争对手"不能"或"不愿"跟随？ |
| **转换成本** | 用户离开的成本 | 用户离开需要付出什么代价？量化这个代价。 |
| **品牌** | 可以收取溢价 | 溢价是多少？溢价的来源是什么？ |
| **垄断资源** | 独占的稀缺资源 | 公司有什么竞争对手无法获得的资源？ |
| **流程优势** | 组织能力差异 | 公司有什么竞争对手难以复制的内部能力？ |

### 2.2 护城河动态分析
```
问：过去5年，护城河是加深了还是变浅了？未来5年呢？
要求：用具体证据支撑判断。
```

## 第三阶段：产品矩阵深度分析（必须完成）

### 3.1 产品生命周期定位

对每个主要产品线，必须回答：

```
| 产品 | 生命周期阶段 | 增长驱动力 | 天花板在哪 | 还有多少空间 |
|------|------------|-----------|-----------|-------------|
```

### 3.2 产品间关系分析
```
问：各产品之间是互补还是替代？协同效应如何量化？
问：如果砍掉一个产品线，其他产品会受什么影响？
```

### 3.3 定价权分析
```
问：公司有定价权吗？证据是什么？
问：如果提价10%，销量会下降多少？（价格弹性估算）
```

---

## 第四阶段：竞争格局深度分析（必须完成）

### 4.1 竞争对手真正的威胁
```
问：谁是真正的竞争对手？（可能不是直觉想到的那个）
问：竞争对手在哪些维度上比这家公司强？
问：竞争对手的战略意图是什么？他们愿意亏损多少来竞争？
```

### 4.2 颠覆性威胁
```
问：有什么新进入者可能从下方颠覆这家公司？
问：有什么技术变化可能让这家公司的优势失效？
```

---

## 第五阶段：分析师学习与超越（必须完成）

### 5.1 不是罗列观点，而是理解框架
```
对每个顶级分析师，必须回答：
1. 他的核心框架是什么？（不是结论，是思维方式）
2. 他的框架基于什么假设？
3. 这些假设在什么情况下会失效？
4. 他的盲区是什么？（他忽略了什么重要因素？）
```

### 5.2 超越分析师
```
问：我比顶级分析师多知道什么？我的独特视角是什么？
要求：必须找到至少1个分析师忽略的重要因素。
```

---

## 第六阶段：创新分析尝试（必须完成）

### 6.1 至少尝试3种非常规分析方法
```
示例：
- 反推估值：如果市值是X，隐含了什么假设？
- 期权定价：把某个业务当期权，用期权模型估值
- 用户价值法：用户数×终身价值=应得市值？
- 边际分析：如果某个变量变化10%，结果如何？
- 情景模拟：如果竞争对手做到X，公司市值会变成多少？
```

### 6.2 即使失败也要记录
```
如果数据不足或方法不成熟，在报告最后说明：
"尝试了X方法，但因为Y原因未能完成，未来可以通过Z方式改进"
```

---

## 执行检查点

在写报告之前，必须确认以下内容已完成：

```
□ 第一性原理三问已回答（每问≥200字）
□ 7 Powers逐一验证完成
□ 护城河动态分析完成
□ 产品矩阵分析完成
□ 竞争格局深度分析完成
□ 至少3位分析师的框架解构完成
□ 至少3种创新分析方法已尝试
□ 所有分析有具体证据支撑
```

**任何一项未完成，报告不得输出。**

## 与现有系统的关系

```
深度思考协议（本文档）
   ↓ 强制执行
CLAUDE.md v11（四条铁律）
   ↓ 原则指导
Quality Gate v2（事后检查）
   ↓ 最终验证
输出报告
```

深度思考协议是**过程控制**，不是事后检查。
它在分析的每个阶段强制深度思考，而不是等到最后再"检查"。
