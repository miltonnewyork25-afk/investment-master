## 🔄 多Agent协作基础设施 (v1.0)

**创建时间**: 2026-02-07
**用途**: 支持多Agent并行协作的标准化工具和流程

### 已部署组件

| 组件 | 路径 | 用途 |
|------|------|------|
| Fast Gate 质量门控 | `tests/research_fast.sh` | 自动检查字符数/标注密度/禁止词/必需章节 |
| 协调状态文件 | `STATUS.md` | 多Agent共享状态+失败方法记录+下一步优先级 |
| 任务锁目录 | `current_tasks/` | Agent选任务前加锁，避免重复工作 |
| Agent日志目录 | `agent_logs/` | SubAgent详细输出持久化 |

### Fast Gate 门控清单 (7项)

| 门控 | 检查内容 | Tier 3 标准 |
|------|---------|------------|
| G1 | 字符数 (wc -m) | ≥ 目标值 |
| G2 | 数据标注密度 | ≥ 15个/万字符 |
| G3 | 必需章节 (免责声明/目录) | 必须存在 |
| G4 | Mermaid图表 | ≥ 3个 |
| G5 | 禁止词 | 0个 |
| G6 | 标注格式 | 无未闭合标注 |
| G7 | Kill Switch/看空 (Phase 4) | KS≥15, 看空≥8 |

### 标准工作循环 (每轮)

```
Step 0. 读 STATUS.md + progress.md → 找最明确的下一件事
Step 1. 选任务 → current_tasks/<task>.txt 加锁 → push
Step 2. 最小改动实施
Step 3. ./tests/research_fast.sh <file> <min_chars> → 必须通过
Step 4. git commit (清晰message)
Step 5. pull → merge → 再跑gate → push
Step 6. 删除锁文件 → push → 更新 STATUS.md
```

### 卡住协议

> 20-30分钟无进展 → 停止 → 写入 STATUS.md (尝试了什么/失败表现/怀疑原因) → 切换到可验证子任务
