## 行业识别决策树

### 步骤1：获取基础信息

```
输入公司名称/股票代码
  ↓
自动查询：
  ├─ GICS行业代码（彭博/FactSet）
  ├─ 10-K "Business" 章节
  ├─ 主营业务收入构成
  └─ 公司自我定位

优先级：
  1. GICS代码（最权威）
  2. 收入构成>50%的业务
  3. 公司官方定位
```

### 步骤2：匹配至8大行业

```
决策流程图：

公司主营业务是什么？
  │
  ├─ 实体产品制造？ → 行业1：制造/硬件
  │   例：特斯拉、苹果、波音、海尔
  │
  ├─ 软件订阅/云服务？ → 行业2：SaaS/订阅
  │   例：微软、Salesforce、Adobe、Zoom
  │
  ├─ 持有并出租物业？ → 行业3：REIT
  │   例：Prologis、Simon Property、万科泊寓
  │
  ├─ 连接买卖双方平台？ → 行业4：平台/市场
  │   例：亚马逊、阿里巴巴、Uber、美团
  │
  ├─ 开采自然资源？ → 行业5：E&P/资源
  │   例：埃克森美孚、必和必拓、中石油
  │
  ├─ 存贷汇金融业务？ → 行业6：金融/银行
  │   例：摩根大通、工商银行、高盛
  │
  ├─ 药物研发/销售？ → 行业7：生物医药
  │   例：辉瑞、Moderna、恒瑞医药
  │
  └─ 消费品牌/零售？ → 行业8：消费品ToC
      例：可口可乐、耐克、星巴克、李宁
```

### 步骤3：边界情况处理

**多业务公司**:
```
苹果：硬件（iPhone 60%）+ 服务（20%）+ 其他
  ├─ 主分类：行业1（制造/硬件）
  ├─ 次要分析：服务业务单独拆分，用SaaS模块分析
  └─ 估值：SOTP（各业务独立估值）

亚马逊：电商平台（50%）+ AWS云服务（15%）+ 广告
  ├─ 主分类：行业4（平台/市场）
  ├─ 次要分析：AWS用SaaS模块，广告用平台模块
  └─ 估值：SOTP
```

**新兴行业**:
```
如无法归入8大行业：
  ├─ 找最接近的行业（例：AI公司→SaaS）
  ├─ 混合使用2个行业模块
  └─ 在报告中说明行业归类依据
```

## 8大行业详细指南

### 行业1：制造/硬件

#### 识别特征
- 生产实体产品（汽车/手机/家电/机械）
- 有工厂、产能、供应链
- BOM成本占比高（>40%）

#### GICS代码
- 25101010 (Automobiles)
- 25202010 (Household Durables)
- 201030 (Capital Goods)
- 452010 (Tech Hardware)

#### 核心命题模板

```markdown
1. **产品竞争力**: [产品X]相对竞品的技术/成本优势是什么？可持续吗？
2. **定价权**: ASP下降是主动降价抢份额，还是被迫跟随？
3. **成本曲线**: 单位成本下降空间还有多少？（学习曲线）
4. **产能利用率**: 当前利用率%？未来需要扩产还是已过剩？
5. **供应链风险**: 关键零部件是否受制于人？（芯片短缺/稀土）
```

#### 必做分析模块

| 模块 | 内容 | 字数 | 数据要求 |
|------|------|------|---------|
| PVM拆解 | Price × Volume × Mix | 1,500 | ✅ 财报数据 + 📊 ASP估算 |
| 单位经济学 | 每辆/每台的收入成本 | 1,200 | ✅ BOM报告 or 📊 同业推断 |
| 产能分析 | 利用率/扩产计划/Capex | 1,000 | ✅ 财报"Capex"章节 |
| 供应链 | Tier 1/2/3穿透 | 1,500 | 📊 供应商财报 + 行业报告 |
| 竞争对标 | ≥3竞品详细对比 | 1,500 | ✅ 竞品财报 |

#### 关键指标（KPI）

```
财务指标：
  • 毛利率趋势（环比/同比）
  • 单位经济学：Revenue/Unit, COGS/Unit
  • Capex占收入比例
  • 库存周转天数（DOI）
  • 应付账款天数（DPO）

运营指标：
  • 产能利用率%
  • 产量增速（YoY）
  • ASP趋势
  • 产品Mix（高/中/低端占比）
  • 学习曲线系数（每翻倍成本下降X%）

竞争指标：
  • 市场份额%
  • 相对竞品ASP溢价/折价
  • 技术差距指数（自定义）
```

#### 数据源清单

```
✅ Tier 1（官方）:
  • 10-K/10-Q "Segment Information"
  • 财报电话会议（产量、ASP指引）
  • 交付/销量月报（Tesla IR、中汽协）

📊 Tier 2（估算）:
  • 拆解报告（IHS Markit BOM分析）
  • 供应商财报推断
  • 可比公司类比

🎯 Tier 3（情景）:
  • 未来产能假设
  • 新产品定价预测
```

#### 估值方法优先级

1. **DCF**（首选）- 基于未来现金流
2. **可比公司P/E, EV/EBITDA**（辅助）
3. **SOTP**（多业务公司必用）

### 行业2：SaaS/订阅

#### 识别特征
- 软件许可/云服务订阅收入
- 经常性收入（ARR/MRR）占比>80%
- 客户留存是核心（NRR）

#### GICS代码
- 451020 (Software)
- 451030 (IT Services)
- 50202010 (Publishing)

#### 核心命题模板

```markdown
1. **增长质量**: ARR增速是新客户驱动，还是老客户扩展？
2. **留存健康度**: NRR（Net Revenue Retention）>110%吗？队列留存改善？
3. **单位经济学**: LTV/CAC比值>3吗？Payback Period<12个月？
4. **Rule of 40**: 增长+利润率≥40%？处于什么阶段？
5. **竞争壁垒**: 客户转换成本（数据/集成/习惯）有多强？
```

#### 必做分析模块

| 模块 | 内容 | 字数 | 数据要求 |
|------|------|------|---------|
| ARR拆解 | New + Expansion - Churn | 1,500 | ✅ 财报披露 or 📊 推算 |
| 队列分析 | 留存曲线（Cohort Retention） | 1,200 | 📊 基于Churn Rate建模 |
| Rule of 40 | 增速% + FCF Margin% | 800 | ✅ 财报计算 |
| 单位经济学 | CAC/LTV/Payback | 1,200 | 📊 S&M÷新客户数 |
| 竞争对标 | NRR/Magic Number对比 | 1,000 | ✅ 竞品财报 |

#### 关键指标（KPI）

```
增长指标：
  • ARR增速（YoY）
  • 新客户ARR vs 扩展ARR占比
  • Logo增速（客户数）
  • ARPU（人均收入）

留存指标：
  • NRR（Net Revenue Retention）
  • Gross Churn %
  • Logo Churn vs Revenue Churn
  • 队列留存率（Cohort-specific）

效率指标：
  • Rule of 40（增速+FCF%）
  • Magic Number（ARR增长/S&M）
  • CAC Payback Period（月）
  • LTV/CAC比值

盈利指标：
  • Gross Margin %（目标>70%）
  • Operating Margin %
  • FCF Margin %
```

#### 数据源清单

```
✅ Tier 1:
  • 10-Q "Revenue" - ARR披露（部分公司）
  • S-1文件（上市前详细披露）
  • 投资者演示文稿（Investor Deck）

📊 Tier 2:
  • 基于Deferred Revenue推算ARR
  • S&M支出÷行业平均转化率 = 新客户数
  • Churn Rate基于队列模型反推

🎯 Tier 3:
  • 未来NRR改善假设
  • TAM渗透率预测
```

#### 估值方法优先级

1. **EV/ARR倍数**（首选，可比公司）
2. **DCF**（辅助，基于ARR预测）
3. **Rule of 40分层估值**（增长型 vs 效率型）

### 行业3：REIT（房地产投资信托）

#### 识别特征
- 持有并出租物业
- 90%收入必须分红
- FFO是核心盈利指标

#### GICS代码
- 60 (Real Estate)

#### 核心命题模板

```markdown
1. **物业质量**: 核心资产位置/租户质量/租约结构如何？
2. **NAV折溢价**: 当前P/NAV是折价还是溢价？合理吗？
3. **利率敏感性**: 加息对估值和融资成本影响几何？
4. **同店NOI增长**: 有机增长（租金上涨）还是靠收购？
5. **分红可持续性**: AFFO覆盖率>1.2吗？
```

#### 必做分析模块

| 模块 | 内容 | 字数 | 数据要求 |
|------|------|------|---------|
| NAV模型 | 物业估值-净负债 | 1,500 | 📊 Cap Rate法 or DCF |
| FFO/AFFO | 调整后现金流 | 1,200 | ✅ 财报披露 |
| 物业组合 | 按类型/地区分布 | 1,000 | ✅ 补充材料 |
| 利率敏感性 | 久期分析 | 1,000 | 📊 债务结构建模 |
| 分红分析 | 覆盖率/增长历史 | 800 | ✅ 历史分红数据 |

#### 关键指标（KPI）

```
估值指标：
  • P/NAV（市值/资产净值）
  • Cap Rate（NOI/物业价值）
  • Implied Cap Rate（市场隐含）

盈利指标：
  • FFO/股
  • AFFO/股
  • Same-Store NOI Growth %

负债指标：
  • Debt/EBITDA倍数
  • 固定利率债务占比
  • 平均债务成本%
  • 债务到期分布

运营指标：
  • 出租率（Occupancy %）
  • 平均租金（$/平方英尺）
  • 租约到期集中度
```

#### 数据源清单

```
✅ Tier 1:
  • 10-K "Property Portfolio"
  • 补充材料（Supplemental Package）
  • 分红公告

📊 Tier 2:
  • Cap Rate基于同类物业交易
  • NAV基于DCF估值

🎯 Tier 3:
  • 未来租金增长假设
  • 利率变化情景
```

#### 估值方法优先级

1. **NAV模型**（首选）
2. **P/FFO倍数**（可比公司）
3. **Dividend Yield**（分红收益率对比）

### 行业4：平台/市场

#### 识别特征
- 连接买卖双方的双边市场
- GMV × Take Rate = 收入
- 网络效应强

#### GICS代码
- 25502010 (Internet & Direct Marketing Retail)
- 50203010 (Interactive Media)

#### 核心命题模板

```markdown
1. **GMV增长来源**: 新用户、活跃度提升，还是客单价？
2. **Take Rate趋势**: 是否有提价空间？竞争压制？
3. **网络效应**: 用户增长^2 vs GMV增长关系验证
4. **双边平衡**: 买家/卖家增速匹配吗？流动性充足？
5. **赢家通吃**: 市场是否已进入寡头稳定期？
```

#### 必做分析模块

| 模块 | 内容 | 字数 | 数据要求 |
|------|------|------|---------|
| GMV×Take Rate | 拆解至买家/订单/客单价 | 1,500 | ✅ 财报披露 |
| 双边UE | 买家CAC/LTV，卖家经济学 | 1,500 | 📊 S&M推算 |
| 网络效应量化 | 梅特卡夫定律验证 | 1,200 | 📊 用户vs GMV回归 |
| 竞争格局 | CR3/HHI指数 | 1,000 | ✅ 行业报告 |

#### 关键指标（KPI）

```
规模指标：
  • GMV（总交易额）
  • 活跃买家/卖家数
  • 订单量
  • 客单价（AOV）

货币化指标：
  • Take Rate %
  • 广告收入占GMV %
  • 增值服务占比

用户指标：
  • MAU/DAU
  • 人均订单频次
  • 留存率（买家/卖家）

效率指标：
  • 买家CAC
  • 买家LTV
  • 流动性指标（等待时间/匹配率）
```

#### 数据源清单

```
✅ Tier 1:
  • 10-Q "Operating Metrics"
  • 投资者日演示

📊 Tier 2:
  • Take Rate = 收入/GMV
  • CAC = S&M/新增用户

🎯 Tier 3:
  • TAM渗透率预测
  • 网络效应加速假设
```

#### 估值方法优先级

1. **EV/GMV倍数**（首选）
2. **P/S倍数**（可比公司）
3. **DCF**（基于GMV×Take Rate预测）

### 行业5：E&P/资源

#### 识别特征
- 开采石油/天然气/矿产
- 收入=产量×价格
- 储量是核心资产

#### GICS代码
- 10102010 (Oil & Gas E&P)
- 15104010 (Diversified Metals & Mining)

#### 核心命题模板

```markdown
1. **成本曲线位置**: 全周期成本在行业什么分位？
2. **储量质量**: 1P储量够开采几年？RRR>100%吗？
3. **价格敏感性**: 盈亏平衡价格是多少？
4. **资本纪律**: Capex控制如何？FCF分红比例？
5. **ESG风险**: 碳排放/社区关系/监管压力？
```

#### 必做分析模块

| 模块 | 内容 | 字数 | 数据要求 |
|------|------|------|---------|
| 储量评估 | 1P/2P/3P，NAV计算 | 1,500 | ✅ 10-K披露 |
| 成本曲线 | 全周期成本vs油价 | 1,200 | ✅ 公司披露+行业对标 |
| 生产指标 | 产量/衰减率/Capex效率 | 1,000 | ✅ 运营数据 |
| 敏感性分析 | 油价±$10影响 | 1,000 | 📊 敏感性表 |

#### 关键指标（KPI）

```
储量指标：
  • 1P储量（百万桶或吨）
  • R/P Ratio（储量/产量）
  • RRR（储量替换率%）

生产指标：
  • 日产量（桶/日 or 吨/年）
  • 自然衰减率%
  • 产量增速

成本指标：
  • All-in Cost（$/桶）
  • 现金成本（$/桶）
  • Breakeven Price

资本效率：
  • Capex/产量
  • ROACE（资本回报率）
  • FCF Yield %
```

#### 数据源清单

```
✅ Tier 1:
  • 10-K "Proved Reserves"
  • 生产报告（季度/年度）
  • 油价：EIA/Bloomberg

📊 Tier 2:
  • 成本曲线基于公司披露+行业报告
  • 未来产量基于衰减率模型

🎯 Tier 3:
  • 油价情景（$60/$80/$100）
  • 新油田产量假设
```

#### 估值方法优先级

1. **NAV（储量估值）**（首选）
2. **EV/EBITDA**（可比公司）
3. **Dividend Yield**（现金牛公司）

### 行业6：金融/银行

#### 识别特征
- 存贷汇业务
- 杠杆经营（资产/权益>10x）
- 受严格监管（资本充足率）

#### GICS代码
- 4010 (Banks)
- 4020 (Diversified Financials)

#### 核心命题模板

```markdown
1. **NIM趋势**: 净息差扩张还是收窄？利率环境影响？
2. **资产质量**: NPL率趋势？拨备覆盖率充足吗？
3. **资本充足**: CET1比率高于监管多少？分红空间？
4. **ROE驱动**: 来自杠杆、效率，还是风险定价？
5. **周期位置**: 信贷周期哪个阶段？拨备前瞻性？
```

#### 必做分析模块

| 模块 | 内容 | 字数 | 数据要求 |
|------|------|------|---------|
| NIM拆解 | 资产收益率-负债成本 | 1,500 | ✅ 财报详细披露 |
| 资产质量 | NPL/拨备/信贷成本 | 1,200 | ✅ 监管报告 |
| 资本管理 | CET1/RWA/分红 | 1,000 | ✅ 资本充足率报告 |
| 杜邦分解 | ROE=ROA×杠杆 | 1,000 | ✅ 财务计算 |

#### 关键指标（KPI）

```
盈利指标：
  • NIM（净息差%）
  • ROA（资产回报率%）
  • ROE（股东回报率%）
  • 成本收入比%

资产质量：
  • NPL Ratio %
  • 拨备覆盖率%
  • 信贷成本（bps）
  • 贷款迁徙率

资本指标：
  • CET1 Ratio %
  • Tier 1 Ratio %
  • RWA（风险加权资产）
  • 杠杆倍数

效率指标：
  • 成本收入比%
  • 人均利润
  • 分支机构ROA
```

#### 数据源清单

```
✅ Tier 1:
  • 10-K "MD&A - Net Interest Income"
  • 巴塞尔III披露报告
  • 分红政策公告

📊 Tier 2:
  • NIM分解（基于资产负债表）
  • 信贷成本趋势预测

🎯 Tier 3:
  • 利率情景（+/-100bps）
  • 信贷周期假设
```

#### 估值方法优先级

1. **P/B倍数**（首选，银行通用）
2. **P/E倍数**（辅助）
3. **DDM（股息贴现模型）**（高分红银行）

### 行业7：生物医药

#### 识别特征
- 药物研发/销售
- 研发周期10-15年
- 临床试验成功率低

#### GICS代码
- 352010 (Pharmaceuticals)
- 352020 (Biotechnology)

#### 核心命题模板

```markdown
1. **管线价值**: rNPV加总vs市值，低估还是高估？
2. **临床风险**: 关键产品处于什么阶段？数据何时公布？
3. **专利悬崖**: 重磅产品何时到期？仿制药影响？
4. **现金跑道**: 够烧几个季度？融资计划？
5. **商业化能力**: 自建销售团队还是授权？
```

#### 必做分析模块

| 模块 | 内容 | 字数 | 数据要求 |
|------|------|------|---------|

## 特殊场景处理

### 场景1：多业务集团公司

**示例**: 亚马逊（电商+AWS+广告）

**处理方式**:
1. 识别主业务（收入占比>50%）→ 确定主行业分类
2. 对每个业务使用对应行业模块分析
3. 估值：必须用SOTP（Sum-of-the-Parts）

```
亚马逊估值示例：
  ├─ 电商业务：用"平台/市场"模块 → EV/GMV估值 = $800B
  ├─ AWS业务：用"SaaS"模块 → EV/ARR估值 = $600B
  ├─ 广告业务：用"平台"模块 → P/S估值 = $200B
  ├─ 其他业务：$50B
  └─ 总估值 = $1,650B
      减：集团折价10%（-$165B）
      = 目标估值 $1,485B
```

### 场景2：新兴行业（AI、元宇宙、Web3等）

**处理方式**:
1. 找最接近的传统行业类比
   - AI公司 → SaaS（按ARR估值）
   - 元宇宙平台 → 平台/市场（按MAU估值）
   - 加密交易所 → 平台（按交易量估值）

2. 调整关键指标
   - AI：用Token使用量替代ARR
   - 元宇宙：用虚拟土地交易替代GMV

3. 风险溢价
   - 折现率WACC +2-5%（技术不确定性）
   - 情景分析权重偏向保守场景

### 场景3：周期股（钢铁、航空、酒店）

**处理方式**:
1. 识别周期位置
   - 当前处于周期顶部/底部/中部？
   - 使用领先指标（PMI、运价指数等）

2. 归一化盈利
   - 用周期平均EBITDA，而非当前TTM
   - 避免在顶部用高P/E，底部用低P/E误导

3. 估值方法
   - 首选：EV/EBITDA（周期平均）
   - 避免：P/E（波动剧烈）

### 场景4：Pre-IPO公司（无公开财报）

**数据获取**:
1. S-1文件（上市前6个月披露）
2. 媒体报道（验证谨慎使用）
3. 竞品类比

**分析调整**:
1. 数据分级：多数为📊 ESTIMATED或🎯 MODELED
2. 敏感性分析范围扩大（±30%）
3. 明确标注"数据有限，不确定性高"

## 附录：行业识别FAQ

**Q1: 特斯拉应该归为"制造/硬件"还是"科技公司"？**
A: 按收入占比，汽车占比>70%，归为"制造/硬件"。但需单独分析储能（硬件）、FSD（SaaS订阅）、Robotaxi（平台）等业务，最终用SOTP估值。

**Q2: 苹果的服务业务占比20%，如何处理？**
A: 主分类"制造/硬件"，但服务业务单独用"SaaS"模块分析（ARR/NRR），SOTP估值。

**Q3: 新兴行业（如AI芯片设计）如何归类？**
A: 如果是Fabless设计（如NVIDIA），归"制造/硬件"（产品型）；如果是AI云服务（如OpenAI），归"SaaS/订阅"。

**Q4: 如果公司跨多个行业且无主业务？**
A: 使用多个行业模块，每个业务独立分析，SOTP加总。在报告开头说明归类依据。
