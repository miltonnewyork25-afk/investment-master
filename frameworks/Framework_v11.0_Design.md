# 投资研究框架 v11.0 - 全面升级方案

**版本**: v11.0
**发布日期**: 2026-01-25
**升级理由**: 基于Tesla项目深度复盘，修复数据质量1.2/100的致命缺陷，建立工业级研究框架
**核心改进**: 数据分级强制执行 + 质量门禁 + 行业适配 + 可复用系统

---

## 执行摘要

### 问题诊断（Tesla项目教训）

**致命问题**：
- 数据质量评分：1.2/100（Competition Analysis）、35/100（整体报告）
- 60%内容为虚构的"Engine 1-6系统"数据
- 关键财务错误：Tesla毛利率标注5.8%（实际18.7%），偏差12.9个百分点
- 无来源验证：97.7%数据无可验证来源
- 法律风险：误导性"实时监控"表述，可能违反证券法

**根本原因**：
1. 缺乏数据分级标注体系
2. 无强制来源验证机制
3. 质量检查清单未覆盖数据真实性
4. 估算方法不透明
5. 框架与示例数据混淆

### v11.0升级策略

**三大支柱**：
1. **数据分级强制执行体系**（Tier 1-4，禁止Unverified）
2. **25项+10项质量门禁**（35项清单，40%阈值）
3. **行业适配引擎**（8大行业×专属模块）

**质量承诺**：
- 可验证数据 ≥40%（vs Tesla项目2.3%）
- 估算数据必须附计算方法
- 禁止虚构任何具体数字
- 每个关键判断附至少1个Tier 1/2数据源

## 第一部分：数据分级强制执行体系

### 1.1 四级数据分类标准

#### Tier 1: VERIFIED（可直接使用作为核心论据）

**定义**: 来自官方、监管机构或交易所的一手数据

**来源清单**:
```
公司类:
  • SEC文件（10-K、10-Q、8-K、Form 4、13F）
  • 交易所公告（港交所、上交所、深交所）
  • 公司IR官网（财报、投资者演示文稿）
  • 财报电话会议官方文字稿（公司网站）

市场类:
  • 交易所官方数据（NYSE、NASDAQ、CBOE）
  • 彭博终端、FactSet原始数据
  • 政府统计（BEA、Census Bureau、中国统计局）
  • 行业协会官方统计（SEMI、WSTS、中汽协）

监管类:
  • FDA批文、临床试验数据（ClinicalTrials.gov）
  • 专利数据（USPTO、WIPO、CNIPA）
  • 环保数据（EPA）
```

**标注格式**:
```markdown
✅ **VERIFIED** [数据描述]
   来源: [具体文件名/网页]，p.XX（如适用）
   访问: [URL或文件编号]
   日期: YYYY-MM-DD

示例：
✅ **VERIFIED** Tesla Q3 2025汽车毛利率18.7%
   来源: Form 10-Q for Quarter Ended Sept 30, 2025, p.23
   访问: https://www.sec.gov/edgar/[具体链接]
   日期: 2025-10-23
```

**使用规则**:
- 核心命题的关键数据≥80%必须Tier 1
- 估值模型的输入变量≥60%必须Tier 1
- Tier 1数据可直接引用，无需额外验证

---

#### Tier 2: ESTIMATED（需说明估算方法）

**定义**: 基于Tier 1数据，通过公开、可复现的方法估算

**适用场景**:
```
财务建模:
  • DCF估值（基于财报数据+市场WACC）
  • SOTP拆分估值（各业务独立估值加总）
  • 单位经济学拆解（Revenue = Price × Volume × Mix）

趋势外推:
  • 基于历史3年CAGR的未来预测
  • 季节性调整后的环比增速
  • 市场份额演变（基于公开销量数据）

同业对标:
  • 可比公司估值倍数（P/E、EV/EBITDA）
  • 行业平均指标（毛利率、NRR）
```

**标注格式**:
```markdown
📊 **ESTIMATED** [数据描述]
   方法: [计算公式或逻辑]
   输入: [所用Tier 1数据]
   假设: [关键假设，如适用]
   置信度: [High/Medium/Low]

示例：
📊 **ESTIMATED** 储能业务合理估值$550-650亿
   方法: DCF，预测期2026-2030，永续增长率3%
   输入: Q3 2025实际收入$66亿（Form 10-Q），毛利率25%
   假设: 年增速120%→80%→50%递减，WACC 10%
   置信度: Medium（增速假设基于管理层指引±20%）
```

**使用规则**:
- 必须完整披露计算方法和假设
- 假设必须合理且可在6-12个月内验证
- 估算结果给出范围而非单点值
- 敏感性分析：至少测试±20%的关键假设变化

---

#### Tier 3: MODELED（情景分析/概率加权）

**定义**: 基于未来情景假设的推演，用于探索可能性而非预测

**适用场景**:
```
情景规划:
  • 牛/熊/基准三场景估值
  • 技术路线A/B/C成功的不同结果
  • 监管批准/拒绝的影响分析

概率加权:
  • Robotaxi商业化概率30% × 成功后估值$XXX
  • 新药三期临床成功概率60% × 上市后NPV
  • 并购交易完成概率75% × 协同价值

敏感性测试:
  • 如果毛利率+/-500bps → 估值影响
  • 如果汇率变动±10% → 收入影响
```

**标注格式**:
```markdown
🎯 **MODELED** [情景描述]
   场景: [牛/基准/熊 或 A/B/C]
   概率: X%（基于[依据]）
   关键假设: [列出2-3个核心假设]
   结果: [该情景下的数值]
   敏感性: [关键变量±X% → 结果变化Y%]

示例：
🎯 **MODELED** Robotaxi业务价值（牛市场景）
   场景: 2027年成功商业化
   概率: 30%（基于历史Tesla产品承诺兑现率35%±5%）
   关键假设:
     • 监管批准（加州、德州先行）
     • 干预率<1次/万英里
     • 2030年车队规模100万辆
   结果: NPV = $350-450亿（WACC 12%）
   敏感性: 商业化延迟1年 → NPV -25%
```

**使用规则**:
- 概率假设必须有依据（历史数据、专家共识、市场隐含）
- 多情景必须覆盖合理范围（不能只建模最优场景）
- 清晰标注"这是探索性分析，非预测"

---

#### Tier 4: FRAMEWORK（方法论演示，需替换真实数据）

**定义**: 分析框架、模板、示例数据，仅供说明方法论

**适用场景**:
```
教学/演示:
  • 数据收集框架说明
  • 分析工具使用示例
  • 报告模板结构

方法论展示:
  • OCI情绪指数计算公式（配示例数据）
  • 供应链影响推断逻辑（示例案例）
  • 技术路线对比矩阵（空白模板）
```

**标注格式**:
```markdown
⚙️ **FRAMEWORK** [框架名称]
   用途: 方法论演示，非真实数据
   使用: 实际应用时需接入[数据源]
   示例: [演示数据，明确标注"示例"字样]

示例：
⚙️ **FRAMEWORK** OCI情绪指数计算框架
   用途: 演示社交媒体情绪量化方法
   使用: 实际需通过Reddit API抓取真实帖子
   示例计算:
     假设抓取100条帖子（示例数据）
     正面60条、负面25条、中性15条
     加权后OCI = +52/100
   说明: 实际OCI需每日更新真实数据
```

**使用规则**:
- 必须醒目标注"仅为方法论演示"
- 不得在投资建议中引用Framework数据
- 如需实际使用，必须替换为Tier 1-3数据

---

#### ❌ UNVERIFIED（禁止使用）

**定义**: 无法归入上述任何类别的数据

**禁用清单**:
```
❌ 匿名消息源（"据知情人士透露"）
❌ 未经验证的社交媒体传言
❌ 无法追溯来源的具体数字
❌ AI生成的虚构数据
❌ "内部渠道"未公开信息
❌ 编造的"Engine X实时监控"数据
```

**违规处理**:
- 自动质量检查失败
- 报告禁止发布
- 需删除或替换为合规数据

### 1.2 数据分级强制执行流程

#### 写作阶段

```
步骤1：每写一个数据点，立即标注分级
  ├─ 是否有官方来源？→ Tier 1 ✅
  ├─ 是否基于模型估算？→ Tier 2 📊
  ├─ 是否情景假设？→ Tier 3 🎯
  ├─ 是否仅演示框架？→ Tier 4 ⚙️
  └─ 都不是？→ ❌ 删除或找来源

步骤2：附来源信息
  ├─ Tier 1：URL + 页码
  ├─ Tier 2：计算公式 + 输入数据
  ├─ Tier 3：概率依据 + 假设清单
  └─ Tier 4："方法论演示"声明

步骤3：核心判断验证
  每个核心命题的关键论据：
  ├─ 至少1个Tier 1数据
  └─ Tier 2/3数据比例<50%
```

#### 审核阶段

```python
# 自动化数据质量检查脚本
def audit_data_quality(report_markdown):
    """
    扫描报告，统计数据分级分布
    """
    # 1. 提取所有数据点
    verified = count_pattern(report_markdown, r'✅ \*\*VERIFIED\*\*')
    estimated = count_pattern(report_markdown, r'📊 \*\*ESTIMATED\*\*')
    modeled = count_pattern(report_markdown, r'🎯 \*\*MODELED\*\*')
    framework = count_pattern(report_markdown, r'⚙️ \*\*FRAMEWORK\*\*')

    # 2. 检测无标注数据（潜在UNVERIFIED）
    all_numbers = extract_numbers(report_markdown)
    tagged_numbers = verified + estimated + modeled + framework
    untagged = len(all_numbers) - tagged_numbers

    # 3. 计算质量评分
    total = verified + estimated + modeled + framework + untagged

    quality_score = (
        verified * 1.0 +      # 满分
        estimated * 0.7 +     # 70%权重
        modeled * 0.4 +       # 40%权重
        framework * 0.0 +     # 不计分
        untagged * (-0.5)     # 扣分
    ) / total * 100

    # 4. 生成报告
    return {
        'quality_score': quality_score,
        'breakdown': {
            'verified': f"{verified} ({verified/total*100:.1f}%)",
            'estimated': f"{estimated} ({estimated/total*100:.1f}%)",
            'modeled': f"{modeled} ({modeled/total*100:.1f}%)",
            'framework': f"{framework} ({framework/total*100:.1f}%)",
            'unverified': f"{untagged} ({untagged/total*100:.1f}%)"
        },
        'pass': quality_score >= 40,  # 质量门禁：40分及格
        'action': 'APPROVED' if quality_score >= 60 else 'NEEDS_REVISION'
    }
```

#### 发布前检查清单

```
□ 质量评分 ≥ 40/100
□ Tier 1数据 ≥ 20%
□ Unverified数据 ≤ 20%
□ 所有Tier 2数据附方法说明
□ 所有Tier 3数据附概率依据
□ 报告开头有数据声明（见1.3）
□ 核心命题关键数据≥50% Tier 1/2
```

### 2.1 行业识别决策树

```
输入：公司名称/股票代码
  ↓
步骤1：获取GICS代码或主营业务描述
  ├─ 数据源：彭博终端、公司10-K "Business"章节
  └─ 备选：手动分类
  ↓
步骤2：匹配至8大行业之一
  ├─ 规则：优先GICS前2位代码
  ├─ 边界情况：多业务公司按收入占比>50%分类
  └─ 输出：行业代码 + 特定模块清单
  ↓

  ├─ 通用模块（所有公司）
```

### 2.2 行业1：制造/硬件（汽车、消费电子、工业设备）

**GICS代码**: 25 (Consumer Discretionary, Auto), 45 (IT Hardware)

**核心特征**:
- 实体产品，可量化单位经济学
- 供应链复杂，BOM成本占比高
- 规模效应显著，产能利用率关键

**必做模块**:

#### A. PVM财务拆解（Price × Volume × Mix）
```
Revenue = Σ(产品i价格 × 产品i销量 × 产品i占比)

拆解维度：
1. Price（定价）
   - 平均售价ASP趋势（环比/同比）
   - 降价幅度 vs 竞品
   - 定价权指标（提价后销量变化）

2. Volume（销量）
   - 总销量及增速
   - 地区分布（中美欧拆分）
   - 渠道分布（直销/经销商）

3. Mix（产品结构）
   - 高端/中端/入门级占比
   - Mix变化对毛利率影响
   - 新品占比

输出表格：
| 季度 | ASP | 销量 | Mix（高/中/低） | 收入 | YoY |
```

#### B. 单位经济学（Unit Economics）
```
每辆车/每台设备的：
- 收入（ASP）
- 直接成本（BOM + 人工 + 制造费用）
- 毛利 = 收入 - 直接成本
- 毛利率趋势

成本结构拆解（BOM）：
- 原材料（电池/芯片/钢铁等）
- 零部件（供应商采购）
- 人工成本（单位人工时×工资）
- 制造费用（折旧、能源）

学习曲线：
- 累计产量每翻倍 → 成本下降X%
- Wright's Law验证
```

#### C. 产能与供应链分析
```
产能利用率：
- 当前产能（辆/年）
- 实际产量（利用率%）
- 在建产能及投产时间
- Capex趋势（是扩产还是维护？）

供应链风险：
- 关键原材料来源（地缘风险）
- 核心供应商集中度（CR5收入占比）
- 库存周转天数（DOI趋势）
- 应付账款天数（DPO，付款压力）

供应链受益分析：
- 识别Tier 1/2/3供应商
- 上市供应商投资机会（见第8章）
```

#### D. 竞争对标（同业对比）
```
对标公司选择：
- 同行业前5名（按市值或销量）
- 相同价格带产品

对比维度：
| 公司 | ASP | 销量 | 毛利率 | R&D占比 | 市值 | P/S | P/E |
|------|-----|------|--------|---------|------|-----|-----|

技术差距量化：
- 核心指标对比（续航/性能/能效）
- 成本结构对比（单位成本）
- 专利数量及质量
```

**字数要求**: 5,000-7,000字
**深度标准**: Level 3（机制层）- 解释PVM变化的驱动因素，而非仅列数字

### 2.3 行业2：SaaS/订阅（软件、云服务、内容订阅）

**GICS代码**: 45 (Software), 50 (Media)

**核心特征**:
- 经常性收入（ARR/MRR），可预测性强
- 用户留存是核心（NRR > 100%）
- 前期投入大，边际成本低

**必做模块**:

#### A. ARR/MRR拆解
```
ARR (Annual Recurring Revenue) = MRR × 12

拆解公式：
ARR_t = ARR_t-1 + New ARR + Expansion - Churn - Contraction

组成部分：
1. New ARR（新客户）
   - 新增客户数
   - 平均合同价值ACV
   - CAC（获客成本）

2. Expansion（老客户升级）
   - Upsell（升级套餐）
   - Cross-sell（购买其他产品）
   - 扩展收入占比

3. Churn（流失）
   - Gross Churn（客户流失）
   - Net Churn（扩展-流失）
   - Logo Churn vs Revenue Churn

4. Contraction（降级）
   - 降级客户占比
   - 降级金额
```

#### B. 队列分析（Cohort Analysis）
```
按获客时间分组，追踪留存：

| 获客季度 | Q1 | Q2 | Q3 | Q4 | Q5 | Q6 | Q7 | Q8 |
|---------|----|----|----|----|----|----|----|----|
| 2023 Q1 | 100%| 95%| 92%| 90%| 88%| 87%| 86%| 85%|
| 2023 Q2 | 100%| 96%| 93%| 91%| 89%| 88%| 87%| - |
| 2023 Q3 | 100%| 97%| 94%| 92%| 90%| 89%| - | - |

观察：
- 留存曲线是否改善（新队列>老队列）
- 第X个季度稳定留存率
- LTV = ARPU × (1 / Churn Rate)
```

#### C. Rule of 40（增长与效率平衡）
```
Rule of 40: ARR增速% + FCF Margin% ≥ 40%

示例：
- ARR增速 30% + FCF Margin 15% = 45% ✓
- ARR增速 50% + FCF Margin -5% = 45% ✓
- ARR增速 20% + FCF Margin 10% = 30% ✗

阶段判断：
- 增长型（高增速、低利润）
- 平衡型（中增速、中利润）
- 效率型（低增速、高利润）

SaaS Magic Number:
= (当期ARR - 上期ARR) / 上期S&M支出
> 0.75 为优秀（每$1销售投入带来>$0.75 ARR增长）
```

#### D. 单位经济学
```
CAC (Customer Acquisition Cost):
= S&M支出 / 新增客户数

LTV (Lifetime Value):
= ARPU × Gross Margin% × (1 / Churn Rate)

LTV/CAC比值：
- <1: 不可持续
- 1-3: 需改进
- >3: 健康
- >5: 优秀

Payback Period（回本周期）：
= CAC / (ARPU × Gross Margin%)
目标 <12个月
```

**字数要求**: 4,500-6,000字
**深度标准**: Level 3 - 解释NRR变化的驱动因素（产品粘性/竞争/定价）

### 2.4 行业3：REIT（房地产投资信托）

**GICS代码**: 60 (Real Estate)

**核心特征**:
- 收入来源：租金 + 物业增值
- 高分红要求（90%收入分红）
- 对利率高度敏感

**必做模块**:

#### A. NAV模型（Net Asset Value）
```
NAV = 物业公允价值 - 净负债

物业估值方法：
1. Cap Rate法
   物业价值 = NOI / Cap Rate

   NOI (Net Operating Income) = 租金收入 - 运营费用
   Cap Rate = 市场资本化率（同类物业交易推断）

2. DCF法
   物业价值 = Σ(未来租金现金流) / (1+WACC)^t

NAV vs 市值：
- 溢价（P/NAV > 1）：市场看好增长
- 折价（P/NAV < 1）：市场担忧或低估
```

#### B. FFO/AFFO分析
```
FFO (Funds From Operations):
= Net Income + 折旧摊销 - 物业出售收益

AFFO (Adjusted FFO):
= FFO - 维护性Capex - 租赁佣金

分红覆盖率：
= AFFO / 分红总额
> 1.2 为安全

FFO增速：
- 同店NOI增长（Same-Store NOI Growth）
- 新增物业贡献
- 债务成本变化
```

#### C. 物业组合分析
```
按类型分类：
- 办公楼
- 零售（商场/街铺）
- 工业（物流/数据中心）
- 住宅（公寓）
- 特殊（医疗/学生公寓）

关键指标：
| 类型 | GLA (可出租面积) | 出租率 | 平均租金 | 租约到期分布 |
|------|-----------------|--------|---------|-------------|

租约结构：
- 平均租期
- 租金增长条款（CPI挂钩/固定%）
- 租约到期集中度风险
```

#### D. 利率敏感性分析
```
利率传导路径：
Fed加息 → 债务成本↑ + REIT估值↓（折现率↑）

久期（Duration）：
= 利率每变化1% → NAV变化X%

对冲措施：
- 固定利率债务占比
- 利率互换（Interest Rate Swap）
- 债务到期期限结构
```

**字数要求**: 4,000-5,500字
**深度标准**: Level 3 - 解释Cap Rate变化与宏观环境的联系

### 2.5 行业4：平台/市场（电商、共享经济、交易平台）

**GICS代码**: 25 (Consumer Discretionary, Internet), 50 (Communication)

**核心特征**:
- 双边市场（买家+卖家）
- 网络效应强
- GMV与Take Rate决定收入

**必做模块**:

#### A. GMV×Take Rate拆解
```
Revenue = GMV × Take Rate

GMV (Gross Merchandise Value) 拆解：
= 活跃买家数 × 人均订单数 × 平均订单金额

Take Rate 拆解：
= (平台佣金 + 广告收入 + 增值服务) / GMV

示例：
- 电商：Take Rate 3-5%
- 外卖：Take Rate 15-20%
- 网约车：Take Rate 20-25%

趋势分析：
- GMV增速 vs Take Rate变化
- Take Rate提升空间（对标竞品）
```

#### B. 双边单位经济学
```
买家侧：
- CAC（获客成本）
- 留存率
- 复购频次
- LTV

卖家侧：
- 入驻成本
- 卖家留存率
- 人均GMV贡献
- 平台依赖度（独家%）

双边平衡：
- 买卖双方增速匹配
- 流动性指标（买家等待时间/卖家闲置率）
```

#### C. 网络效应量化
```
梅特卡夫定律（Metcalfe's Law）:
网络价值 ∝ 用户数²

验证方法：
1. 计算用户增长 vs GMV增长关系
   如果GMV增速 > 用户增速 → 网络效应显现

2. 边际获客成本（Marginal CAC）
   如果Marginal CAC下降 → 网络效应发挥作用

3. 交叉网络效应
   买家增加 → 卖家增加 → 买家体验改善 → 更多买家
```

#### D. 赢家通吃分析
```
市场集中度：
- CR3（前3名份额）
- HHI指数（Herfindahl-Hirschman Index）

护城河来源：
- 网络效应强度
- 转换成本（数据/社交关系）
- 规模经济（单位成本优势）

竞争态势：
- 是否已进入寡头稳定期
- 新进入者威胁（资本/监管门槛）
```

**字数要求**: 4,500-6,000字
**深度标准**: Level 3 - 解释网络效应的具体传导机制

### 2.6 行业5：E&P/资源（石油天然气、矿业）

**GICS代码**: 10 (Energy), 15 (Materials)

**核心特征**:
- 商品价格敏感度极高
- 储量是核心资产
- 资本密集，周期性强

**必做模块**:

#### A. 储量评估（Reserves Analysis）
```
储量分类（SEC定义）：
- 1P（Proven）：90%概率可采
- 2P（Proven + Probable）：50%概率
- 3P（Proven + Probable + Possible）：10%概率

储量价值：
NAV = Σ(每桶/吨储量 × 实现价格 - 开采成本) / (1+折现率)^开采年份

储量替换率（RRR）:
= 年度新增储量 / 年度产量
> 100% 为健康（储量不减少）
```

#### B. 成本曲线分析
```
全周期成本（All-in Cost）:
= 勘探成本 + 开发成本 + 生产成本 + 运输成本

成本曲线位置：
- 低成本生产商（<$30/桶）：全周期盈利
- 中成本（$30-50/桶）：油价敏感
- 高成本（>$50/桶）：页岩油，衰减快

边际成本 vs 油价：
- 当前油价高于边际成本 → 增产动力
- 低于边际成本 → 减产
```

#### C. 生产指标与衰减率
```
产量拆解：
= 现有油田产量 × (1 - 自然衰减率) + 新油田产量

自然衰减率（Decline Rate）:
- 常规油田：5-10%/年
- 页岩油：30-50%/年（首年）

Capex效率：
- 每投入$1 Capex → 新增多少产量
- IRR（内部收益率）vs WACC
```

#### D. 油价敏感性分析
```
盈亏平衡价格（Breakeven Price）:
= 全周期成本

敏感性测试：
| 油价 | 产量 | EBITDA | FCF | 分红覆盖率 |
|------|------|--------|-----|----------|
| $60  | 100万桶/日 | $X | $Y | Z% |
| $70  | 105万桶/日 | ... | ... | ... |
| $80  | 110万桶/日 | ... | ... | ... |

对冲策略：
- 期货对冲比例
- 固定价格合同占比
```

**字数要求**: 4,000-5,000字
**深度标准**: Level 3 - 解释储量质量差异对估值的影响

### 2.7 行业6：金融/银行（商业银行、资产管理）

**GICS代码**: 40 (Financials)

**核心特征**:
- 杠杆经营（存贷差）
- 信用风险是核心
- 强监管（资本充足率）

**必做模块**:

#### A. NIM拆解（Net Interest Margin）
```
NIM = (利息收入 - 利息支出) / 生息资产

拆解：
NIM = 资产收益率 - 负债成本率

驱动因素：
1. 资产端
   - 贷款组合（零售/企业/房贷）
   - 贷款利率定价
   - 证券投资收益率

2. 负债端
   - 存款成本（活期/定期比例）
   - 同业拆借成本
   - 债券发行成本

利率敏感性：
- Asset Sensitivity: 加息→NIM扩张
- Liability Sensitivity: 加息→NIM收窄
```

#### B. 资产质量分析
```
不良贷款率（NPL Ratio）:
= 不良贷款 / 总贷款

拨备覆盖率（Provision Coverage）:
= 贷款损失准备 / 不良贷款
> 150% 为安全

信贷成本（Credit Cost）:
= 新增拨备 / 平均贷款余额
趋势：上升→资产质量恶化

贷款迁徙率（Migration Rate）:
= 正常类→关注类→次级类→可疑类→损失类
早期预警指标
```

#### C. 资本充足率
```
CET1 Ratio（核心一级资本充足率）:
= CET1 / 风险加权资产（RWA）

监管要求：
- 巴塞尔III：≥7%
- 系统重要性银行：+1-2.5%

资本管理：
- ROE vs CET1 Ratio权衡
- 分红派息率（股息/净利润）
- 股票回购计划
```

#### D. 杜邦分解（ROE驱动）
```
ROE = (净利润 / 收入) × (收入 / 资产) × (资产 / 股东权益)
    = 净利率 × 资产周转率 × 权益乘数

银行特定：
ROE = ROA × 权益乘数
ROA = NIM × (1 - 成本收入比 - 信贷成本)

优化路径：
- 提升NIM（利率定价）
- 降低成本收入比（数字化）
- 控制信贷成本（风控）
- 提高杠杆（在监管允许范围）
```

**字数要求**: 4,000-5,500字
**深度标准**: Level 3 - 解释NIM变化与宏观利率环境的传导机制

### 2.8 行业7：生物医药（制药、生物技术）

**GICS代码**: 35 (Healthcare)

**核心特征**:
- 研发周期长（10-15年）
- 临床试验成功率低
- 专利到期悬崖

**必做模块**:

#### A. 管线价值（Pipeline Valuation）
```
rNPV (risk-adjusted NPV) 模型:
= Σ(产品i的NPV × 临床成功概率)

单个产品NPV：
= Σ(年度销售额 × 净利率 - 研发成本) / (1+WACC)^t

临床成功概率（行业平均）：
- I期 → II期：63%
- II期 → III期：31%
- III期 → 上市：58%
- I期 → 上市：11%

示例：
某药物处于II期，峰值销售预期$500M
rNPV = NPV($2B) × 31% × 58% = $360M
```

#### B. 管线组合矩阵
```
BCG矩阵（药物版）：

明星（Stars）：
- III期临床/刚上市
- 大适应症（肿瘤/糖尿病）
- 峰值销售>$1B（Blockbuster）

现金牛（Cash Cows）：
- 已上市，专利期内
- 稳定销售
- 高毛利率

问题（Question Marks）：
- I/II期临床
- 创新机制，高风险高回报
- 需大量研发投入

瘦狗（Dogs）：
- 临床失败
- 或专利到期，仿制药竞争
```

#### C. 专利悬崖分析
```
专利到期时间表：
| 药物 | 销售额 | 专利到期 | 仿制药影响 | 对冲措施 |
|------|--------|---------|-----------|---------|
| 药A  | $2B    | 2027    | -80%      | 新适应症 |
| 药B  | $1.5B  | 2029    | -70%      | 新剂型 |

仿制药侵蚀模型：
- 专利到期第1年：销售额-50%
- 第2年：再-20%
- 第3年后：稳定在20-30%原销售额

对冲策略：
- 新适应症开发（延长生命周期）
- 改进剂型（缓释/长效）
- 授权合作（License Out）
```

#### D. 现金跑道（Cash Runway）
```
Burn Rate（烧钱率）:
= 季度运营亏损 / 3

现金跑道：
= 当前现金及等价物 / Burn Rate
单位：季度数

融资需求评估：
- 跑道<4季度 → 高融资风险
- 跑道4-8季度 → 需关注
- 跑道>8季度 → 安全

稀释风险：
- 预计融资金额 / 当前市值
- 稀释%对每股价值影响
```

**字数要求**: 5,000-6,500字
**深度标准**: Level 3 - 解释临床数据对估值的具体影响路径

### 2.9 行业8：消费品ToC（品牌、零售、餐饮）

**GICS代码**: 25 (Consumer Discretionary/Staples)

**核心特征**:
- 品牌是核心资产
- 用户忠诚度决定LTV
- 渠道变革影响大

**必做模块**:

#### A. 品牌资产量化
```
品牌价值评估：

1. 品牌溢价法
   = (品牌产品价格 - 同类无牌产品价格) × 销量

2. Royalty Relief法
   = 假设授权品牌给他人，应收取的特许权使用费
   = 品牌收入 × 行业平均特许费率%

3. 消费者心智份额
   - Top-of-Mind Awareness（第一提及率）
   - NPS（Net Promoter Score）
   - 品牌关联度（Social Listening）

品牌健康度趋势：
- Google Trends搜索量
- 社交媒体提及量及情绪
- 门店客流量变化
```

#### B. 用户LTV与留存
```
LTV = ARPU × 购买频次 × 客户生命周期（年）

队列分析（同2.3 SaaS）：
| 获客年份 | Y1 | Y2 | Y3 | Y4 | Y5 |
|---------|----|----|----|----|-----|
| 2021    | 100%| 60%| 40%| 30%| 25%|
| 2022    | 100%| 65%| 45%| 35%| - |
| 2023    | 100%| 70%| 50%| - | - |

复购率（Repeat Purchase Rate）:
= 第N次购买客户数 / 总客户数

CAC vs LTV:
- LTV/CAC > 3 为健康
- Payback Period < 12个月
```

#### C. 渠道分析
```
全渠道销售拆解：
- 线上（电商自营/平台）
- 线下（直营/加盟/经销商）
- 新兴（社交电商/直播）

渠道经济学：
| 渠道 | 收入占比 | 毛利率 | 获客成本 | 增速 |
|------|---------|--------|---------|------|
| 天猫 | 30%     | 45%    | 高      | 15%  |
| 抖音 | 10%     | 50%    | 中      | 80%  |
| 门店 | 60%     | 55%    | 低      | 5%   |

渠道迁移风险：
- 线下→线上：租金节省 vs 流量成本
- 中心化电商→去中心化（社交）：流量逻辑变化
```

#### D. 单店模型（门店业务）
```
单店经济学：
收入 = 日均客流 × 客单价 × 营业天数

成本结构：
- 租金（固定）
- 人工（半固定）
- 原材料（可变）
- 水电物业（半固定）

单店盈利能力：
- EBITDA Margin
- 投资回收期（Payback）
- IRR

同店增速（Same-Store Sales Growth）:
= 开业≥12个月门店的销售增速
关键指标：剔除新店扩张影响，反映品牌真实健康度
```

**字数要求**: 4,500-6,000字
**深度标准**: Level 3 - 解释品牌力变化的驱动因素（产品/营销/渠道）

## 第四部分：飞轮效应与可复用系统

### 4.1 飞轮设计理念

```python
# 自动生成对标表格
def generate_peer_comparison(target_company, industry):
    peers = db.query(f"SELECT * FROM peer_companies WHERE industry='{industry}' ORDER BY market_cap DESC LIMIT 5")

    table = f"| Company | Market Cap | P/E | P/S | Gross Margin |\n"
    table += f"|---------|-----------|-----|-----|-------------|\n"
    for peer in peers:
        table += f"| {peer.company_name} | ${peer.market_cap/1000:.0f}B | {peer.pe_ratio:.1f}x | {peer.ps_ratio:.1f}x | {peer.gross_margin*100:.1f}% |\n"

    return table

# 一行代码生成表格
print(generate_peer_comparison('Google', 'SaaS'))
```
