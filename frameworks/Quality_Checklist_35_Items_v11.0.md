## 使用说明

### 检查流程

```
报告完成后：
  ↓
步骤1：自动化检查（10分钟）
  ├─ 运行 data_quality_monitor_v2.py
  ├─ 生成质量评分和详细报告
  └─ 自动标注通过/失败项
  ↓
步骤2：人工复核（30分钟）
  ├─ 逐项核对本清单
  ├─ 重点检查自动化无法覆盖的项
  └─ 记录未通过项及改进计划
  ↓
步骤3：质量门禁判断
  ├─ 通过：≥30/35项 + 所有P0项
  ├─ 待修订：25-29项通过
  └─ 禁止发布：<25项通过
  ↓
步骤4：发布/返回修订
```

### 优先级说明

- **P0**（Must-Have）：必须100%通过，否则禁止发布
- **P1**（Should-Have）：建议通过，影响质量评分
- **P2**（Nice-to-Have）：加分项，优秀报告标准

### 1. ✓ 报告总字数 ≥ 20,000字

**检查方法**:
```bash

```

**通过标准**:
- 中文报告：≥20,000字
- 英文报告：≥15,000词（中英字数比约1.3:1）

**常见问题**:
- ❌ 只计算正文，未排除代码块/表格
- ❌ 用图表撑字数，文字说明不足

**修正建议**:
- 扩展分析深度，而非堆砌无效内容
- 每个核心命题≥500字详细论述

**自动化**:
```python
def check_word_count(report_path):
    with open(report_path, 'r', encoding='utf-8') as f:
        content = f.read()
        # 移除代码块
        content = re.sub(r'```.*?```', '', content, flags=re.DOTALL)
        # 中文字符数
        chinese_chars = len(re.findall(r'[\u4e00-\u9fff]', content))
        return chinese_chars >= 20000
```

```python
def check_data_date(report):
    # 检查是否有明确日期标注
    date_pattern = r'\*\*数据截止日期\*\*.*(\d{4}-\d{2}-\d{2})'
    match = re.search(date_pattern, report)
    if not match:
        return False
    # 检查日期是否过期（>30天视为警告）
    data_date = datetime.strptime(match.group(1), '%Y-%m-%d')
    if (datetime.now() - data_date).days > 30:
        print("⚠️ 警告：数据已过期30天")
    return True
```

```python
def check_core_contradiction(report):
```

### 11. ✓ 层级穿透: ≥50%命题达到Level 3+（P0）

**层级定义**:
```
Level 1 - 表象层：市场价格/新闻叙事 → ❌ 不合格
Level 2 - 数据层：财务数字/指标趋势 → ⚠️ 需深化
Level 3 - 机制层：驱动因素/结构性原因 → ✓ 合格
Level 4 - 本质层：根本原因/不可替代性 → ✓✓ 优秀
```

**检查方法**（人工评估）:
```
对每个核心命题（假设5个）：
  Q1: Level [1/2/3/4]
  Q2: Level [1/2/3/4]
  Q3: Level [1/2/3/4]
  Q4: Level [1/2/3/4]
  Q5: Level [1/2/3/4]

计算：Level 3-4占比 ≥ 50%（即≥3个）
```

**示例对比**:
```
❌ Level 2（数据层）：
  "Tesla Q3毛利率18.7%，环比下降2pp"
  → 仅描述数据，未解释原因

✓ Level 3（机制层）：
  "Tesla Q3毛利率18.7%（-2pp环比），主因：
   1. Cybertruck产能爬坡，单位成本仍高于成熟产品
   2. 价格战环境下，Model 3/Y降价5-8%
   3. 学习曲线效应被稀释（新产品占比↑）
   → 短期压力，但随Cybertruck规模化将改善"
  → 解释了机制和未来趋势
```

### 14. ✓ 反证句: 关键判断有"但如果___则不成立"（P0）

**要求**:
- 每个核心判断后附反证句
- 强制暴露判断的脆弱点
- 提醒读者（和自己）风险

**示例**:
```
✓ 正确写法：
  "我们认为Tesla储能业务2026年收入将达$24B（+120% YoY）。
   **反证句**: 但如果电池供应链（CATL/LG）产能扩张不及预期，
   或电网级储能项目审批延迟，则增速可能降至80-100%。"

❌ 错误写法：
  "我们认为Tesla储能业务2026年收入将达$24B（+120% YoY）。"
  → 没有反证句，过度自信
```

**检查方法**:
```python
def check_反证句(report):
    # 提取所有关键判断（通常包含"我们认为"、"判断"等）
    judgments = re.findall(r'我们认为.*?[。\n]', report)

    # 检查每个判断是否附近有反证句
    for judgment in judgments:
        # 在判断后500字符内搜索"但如果"、"除非"等
        context = report[report.index(judgment):report.index(judgment)+500]
        if not re.search(r'但如果|除非|然而若', context):
            print(f"⚠️ 缺少反证句：{judgment[:50]}...")
            return False
    return True
```

### 15. ✓ 反事实检验: 已执行（P0）

**定义**:
- 对关键判断，问"如果去掉因素X，结论是否改变？"
- 如果去掉后结论不变 → X不是真正驱动因素

**格式**:
```markdown
## 反事实检验

### 检验1: 如果Robotaxi完全失败
**问题**: 仅凭汽车+储能，Tesla值多少？
**计算**:
  • 汽车业务（P/E 30x）：$150B
  • 储能业务（P/S 5x）：$120B
  • 总估值：$270B
**对比**: 当前市值$1.43T
**结论**: 市场隐含Robotaxi价值 = $1.43T - $270B = $1.16T
  → 说明Robotaxi是当前估值的核心驱动（占81%）

### 检验2: 如果Musk离开Tesla
**问题**: 管理层溢价有多少？
**类比**: Apple失去乔布斯后估值倍数从15x→12x（-20%）
**假设**: Tesla失去Musk，估值倍数从112x→70x（-37%）
**影响**: 市值下修$530B
**结论**: Musk个人品牌价值巨大，是重要风险
```

**检查清单**:
```
□ 至少2个反事实检验 ✓
□ 每个都有定量计算 ✓
□ 结论清晰（是/否核心驱动因素） ✓
```

### 16. ✓ 单位经济学: 到"每辆/每用户/每单位"级别（P0）

**行业单位清单**:

```python
def quality_gate_check(report_path):
    """
    质量门禁检查
    """
    report = read_file(report_path)

    results = {
        'word_count': check_word_count(report),  # ≥20,000
        'required_sections': check_sections(report),  # 全部存在
        'data_tagging_rate': check_tagging_rate(report),  # ≥80%
        'tier1_ratio': check_tier1_ratio(report),  # ≥20%
        'unverified_ratio': check_unverified_ratio(report),  # ≤20%
        'quality_score': calculate_quality_score(report),  # ≥40
        'data_declaration': check_data_declaration(report),  # 存在
        # ... 其他检查
    }

    # 计算通过项
    passed_items = sum(1 for v in results.values() if v)
    total_items = 35

    # 判断
    if passed_items >= 30 and results['quality_score'] >= 40:
        decision = 'APPROVED - 可发布'
    elif passed_items >= 25:
        decision = 'NEEDS_REVISION - 需修订'
    else:
        decision = 'REJECTED - 禁止发布'

    # 生成报告
    return {
        'decision': decision,
        'passed_items': f'{passed_items}/35',
        'quality_score': results['quality_score'],
        'details': results
    }
```
