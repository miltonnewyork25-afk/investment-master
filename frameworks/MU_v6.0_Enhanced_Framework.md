## 2.1 标准化数学指标体系 (SIS/RAR/CWR/CRR)

### 模块编号: U-SIS/RAR/CWR/CRR
### 位置: Phase 0 执行摘要
### 深度: L4
### 字数目标: 3,000-4,000字

#### SIS (Standard Investment Score) — MU专属计算框架

```
MU-SIS = Σ(维度得分 × 权重)

┌─────────────────────────────────────────────────────────┐
│ 维度           │ 满分 │ 权重 │ MU特殊考量              │
├─────────────────────────────────────────────────────────┤
│ 基本面得分     │  30  │ 30%  │ 周期性盈利需标准化       │
│ 成长性得分     │  25  │ 25%  │ HBM超级增长+DRAM周期恢复 │
│ 估值合理性     │  20  │ 20%  │ 周期股估值需用P/B+EV/EBITDA│
│ 风险控制       │  15  │ 15%  │ 周期风险权重上调          │
│ ESG可持续      │  10  │ 10%  │ 半导体环保+水资源         │
└─────────────────────────────────────────────────────────┘
```

**MU特殊调整**: 
- 基本面得分中，盈利能力使用**峰-谷标准化**（不直接用当期P/E，避免周期底部失真）
- 成长性得分中，结构性增长(HBM)与周期性增长(DRAM commodity)分开计算
- 估值合理性中，引入**周期调整PE (CAPE)**和**P/B-ROE回归**

#### RAR (Risk-Adjusted Return) — MU风险因子分解

```
RAR = 超额收益 / 综合风险系数

MU风险因子分解（vs TSM有差异）:
• 市场Beta风险: ×30% → MU Beta≈1.6 (高于TSM的1.25)
• 基本面风险: ×20% → 周期性盈利波动大
• 周期风险⭐: ×20% → Memory周期特有 (TSM无此项)
• 特殊事件风险: ×15% → 地缘+客户集中
• 流动性风险: ×5%  → 流动性良好
• 技术替代风险: ×10% → CXL/新兴存储威胁
```

#### CWR (Confidence-Weighted Return) — MU三场景加权

```
CWR = Σ(场景概率 × 预期收益 × 数据置信度)

MU三场景驱动因素:
Bull: HBM供不应求+DDR5全面起量+NAND复苏 → P(25%) × Return(xx%) × Conf(85%)
Base: HBM稳健+DDR5温和增长+NAND持平 → P(55%) × Return(xx%) × Conf(90%)
Bear: AI CapEx放缓+DRAM降价+库存积压 → P(20%) × Return(xx%) × Conf(80%)
```

#### CRR (Comprehensive Risk Rating) — MU风险矩阵

```
CRR = 10 - Σ(风险因子 × 权重)

MU风险因子(vs TSM差异):
• 基本面风险: ×30% → 盈利波动性 (MU > TSM)
• 市场风险: ×20% → Beta较高
• 周期风险⭐: ×20% → Memory周期独有
• 行业风险: ×15% → 技术壁垒+产能竞争
• 地缘风险: ×15% → 中国业务风险+出口管制
```

   ┌───────────────────────────────────────────┐
