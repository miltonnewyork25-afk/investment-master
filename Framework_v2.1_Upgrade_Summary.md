# 框架v2.1升级总结 — 基于ASML深度反思

## 🎯 核心发现

**用户反思最高价值**: **工程化结构设计**
- ASML报告优势在于把投资分析从"艺术"变成"工程"
- 最值得复制的是"CQ→证据→KS/TS→动作"的工程化结构

## 🏆 直接借鉴 (立即纳入标配)

### 1. CQ化结构 ⭐⭐⭐⭐⭐
- **核心**: 把"垄断"从一句话变成6-8个可争议核心问题
- **价值**: 横向对比半导体公司的最强统一骨架
- **标准**: 争议性+可回答性+决策相关性+特异性

### 2. KS分级SOP ⭐⭐⭐⭐⭐
- **核心**: A/B/C/D/X级响应机制 + CQ↔KS映射表
- **价值**: 从研究→监控→动作的完整闭环
- **标准**: 可观测+可执行+前瞻性+特异性

### 3. IB服务业务量化 ⭐⭐⭐⭐
- **核心**: 装机基座+Attach rate+单位经济学
- **价值**: 对设备类公司极其重要的标准模块
- **字段**: IB规模/服务占比/单台ARPU/毛利率等

## 🔄 整合升级 (与现有框架融合)

### 4. MCI指数 vs Chokepoint融合
- **升级**: Chokepoint 8维度 → MCI 0-100分制
- **新增**: 替代路径时间、监管敏感程度维度
- **公式**: MCI = 8维度平均分 × 10

### 5. 供给约束建模 (新增核心)
- **流程**: 订单→backlog→产能→交付→收入确认
- **价值**: 把"增长"变成可量化的约束链分析
- **适用**: 制造业和设备类公司必备

### 6. 地缘政治财务化
- **拆分**: 中国Systems vs Services收入
- **量化**: 销量损失+毛利变化+二级影响
- **工具**: 替代树3层分析

## 🚀 v2.1架构：三轴+双引擎+五模块

### 三轴 (跨公司可比)
```yaml
轴1: 价值链位置 (设计→制造→设备→材料→封装→终端)
轴2: 周期敏感度 (L1-L6雷达，保留v2.0优势)
轴3: 咽喉锁定度 (MCI指数+生态锁定+切换成本)
```

### 双引擎 (标准化建模)
```yaml
引擎A: MCI垄断指数 (0-100分量化垄断/咽喉程度)
引擎B: 供给约束模型 (订单→产能→交付全流程建模)
```

### 五模块 (专业深度)
```yaml
模块1: Chokepoint指数升级版 (8维度MCI)
模块2: 周期暴露矩阵 (L1-L6保留)
模块3: 结构性变量+地缘财务化
模块4: IB服务经济学 (设备类专用)
模块5: 供给约束链 (制造类专用)
```
