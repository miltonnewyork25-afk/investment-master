# 超级标题引擎 (Super Headline Engine)
# 版本: v1.1
# 创建: 2026-01-24
# 能力: 从任意信息生成超级传播力标题，面向顶级投资人群，自适应多平台
# 理论基础: Gene Schwartz + Dan Koe + Berger & Milkman + 行为经济学

---

## 核心定理

**"同一个承诺，表达方式不同，效果可以相差500%。" — Gene Schwartz**

标题不是总结内容——是**激活读者心理机制使其无法不点击**的文字装置。

```
标题效力 = 心理触发强度 × 认知流畅度 × 平台适配度 × 身份共鸣度
```

## Part 0: 三条铁律 (Inviolable Rules)

**这3条规则优先级高于一切。任何违反铁律的标题=0分，直接丢弃。**

### 铁律1: 去身份化 + 合规安全 (No Identity, No Advice)

```
❌ 绝对禁止: 暴露任何身份(AI或个人)
   - "Our AI framework..." → 禁止
   - "AI says SELL..." → 禁止
   - "I score it..." / "My framework..." → 禁止
   - "我看空..." / "我的判断..." → 禁止

❌ 绝对禁止: 主观方向性判断(荐股/诱导投资)
   - "BUY" / "SELL" / "看多" / "看空" → 禁止
   - "该买了" / "该跑了" / "赶紧卖" → 禁止
   - "I'm bearish" / "I'm bullish" → 禁止
   - 任何暗示读者应该买入/卖出的措辞 → 禁止

✅ 必须使用: 客观第三方视角
   - 数据陈述: "50.4x P/E, 历史均值20-30x"
   - 现象描述: "25位分析师看多，周期信号亮红灯"
   - 提问式: "...意味着什么？" / "...是否合理？"
   - 教育式: "一文读懂..." / "...的底层逻辑"
   - 框架展示: "周期框架评分23.5/100" (不加"我的")

✅ 安全表达方式:
   - "值得关注的信号" → 替代 "危险信号"
   - "历史上类似情况的结果" → 替代 "一定会跌"
   - "估值与历史均值的偏离" → 替代 "太贵了"
   - "共识与数据的分歧" → 替代 "分析师都错了"

原理:
- 各平台合规: 避免被判定为"荐股"/"投资建议"/"诱导交易"
- 封号风险规避: X/雪球/小红书对金融内容审核趋严
- 客观视角反而更可信: 数据说话>个人意见
- 提问式=更高互动率: 读者回复补充自己的判断
```

### 铁律2: 忠实于原文 (Content Fidelity)

```
❌ 绝对禁止: 标题超出内容实际覆盖的范围
   - 内容说"概率权重期望收益+1%" → 不能标题"必跌"
   - 内容说"周期风险" → 不能标题"即将崩盘"
   - 内容说"历史上..." → 不能标题"一定会..."

✅ 必须做到: 标题是内容核心洞察的精炼表达
   - 标题中的每个数据点必须在正文中有出处
   - 标题的方向性判断必须与正文结论一致
   - 标题引发的期待必须被正文满足(否则=欺骗→取关)

验证清单:
□ 标题中的数字在正文中存在?
□ 标题的结论方向=正文结论方向?
□ 读者点击后会觉得"标题说的确实有"而非"标题骗我进来"?
□ 删掉标题中任何正文不支持的暗示?
```

### 铁律3: 洞察驱动，非夸张驱动 (Insight > Hype)

```
❌ 过度标题党特征:
   - 空洞情绪词: "Shocking!" "Insane!" "Mind-blowing!"
   - 虚假紧迫: "You MUST see this NOW!"
   - 过度承诺: "The ONLY strategy you'll ever need"
   - 情绪操控: 与内容不匹配的恐惧/贪婪

✅ 高传播力的正确来源:
   - 真实的认知冲突: "好公司≠好投资" (来自数据，非炒作)
   - 具体的反直觉数据: "57% ROIC，但期望收益仅+1%"
   - 可验证的时间窗口: "周三财报后这组数据将被检验"
   - 结构性悬念: 来自信息本身的张力，非人工制造

判断标准:
"如果去掉所有形容词和情绪词，这个标题是否仍有信息量?"
→ 有 = 洞察驱动 ✅
→ 无 = 夸张驱动 ❌
```

## Part 1: 标题底层原理 (Headline Psychology Fundamentals)

### 1.1 七大心理驱动力

| # | 原理 | 机制 | 效果倍数 | 来源 |
|---|------|------|---------|------|
| 1 | **好奇心缺口** | 已知与想知之间的空隙产生"剥夺感" | 3-5x CTR | Loewenstein/CMU |
| 2 | **惊讶/打破模式** | 大脑快感中枢对意外的激活>预期 | 2-3x 记忆 | Heath兄弟 |
| 3 | **具体数字** | 大脑信任具体>模糊; 奇数>偶数 | +20% CTR | CMI研究 |
| 4 | **损失厌恶** | 失去的痛=获得的乐×2 | 2x 动力 | Kahneman/Tversky |
| 5 | **情绪唤醒** | 高唤醒(敬畏/愤怒/焦虑)>低唤醒(悲伤) | 70%病毒内容 | Berger/Milkman |
| 6 | **认知流畅** | 越容易理解→越觉得可信 | +信任度 | 认知科学 |
| 7 | **身份指认** | "这是写给我的"→大脑生存本能激活 | +关注度 | 受众心理学 |

### 1.2 Dan Koe的RAE框架

```
一个标题必须同时满足3个条件:

R = Relevance (相关性)
    "这跟我的日常生活/投资决策有什么关系?"
    → 痛点=已有 或 利益=想要

A = Awareness (认知匹配)
    "我能理解这个概念的级别吗?"
    → 新手用简单词, 专家用精确术语

E = Effort (获取成本)
    "我需要花多少精力得到答案?"
    → 暗示低成本高回报 (数字/时间框架)
```

### 1.3 Dan Koe的Hook构建模块

| 模块 | 定义 | 投资大师示例 |
|------|------|------------|
| **BIG Problem** | 所有痛点浓缩为一句 | "大多数投资者在周期顶部买入" |
| **BIG Benefit** | 所有利益浓缩为一句 | "识别周期位置的量化方法" |
| **BIG Idea** | 最有冲击力的核心概念 | "好公司≠好投资" |
| **Transformation** | 数字/命名暗示过程 | "5维评分系统" / "Top Divergence信号" |
| **Timeframe** | 量化时间/阅读成本 | "3分钟理解" / "20年历史数据" |
| **Historical Pattern** | 暗示历史重复性 | "历史上每次50x PE后12个月内..." |

### 1.4 Gene Schwartz 10大标题强化术 (从38术中精选)

| # | 技术 | 原理 | 示例 |
|---|------|------|------|
| 1 | **量化承诺** | 具体>模糊 | "避开亏损"→"避开45%的暴跌" |
| 2 | **量化速度** | 时间框架+紧迫 | "识别信号"→"30秒识别顶部信号" |
| 3 | **对比强化** | 参照物放大 | "比华尔街更准"→"准确率是分析师的3倍" |
| 4 | **隐喻转化** | 抽象→感官化 | "避免风险"→"别在蒸汽压路机前捡硬币" |
| 5 | **感官化** | 让读者看到/感受到 | "亏损"→"看着账户每天缩水$5000" |
| 6 | **案例证明** | 具体实例 | "有效"→"LRCX验证: 3个月前预测, 今天应验" |
| 7 | **戏剧化** | 放大结果的冲击 | "框架有用"→"一个AI vs 25位顶级分析师" |
| 8 | **悖论表达** | 违反直觉 | "最好的季报=周期见顶信号" |
| 9 | **消除限制** | 移除反对理由 | "不需要金融学位" / "完全免费" |
| 10 | **身份关联** | 与目标身份绑定 | "逆向投资者的秘密武器" |

## Part 2: 标题评分系统 (Headline Scoring System)

### 2.1 八维标题评分

对每个候选标题评分(0-10):

```
Headline Score = H×0.20 + E×0.15 + A×0.15 + D×0.15 + L×0.10 + I×0.10 + N×0.10 + S×0.05

H = Hook Power (第一眼冲击力)
E = Emotion (情绪唤醒强度)
A = Accuracy (与内容的匹配度)
D = Debate (引发讨论/回复的潜力)
L = Length Fit (平台字数适配)
I = Identity (身份共鸣度)
N = Numbers (具体数字/量化)
S = Shareability (2阶选美通过率)
```

### 2.2 评分基准

| 总分 | 标题质量 | 行动 |
|------|---------|------|
| ≥8.5 | 极强标题 | 直接使用 |
| 7.0-8.4 | 强标题 | 微调后使用 |
| 5.5-6.9 | 中等 | 用Schwartz强化术改写 |
| <5.5 | 弱标题 | 丢弃重写 |

### 2.3 快速自检清单

```
□ 能在2秒内理解核心信息? (认知流畅)
□ 触发至少2个心理驱动力?
□ 包含具体数字或量化?
□ 有好奇心缺口(读者必须点击才能填补)?
□ 在高唤醒情绪象限?
□ 适配目标平台字数限制?
□ 通过2阶选美("分享这个标题让我显得聪明")?
□ 不可替代性(只有我们能写这个)?
□ RAE三要素满足(相关/适配/低成本)?
```

## Part 4: 合成受众标题测试协议 (Synthetic Headline Testing)

### 4.1 标题A/B/C测试流程

对每个内容单元生成3个标题变体，用合成受众测试:

```
Step 1: 生成3个变体
  - 变体A: 高情绪唤醒(焦虑/敬畏)
  - 变体B: 高好奇心缺口(悬念/反直觉)
  - 变体C: 高身份共鸣(受众指认/社交货币)

Step 2: 7类Persona快速反应测试 (30人/类)
  对每个变体评估:
  - 点击意愿 (0-10)
  - 情绪反应 (哪种情绪?)
  - 是否会回复/评论?
  - 是否会转发? (2阶选美检验)
  - 第一反应是什么词?

Step 3: 加权得分计算
  Score = Σ(persona_weight × click_score × influence)

Step 4: 选择最高分变体, 或融合两个变体的优势元素
```

### 4.2 快速标题迭代法 (10×10规则)

```
规则: 写10个标题, 选最好的1个, 再写10个变体, 选最终版

Round 1 (10个初始):
  - 2个对抗型
  - 2个数据型
  - 2个好奇型
  - 2个损失型
  - 2个身份型
  → 选Top 1

Round 2 (10个变体):
  - 用Schwartz 10术对Top 1进行10次强化变形
  → 选最终版

```

## Part 5: 从信息到多平台标题的完整协议

### 5.1 输入→输出流程

```
INPUT: 任意信息/数据/事件
       ↓
Step 1: 核心洞察萃取
  "这条信息最让人吃惊/焦虑/兴奋的点是什么?"
       ↓
Step 2: 应用Dan Koe构建模块
  BIG Problem / BIG Benefit / BIG Idea / Transformation / Timeframe / Negative
       ↓
Step 3: 应用Schwartz强化术 (选2-3个)
  量化 / 速度 / 对比 / 隐喻 / 感官 / 证明 / 戏剧 / 悖论 / 消除限制 / 身份
       ↓
Step 4: 生成3变体(情绪/好奇/身份)
       ↓
Step 5: 平台适配
  X(47-73字符) / YouTube(60-70字符) / Reddit(80-150字符+标签)
  雪球(30-60中文字+$代码) / 小红书(≤20中文字+关键词)
       ↓
Step 6: 合成受众测试 → 选最优
       ↓
OUTPUT: 5平台×最优标题
```

### 5.2 实战示例

**INPUT**: "LRCX Q4营收$4.5B超预期, 但guidance低于预期$200M, 股价盘后-5%"

**Step 1 核心洞察**: 3个月前框架评分23.5/100——Beat but Guide Down正是Top Divergence模式的典型表现

**Step 2 Dan Koe模块**:
- BIG Idea: "好季报=周期见顶的典型模式"
- Transformation: "23.5/100评分系统"
- Timeframe: "3个月前的框架评分"
- Negative: "Beat后股价反跌5%"

**Step 3 Schwartz强化**:
- 量化: "23.5分" + "3个月前" + "-5%"
- 对比: "25分析师共识 vs 周期框架评分"
- 悖论: "最好的季报=周期见顶的典型模式"

**Step 4 三变体**:
- 数据冲突型: "LRCX beat earnings and still dropped 5%. The cycle framework scored 23.5/100 three months ago."
- 提问型: "What if the best quarter IS the peak signal? Here's what history shows."
- 现象型: "25 analysts celebrated the beat. The stock dropped. A familiar pattern at cycle peaks."

### 6.1 通用高效标题模板 (20个)

```

4. "[数量]个[标的]的数据中，隐藏着1个被忽视的规律"
```

## Part 7: 面向顶级投资人群的特殊规则

### 7.1 投资人群的标题反感清单

```
❌ 过度承诺: "guaranteed returns" "never lose money"
❌ 模糊数据: "a lot" "very good" "amazing returns"
❌ 空洞声明: "best strategy ever" "game-changing"
❌ 不专业语气: "OMG" "insane" (在X上可适度使用)
❌ 无来源的权威: "experts say" (哪个expert?)
❌ 合规风险: "投资建议" "保证收益" "稳赚不赔"
```

### 7.2 投资人群的标题信任放大器

```
✅ 具体数字+来源: "23.5/100 (v5.0 cycle framework)"
✅ 可验证声明: "publicly tracked" "open-source"
✅ 风险明示: "probability-weighted" "scenario analysis"
✅ 透明度信号: "methodology open-source" "full data available"
✅ 专业术语(适度): "ROIC" "EV/EBITDA" "FCF Margin"
✅ 时间锚定: "3 months ago" "since 2018" "20-year data"
✅ 客观框架: "cycle framework" "quantitative scoring" "historical pattern"
```

### 7.3 投资人群的2阶选美标题设计

```
转发者获得的身份标签:
- "关注非共识数据信号的人" → 用: 数据冲突型标题
- "有估值纪律的理性投资者" → 用: 历史类比型标题
- "掌握量化周期方法论的先驱" → 用: 教育型标题
- "独立思考不跟风的投资者" → 用: 提问型标题

最高V₃标题特征:
1. 分享者无需额外解释就能展示自己的品味
2. 标题本身就是一个可独立传播的洞察
3. 看到标题的人会觉得"分享这个的人很有insight"
```
