**框架1: Yield-Learning Model（良率学习曲线）**

**学术价值**:
```
来源: Wright (1936) + Weber (2004) + Tirkel (IEEE 2013)

核心公式:
Y(t) = Y₀ + (Y∞ - Y₀) × (1 - e^(-λt))

应用于TSM:
- Y₀ = 55%（N2初始良率）
- Y∞ = 95%（极限良率）
- λ = 0.14（TSM学习率 vs Samsung 0.10）

预测能力:
- 2026 Q4 N2良率: 75%±3ppt
- 2027 Q2 N2良率: 82-85%
- 学习率差距 = 护城河宽度量化（TSM领先Samsung 15ppt/18月）
```

| 方法 | 传统分析 | Yield-Learning Model | 提升 |
|------|---------|---------------------|------|

| 竞争优势 | 定性（"技术领先"） | 定量（λ=0.14 vs 0.10） | 可量化+100% |
| 周期预测 | 无 | 良率>85%后6-9月见顶 | 新增能力 |

**步骤1: 确定估值锚**
- 选择SOTP概率加权$341作为基础锚（最保守）

**步骤2: 逐项调整**
```
$341（基础）
× 1.25（周期溢价+25%）
× 1.15（护城河溢价+15%）
× 1.20（AI结构性溢价+20%）
```

| **5. Mermaid图复杂度** | 部分图节点过多 | 可读性下降 | 简化至<10个节点 |

### 3.3 MU v18.2 vs TSM v18.3框架选择

**场景1: 周期股分析（如半导体、化工）**
- **推荐**: TSM v18.3
- **理由**: 4层Radar System + EUV良率洞察 + 周期温度计
- **适用**: LRCX, AMAT, ASML, MU

**场景2: 成长股分析（如SaaS、消费）**
- **推荐**: MU v18.2（简化版）
- **理由**: 不需要Radar System，聚焦护城河+产品飞轮
- **适用**: NVDA, CRM, SNOW

**场景3: 对外发布报告**
- **推荐**: TSM v18.3
- **理由**: Mermaid可视化 + Insight Cards更适合展示
- **适用**: 面向客户/公众的研究报告

**场景4: 内部投资决策**
- **推荐**: 两者结合
- **理由**: MU v18.2估值纪律 + TSM v18.3学术深度
- **适用**: 自营投资/基金内部研究
