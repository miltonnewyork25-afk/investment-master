## Part 1: 今天的公司

### 1.1 财务现状

PLTR今天是一家**增速惊人、利润率极高、但估值极端、且高度依赖美国市场**的公司。

| FY2026指引 | $7.18-7.20B (+61%) | 管理层极度自信 [硬数据: 财报电话会] |

### 1.2 产品本质

**一句话**: PLTR不是数据仓库、不是AI API、不是BI工具——它是把异构数据源变成可操作决策的**数据操作系统**。

核心技术是**Ontology**（本体论引擎）：把数据库、API、传感器映射为统一的数字孪生模型，让非技术人员也能在真实数据上构建决策流程。AIP在此基础上嵌入LLM，提供企业级AI能力。

```
传统方式: 数据 → 数据仓库 → BI报表 → 人类解读 → 决策 → 执行
PLTR方式: 数据 → Ontology(数字孪生) → AIP(AI决策) → 直接执行
                                                ↑
                                    差异化: 从"看数据"到"用数据做事"
```

### 1.3 市场结构

| 细分 | FY2025营收 | 增速 | 占比 | 信号 |
|------|----------|------|------|------|
| 美国政府 | ~$2.41B | +30-35% | 54% | 稳定但受DOGE风险 [硬数据] |
| 美国商业 | ~$1.45B | **+109%** | 32% | AIP驱动的爆发 [硬数据] |
| 国际政府 | ~$0.55B | +12% | 12% | NATO/UK，增速较低 [硬数据] |
| 国际商业 | ~$0.61B | **+2%** | 2% | 结构性困境 [硬数据] |

**最重要的事实**: 美国商业+109%和国际商业+2%之间的107个百分点差距，不是暂时现象——CEO Karp自己也没有解释清楚。

## Part 2: 产品边界与天花板分析（类型B适配）

> 对PLTR这类公司，核心问题不是"会变成什么"，而是"能做多大"。以下分析PLTR产品的边界在哪里。

### 2.1 产品适用边界

PLTR的Ontology+AIP**适用**于什么类型的组织？

**强适用**:
- 数据源多且异构（军事、医疗、制造）
- 决策后果严重（国防、反恐、医药）
- 安全要求极高（FedRAMP High/IL6/TS-SCI）
- 需要跨系统协作（大型企业/政府机构）

**弱适用或不适用**:
- 数据结构简单的企业（中小零售、本地服务）
- 已有成熟数据栈的科技公司（Google/Meta不需要PLTR）
- 成本敏感型客户（Ontology部署成本$2.5-7.5M）
- 隐私/本地化要求高的市场（欧盟GDPR → 国际商业+2%的根因之一）

**关键洞察**: PLTR的产品天然适合"复杂+高安全+高价值"场景。这个市场够大吗？取决于你怎么画边界——如果只算"复杂数据决策"的企业，可能是$50-100B的市场；如果PLTR能让"不复杂"的企业也变得复杂（AIP降低门槛），市场可以扩展到$200-500B。

### 2.2 采纳曲线位置

PLTR目前处于S曲线的哪个阶段？

```
     ┌──────────────────────────────────────┐
100% │                              ╭───── │  ← 主流采纳
     │                          ╭───╯      │
     │                      ╭───╯          │
     │                  ╭───╯              │
     │              ╭───╯                  │
     │    ○ PLTR ─→╱                       │  ← 早期大众
     │          ╭─╯  "鸿沟"                │
     │      ╭───╯                          │
     │  ╭───╯                              │  ← 早期采纳者
     │──╯                                  │
   0%└──────────────────────────────────────┘
      创新者   早期     鸿沟   早期    主流    落后
              采纳者         大众
```

**判断**: PLTR正在穿越鸿沟。

**支持证据**:
- 美国商业客户571家(+49% YoY) → 不再是少数先行者 [硬数据]
- Bootcamp模式（5天试用→签约→扩展）→ 降低采纳门槛 [硬数据]
- NDR 139% → 用过的人在买更多 [硬数据]
- Top 20客户平均$94M TTM → 大客户在深度嵌入 [硬数据]

**反面证据**:
- 客户增速在放缓（+34% → +27%趋势）→ 可能是"容易抓的客户已经抓完了" [硬数据]
- Karp自己说"未来增长靠ARPU不靠客户数" → 隐含客户拓展遇到阻力 [硬数据: 财报电话会]
- 国际商业+2% → 在非美国市场完全没有穿越鸿沟 [硬数据]

### 2.3 天花板在哪里

三个层级的天花板：

**天花板1 — 地理**: 如果国际商业保持+2%（结构性），PLTR的可触达市场限于美国。美国企业AI市场~$200-250B。

**天花板2 — 客户类型**: 如果PLTR只能服务"数据复杂+高安全需求"的组织，可触达客户可能只有Fortune 500中的200-300家+政府机构。假设平均$50-100M/年 → $10-30B天花板。

**天花板3 — 竞争替代**: 如果Hyperscaler（Azure Fabric/AWS）在2-3年内复制了Ontology 80%的功能，PLTR的差异化缩小到"最后20%的安全和定制" → 定价权被侵蚀。

**天花板的另一面 — 如果AIP真的降低了门槛**: Bootcamp让原本需要6-9个月部署的Ontology变成5天试用。如果这个模式持续有效，PLTR可能打破天花板2（让更多企业成为客户）。但天花板1（地理）和天花板3（竞争）不受Bootcamp影响。

### 2.4 护城河深化还是侵蚀？

| 护城河维度 | 当前强度 | 趋势 | 依据 |
|----------|:-------:|:----:|------|
| 转换成本(Ontology) | 9/10 | → 稳定 | 重新部署$2.5-7.5M，6-9个月 [合理推断] |
| 安全壁垒(政府) | 10/10 | → 稳定 | 20年安全认证积累，不可复制 [硬数据] |
| 数据锁定 | 7/10 | ↗️ 加深 | NDR 139%说明数据越用越深 [硬数据] |
| 网络效应 | 4/10 | ? 不确定 | 目前不是多边平台，但AIP可能改变这一点 |

**核心判断**: 护城河对已有客户在加深（NDR 139%），但对新客户的获取能力是否在加强还是减弱，Bootcamp数据给出了矛盾信号（签约快，但客户数增速在放缓）。

### 2.5 量级的关键变量

PLTR能做多大，取决于以下变量的组合：

```
最终量级 = f(AIP采纳广度, 国际突破, 竞争替代速度, ARPU持续性)

宽度可能:
  如果AIP成为企业标配 + 国际突破 + 竞争护城河持续
    → $30-50B营收的"企业AI平台"公司

  如果AIP仅限复杂场景 + 美国为主 + 护城河持续
    → $15-25B营收的"高端企业软件"公司

  如果竞争侵蚀 + 增速放缓
    → $7-12B营收的"政府+高端企业"利基公司
```

> **注意**: 以上不是估值，不赋概率。只是描述量级的可能范围。

## Part 3: 开放问题

### Q1: AIP的Bootcamp模式是否可持续规模化？

**类型**: 量级问题
**为什么重要**: Bootcamp是US Commercial +109%的核心驱动力。如果Bootcamp效果衰减（抓完了"容易抓"的客户），增速会断崖
**可观测信号**:
  - 每季度新增Bootcamp客户数量趋势
  - Bootcamp→正式合同的转化率变化
  - 每个Bootcamp的平均首年ACV（如果下降=客户质量在下降）
**时间窗口**: 2026H2是关键——面临最难的同比基数（2025H2 Bootcamp起飞期）

### Q2: 国际商业+2%是结构性还是时机性？

**类型**: 存在问题
**为什么重要**: 如果结构性 → TAM被锁死在美国（$200-250B）；如果时机性 → 全球$500B TAM仍可触达
**可观测信号**:
  - 欧洲GDPR对数据OS产品的实际执法力度
  - 国际商业客户数量（而非仅营收）变化
  - 是否有任何国际大单（>$10M）突破
  - AIP Bootcamp在非美国市场的推广进度
**时间窗口**: 如果2026全年国际商业仍<+10%，大概率是结构性

### Q3: Hyperscaler什么时候能复制Ontology？

**类型**: 时机问题
**为什么重要**: Azure Fabric/AWS已经覆盖90%的企业，如果能复制Ontology 80%的功能 → PLTR的差异化仅剩"安全+最后20%定制" → 定价权被压缩
**可观测信号**:
  - Microsoft Fabric/AWS对"操作型AI"（非分析型）的功能发布节奏
  - Databricks ($100B估值, +50%增速) 的产品路线图是否向Ontology方向演进
  - PLTR客户的竞品替换案例（目前有吗？）
**时间窗口**: 2-3年内应该能看到方向

### Q4: DOGE对PLTR是净正还是净负？

**类型**: 反转问题
**为什么重要**: 市场共识认为DOGE削减政府预算 = 对PLTR不利。但DOGE的目标是"政府效率化" → 可能反而增加对PLTR类工具的需求
**可观测信号**:
  - DOGE削减后，新的"效率化"项目合同是否流向PLTR
  - 国防部AI预算($134B)是增加还是被DOGE削减
  - PLTR政府收入增速在DOGE政策落地后是加速还是减速
**时间窗口**: 2026Q2-Q3（DOGE政策效果开始显现）

### Q5: SBC稀释何时收敛？

**类型**: 量级问题
**为什么重要**: $769M SBC = 营收的17%。GAAP和Adjusted利润率差16pp。如果SBC不收敛，"50%利润率"的叙事是虚假的
**可观测信号**:
  - SBC占营收比例趋势（FY2023 25% → FY2025 17% → ?）
  - Karp个人持续减持（$2.2B已卖 + $1.0B计划中）是否加速
  - 稀释后股数增长率
**时间窗口**: 持续追踪

### Q6: 从"高端定制"到"标准化平台"的转型是否在发生？

**类型**: 存在问题
**为什么重要**: 定制项目 = 高毛利但规模不经济。标准化平台 = 规模经济但可能降低差异化。PLTR的未来量级完全取决于走向哪边
**可观测信号**:
  - 平均部署时间变化（从6-9个月→Bootcamp 5天，是真的标准化了吗？还是只是前端简化了？）
  - 自助服务(self-service)的进展
  - 客户成功团队人数/客户比率变化
**时间窗口**: 2-3年

## Part 5: 转折点指南

### 信号A: Bootcamp增长拐点

**二阶变化**: 不是"Bootcamp签约了多少客户"，而是**"每季度Bootcamp→正式合同的转化率是否在变化"**
- 如果转化率上升 → AIP产品力在增强
- 如果转化率持平但Bootcamp数量在增 → 靠推广而非产品力
- 如果转化率下降 → "容易抓的客户"已经用完，增长见顶信号

### 信号B: 国际破冰

**反事实信号**: 如果PLTR发布一系列国际合作伙伴/渠道协议 → 意味着他们承认直销模式在海外不行，在转变策略（这本身是积极信号）。如果继续沉默 → 结构性困境确认。

### 信号C: Hyperscaler进入操作层

**跨维度共振**: 当以下信号同时出现时 → PLTR差异化受到实质威胁：
- Microsoft发布"操作型AI"产品(不是分析报表，是直接驱动业务行动)
- 至少1家PLTR大客户公开测试Azure替代方案
- PLTR的NDR开始下降(从139%往下)

### 信号D: SBC-增长脱钩

**二阶变化**: 追踪SBC占营收比例的变化速度。
- 如果SBC比例下降速度加快（17%→12%→8%）→ 真正的利润率释放
- 如果SBC比例下降停滞或反弹 → "不断用股权换增长"的模式没有改变

### 噪音过滤

以下不是转折点：
- Karp在媒体上的哲学宣言（信号/噪音比极低）
- 单笔大额政府合同（波动性大，不代表趋势）
- 分析师目标价调整（跟随性强）
- 短期股价波动（P/E 231x本身就意味着高波动是常态）

## Part 6: 价格在赌什么

### 6.1 当前价格分解

**$139.51/股 = $319B市值 = P/E 231x = EV/Sales 78x**

市场在为PLTR定这个价格时，隐含的预期：

| 隐含预期 | 数值 | 合理性 |
|---------|------|--------|
| 未来5年营收CAGR | ~35-40% | 需要从$4.5B增长到$20-25B [合理推断: PE反推] |
| 终态净利率 | 25-30% | 与顶级SaaS一致，可能合理 |
| 终态P/E | 40-50x | 假设增速放缓后仍享有溢价 |

### 6.2 买入$139的投资者在赌什么

1. **AIP是一次性的范式突破** — 不是普通的产品更新，而是让PLTR从利基走向主流的催化剂
2. **美国市场足够大** — 即使国际不行，美国企业AI市场也能撑起$20B+营收
3. **Hyperscaler不会进入操作层** — 或者进入了但做不好（复杂系统集成非互联网公司所长）
4. **增速能维持3-5年** — 60%→40%→25%的有序减速，而非断崖

### 6.3 价格可能忽略的

- **GAAP vs Adjusted的16pp差距**: 如果市场用Adjusted利润率(50%)给估值但SBC永远不收敛 → 利润率叙事有水分
- **客户增速放缓的含义**: Karp说"未来靠ARPU"实际上是承认水平扩张遇到瓶颈。靠ARPU增长的公司估值倍数通常低于靠客户数增长的公司
- **国际+2%不仅是增长问题，更是"产品是否真的通用"的验证**: 如果Ontology只在美国好用，它可能不是"数据操作系统"而是"美国政府+企业的定制工具"

### 6.4 当前体制分类

**高不确定性积累期** — 但正在接近**证据收敛期**的转换点

AIP Bootcamp的数据正在快速积累。2026H2面临最难基数，将是第一次真正检验：增速是从109%有序回落到50-60%（牛市逻辑成立），还是断崖到20-30%（鸿沟没有穿越成功）。

- Microsoft/AWS/Google内部是否在开发类Ontology的操作层产品？如果是，2-3年后PLTR的差异化壁垒会被侵蚀多少？外部无法观测。

**4. Karp出售股票的真实动机**
- $2.2B已售+$1.0B计划。10b5-1预设计划是合理解释，但也可能是"内部人知道估值不可持续"。两种解释都有道理，无法从外部区分。

**5. "企业数据操作系统"这个品类是否真的存在**
- PLTR定义了"数据操作系统"这个品类。但如果这个品类不被广泛认可——如果企业更愿意用Hyperscaler的"够用"方案——那PLTR的TAM就被大幅高估了。这是一个存在性未知。

## 框架适配反思

**类型B(量级不确定性)与类型A(类别不确定性)的差异**:

TSLA的发现系统报告试图回答"这家公司会变成什么"→ 5个质的不同的未来状态。
PLTR的发现系统报告试图回答"这个产品能做多大"→ 产品边界+采纳曲线+天花板分析。

**Part 2的差异最显著**: TSLA有"能力基元→未来状态推演"，PLTR换成了"产品适用边界→采纳曲线→天花板层级"。这不是同一个分析模板套不同公司——而是识别出不确定性的本质不同后，用不同的分析工具。

**Part 3-7基本通用**: 开放问题、证据方向、转折点检测、价格含义、未知清单的框架结构在两家公司上都适用，只是具体内容不同。
