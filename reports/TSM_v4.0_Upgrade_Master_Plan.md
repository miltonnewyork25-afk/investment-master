## 3. 架构设计：AMD混合模式

### 3.1 为什么选择混合模式

| 模式 | 标注密度 | 速度 | 风格一致性 | 案例 |
|------|---------|------|-----------|------|
| **AMD混合** | **16.3/万** | 中 | 高 | AMD v3.1 |
| GOOGL全并行 | 14.8/万 | 快 | 中 | GOOGL v3.5 |
| COST多路并行 | 15.2/万 | 快 | 中 | COST v20 |

**选择理由**: 标注密度最高 + 风格一致性最佳。TSM v2.0的核心问题正是标注缺失(0.09/万)，必须用最可靠的方式解决。

### 3.2 两层架构图

```
Phase 0 (并行): 7 Agent数据预取 + MDB扫描 → shared_context + CQ
                    ↓
Phase 1-4 (单线程): 主Agent逐Phase写入，引用P0数据+WebSearch补充
  P1: 定位+生态 ≥28K → FG → commit
  P2: 财务+估值 ≥33K → FG → commit
  P3: 战略+引擎 ≥37K → FG → commit
  P4: 对抗+决策 ≥35K → FG → commit
                    ↓
Phase 5 (整合): 合并P1-4 → TSM_v4.0_Complete.md ≥130K → 7/7 FG
```

### 3.3 单线程模式的工作节奏

AMD验证的黄金节奏：**每650字符≈1个标注**

```yaml
单模块写作流程:
  1. 叙述开篇 (200字符) — 设立论点
  2. 数据表格 (300字符) — 结构化数据呈现
  3. 标注 [A: 来源, 日期] 或 [B: 来源, 日期]

  关键: 表格+标注+分析 紧密交织，不是先写完再补标注
```

## 5. 标注密度优化策略 (AMD v3.1核心教训)

### 5.1 模块级密度分层

AMD实测: 整体16.3/万 = 高密度模块(20+/万)拉高 + 叙述模块(10-12/万)

```yaml
高密度模块 (≥18/万) — 占总字符~35%:
  - SOTP分部估值: 每个分部=5行表格+2个标注+推导[E:] → ~22/万
  - 5年财务趋势: 每年=1行数据+1个[A:]财报标注 → ~20/万
  - 护城河量化矩阵: 每维度=得分+数据+标注 → ~18/万
  - 技术路线图对比: 每工艺节点=参数+时间+标注 → ~18/万

中密度模块 (14-16/万) — 占总字符~35%:
  - 产业链映射: 每节点=数据+标注
  - 可比公司估值: 每公司=倍数+标注
  - 客户分析: 每客户=份额+标注
  - Kill Switch: 每KS=阈值+概率+标注

低密度模块 (10-12/万) — 占总字符~30%:
  - 公司画像叙述: 历史沿革为主
  - 行为金融分析: 定性判断为主
  - 反证挑战: 推理论证为主
  - 市场注意力雷达: 整理为主

加权平均: 0.35×20 + 0.35×15 + 0.30×11 = 7.0+5.25+3.3 = 15.55/万 ≥15 ✅
```
