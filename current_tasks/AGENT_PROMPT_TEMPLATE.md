# Agent Prompt 模板 v1.0

> 基于AMD v3.1验证的最佳实践。每个SubAgent dispatch前复制此模板填写。

---

## 模板

```markdown
# Agent任务: [模块名称]

## 上下文
- **公司**: [TICKER] ([公司名])
- **Phase**: [Phase N]
- **CQ关联**: 回答 Core Question #[N]: "[具体问题全文]"
- **前序发现**:
  - [Phase前序关键结论1]
  - [Phase前序关键结论2]
  - [Phase前序关键结论3]

## 具体数据需求
请搜索并返回以下数据:
1. [指标1]: [来源建议, 如"Yahoo Finance/SEC 10-K"]
2. [指标2]: [来源建议]
3. [指标3]: [来源建议]
4. [指标4]: [来源建议]
5. [指标5]: [来源建议]
(至少列出5个具体数据需求)

## 分析要求
- [具体分析任务1]
- [具体分析任务2]
- [具体分析任务3]

## 输出格式要求
- **字符目标**: [X,000-Y,000] 字符
- **标注格式**: 每个数据点必须标注来源
  - 一级数据(财报/SEC): `[A: 来源, 日期]`
  - 二级数据(数据库/平台): `[B: 来源, 日期]`
  - 预测市场: `[P: 来源, 日期]`
  - 分析师估算: `[E: 推导说明]`
- **标注密度**: ≥15个标注/万字符 (约每650字符至少1个标注)
- **表格**: ≥[N]张数据表格
- **Mermaid图**: ≥[N]个 (如适用)

## 禁止事项
- 不写"众所周知"/"显而易见"等模糊表述
- 不编造数据 — 搜索不到时标注"该指标未获取到公开数据"
- 不写空洞总结段落
- 不重复Phase前序已有的内容
- 不使用"建议买入"/"建议卖出"
```

---

## 使用指南

### 何时使用
- Phase 0: 每个数据预取Agent
- Phase 1-4: 可并行执行的独立模块

### 关键要点
1. **数据需求越具体越好**: "查2024-2025年季度营收" >> "查财务数据"
2. **前序发现必须提供**: 避免Agent重复已有结论
3. **字符目标要给范围**: 太少缺深度，太多无法整合
4. **标注密度是硬指标**: 达不到15/万的Agent产出需要主Agent补充标注

### AMD验证数据
- 9个Phase 0 Agent: 全部返回有效数据 (平均8K字符)
- Prompt详细度与输出质量正相关: r≈0.85
- SubAgent标注率: 约10-12/万 (主Agent需补充到15/万)
