# Core Questions 方法论 v1.0

> 争议驱动分析的核心引擎。将 Phase 0.5 发现的市场维度转化为 5-8 个 Core Questions，以此组织整篇报告。

---

## 设计哲学

**旧模式**: 按 U1→U25 线性填空 → 读者找不到"为什么要看这只股"
**新模式**: 先提取市场最关心的问题 → 报告围绕这些问题组织 → 模块成为回答问题的素材

---

## Step 1: 从 Top 10 维度提取 Core Questions

### 1.1 转化规则

将 Phase 0.5 的 Top 10 维度转化为投资者视角的问题：

```
维度: "Kirkland自有品牌占比持续上升"
  → CQ: "Kirkland品牌扩张是否会侵蚀供应商议价权，反噬毛利率？"

维度: "会员费提价周期"
  → CQ: "会员费提价能否在不流失会员的情况下贡献增量利润？"

维度: "中国/海外扩张速度"
  → CQ: "国际扩张的单店经济模型能否复制北美的成功？"
```

### 1.2 合并与精简

- 语义重叠的维度合并为一个 CQ（如"毛利率压力"和"成本通胀"→同一CQ）
- 最终保留 **5-8 个 CQ**，按注意力分加权排序
- 每个 CQ 必须是具体的、可用数据回答的投资问题

### 1.3 CQ 质量检验

每个 CQ 必须通过三项检验：

| 检验 | 标准 | 不通过处理 |
|------|------|-----------|
| **具体性** | 能指向具体业务/财务变量 | 拆分或重写 |
| **争议性** | 市场存在分歧（非共识结论） | 降级为背景信息 |
| **可回答性** | 可用公开数据/合理推断回答 | 标注数据限制后保留 |

## Step 2: CQ-模块相关性评分

### 2.1 评分矩阵

对每个 CQ，评估所有模块（通用 U1-U23b + 行业 C1-C10 等）的相关性：

```
相关性评分:
  3 = 高相关: 该模块是回答此CQ的核心素材
  2 = 中相关: 该模块提供有用的辅助信息
  1 = 低相关: 该模块与此CQ关系微弱
  0 = 无关: 该模块与此CQ无关
```

### 2.2 模块执行权重

根据相关性评分确定模块深度：

| 模块最高相关性 | 执行深度 | 字数指导 |
|:---:|---------|:---:|
| 3 (高) | 完整深度展开 | ≥2,000字符 |
| 2 (中) | 聚焦核心段落 | 500-1,000字符 |
| 1 (低) | 3行摘要 | ≤150字符 |
| 0 (无) | 可跳过（仅保留标题） | 0 |

### 2.3 输出模板

```markdown
## CQ-模块相关性矩阵

|  | CQ1: ... | CQ2: ... | CQ3: ... | 最高分 | 执行深度 |
|--|:---:|:---:|:---:|:---:|------|
| U7 财务数据 | 3 | 2 | 1 | 3 | 完整 |
| U12 护城河 | 1 | 3 | 2 | 3 | 完整 |
| U17 学术框架 | 0 | 0 | 1 | 1 | 摘要 |
| C3 PVM拆解 | 3 | 0 | 0 | 3 | 完整 |
```

## Step 3: 报告结构生成

### 3.1 章节组织

报告按 CQ 分章，而非按 U 编号分章：

```markdown
# [公司名] 投资分析

## 执行摘要 (500字)

## CQ1: [最高注意力分的问题]
  ### 结论先行 (200字)
  ### 证据展开 (引用相关模块的分析)
  ### 反证与风险
  ### 置信度标注

## CQ2: [第二高的问题]
  ...

## CQ5-8: [剩余问题，可合并为一章]

## 综合判断
  ### 估值 (引用U19/U20)
  ### Kill Switch (引用U21)
  ### 行动建议 (引用U23)

## 附录: 完整数据表格
```

### 3.2 模块内容分配

一个模块可能服务于多个 CQ——将其分析内容拆分到各相关章节，而非集中在一处：

```
U12 护城河分析:
  → CQ1(品牌扩张)章节: 引用品牌护城河部分
  → CQ3(竞争威胁)章节: 引用转换成本/规模护城河部分
  → 附录: 完整护城河评分表
```

### 3.3 未映射模块处理

某些模块可能与所有 CQ 相关性都低（评分≤1），但属于必选模块（如 U1 公司画像）：

- **必选且低相关**: 压缩到 3 行摘要，放入附录
- **可选且低相关**: 标注"本次分析略过，原因: 与核心问题相关性低"

## 与现有流程的集成

### 执行时序

```
Phase 0.5 (现有): 市场雷达 → Top 10 维度
  ↓ 新增 Step 2.5
CQ 提取: Top 10 → 5-8 Core Questions
  ↓ 新增 Step 3.5
模块评分: CQ × 模块 → 相关性矩阵
  ↓ 新增 Step 5.5
结构生成: CQ 驱动的报告大纲
  ↓
Phase 1-5: 按 CQ 组织执行，模块带权重
```

### 与 depth_assurance.md 的关系

- 旧标准: "每模块≥7分" → 所有模块等权
- 新标准: "高相关模块≥7分，低相关模块可压缩" → 资源集中在核心问题

### 与 Hot-Patch 的关系

- Hot-Patch 模块自动获得相关性评分 3（因为它们本就是为回答特定维度生成的）
- Hot-Patch 内容直接嵌入对应 CQ 章节

## Step 4: CQ置信度演化表 (v2.0新增)

> **来源**: AMD报告CQ-Weighted Confidence 47.1% — 每个CQ跨Phase的置信度变化轨迹是重要的元分析信号。

### 4.1 Phase逐级置信度追踪

Phase 5中对每个CQ输出置信度演化表：

```markdown
### CQ置信度演化

| CQ | P0.5初始 | P1 | P2 | P3 | P4挑战后 | P5最终 | 变化 | 驱动因素 |
|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|------|
| CQ1 | 50% | 55% | 60% | 65% | 55% | 58% | +8pp | P4发现竞争数据削弱假设 |
| CQ2 | 40% | 45% | 50% | 52% | 38% | 42% | +2pp | P4压力测试揭示脆弱性 |
| ... | | | | | | | | |
| **加权平均** | | | | | | **47.1%** | | 权重=CQ注意力分 |
```

### 4.2 CQ加权置信度计算

```
CQ-Weighted Confidence = Σ(CQi最终置信度 × CQi注意力权重) / Σ(CQi注意力权重)
```

- 权重来自Phase 0.5的注意力分排序
- 结果是一个0-100%的数字，表示"我们对整体投资论文的信心"
- **解读指导**:
  - ≥60%: 论文较强，关键假设大多可验证
  - 40-60%: 论文有争议，需密切追踪TS
  - ≤40%: 论文脆弱，不确定性高，适合观望

### 4.3 异常信号检测

- **P4大幅下调(≥15pp)**: 看空挑战有效，论文需重估
- **P1→P5单调上升**: 可能存在确认偏误，Phase 4挑战不足
- **CQ间置信度离散>30pp**: 论文内部一致性差，核心假设相互矛盾
