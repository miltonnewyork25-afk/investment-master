## 问题诊断与框架优化

### 问题1：2021-06未发出明确卖出信号

**原因分析**：
- 框架在"扩张期"倾向于"持有"
- 但订单簿已发出警告信号

**优化建议**：
```
新增规则：
当处于扩张期，如果满足以下条件，提前触发"警惕/减仓"：
- 订单簿/船队比 > 20% AND
- 订单簿MoM > 5%（加速上升）
```

### 问题2：2023-06过于保守

**原因分析**：
- 框架要求"边际改善"才买入
- 实际底部往往在绝望中形成

**优化建议**：
```
新增规则（航运特殊）：
当满足以下条件，可以"试探性建仓"：
- 估值得分 ≥ 8（深度低估）AND
- P/B < 0.5x AND
- 行业亏损面 > 50%
调整分：+10分（极端悲观溢价）
```

### 问题3：地缘事件处理

**原因分析**：
- 红海危机等地缘事件会扭曲指标
- 框架需要识别"假信号"

**优化建议**：
```
新增规则：
当SCFI上涨主要由地缘事件驱动：
- 标记为"非需求驱动"
- 先行指标得分打折50%
- 订单簿权重提升至30%
```

## 优化后框架 v1.1

### 航运特殊规则

```yaml
shipping_specific_rules:
  # 规则1：订单簿预警
  orderbook_warning:
    trigger: "orderbook_ratio > 20% AND orderbook_mom > 5%"
    adjustment: "-10分"
    note: "供给过剩早期预警"

  # 规则2：极端悲观溢价
  extreme_pessimism_bonus:
    trigger: "valuation_score >= 8 AND pb_ratio < 0.5 AND industry_loss_rate > 50%"
    adjustment: "+10分"
    note: "底部区域可试探"

  # 规则3：地缘事件过滤
  geopolitical_filter:
    trigger: "scfi_surge driven by geopolitical event"
    adjustment: "leading_indicator_score * 0.5"
    orderbook_weight: "30%"
    note: "识别假信号"

  # 规则4：PE陷阱识别
  pe_trap_detection:
    trigger: "reported_pe < 5 AND cycle_stage == 'overheating'"
    action: "使用正常化PE（按历史平均盈利计算）"
    note: "周期高点的低PE是陷阱"
```

## 框架验证结论

### 与半导体回测对比

| 维度 | 半导体 | 航运 |
|------|--------|------|
| 初版准确率 | 30%（v4.0） | 80%（v1.0） |
| 优化后准确率 | 93%（v5.0） | 100%（v1.1理论） |
| 主要问题 | 顺周期逻辑错误 | 细节规则缺失 |
| 优化方向 | 逆周期逻辑重建 | 增加行业特殊规则 |

### 关键发现

1. **核心逻辑通用**：逆周期投资、背离信号检测在航运同样有效
2. **需要行业适配**：航运有特殊规则（订单簿预警、地缘过滤、PE陷阱）
3. **估值指标补充**：航运需要P/B作为PE的补充

### 过拟合警告

⚠️ **重要提醒**：
- v1.1的"100%准确率"是对历史数据的优化
- 真正的验证需要样本外数据（2026年后）
- 应锁定v1.1参数，用未来数据验证
