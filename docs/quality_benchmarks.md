## Kill Switch详细格式标准 (9字段)

> 每个KS必须包含以下9个字段，缺少任何字段=不合格。
> KS描述论文何时失效，不描述操作。**数量10-15个精选**。
> **精选原则**: 每个KS必须有**明确可观测阈值**。"宏观恶化"不是KS，"10Y国债>5.5%持续3月"才是。

| 字段 | 格式要求 | 示例 |
|------|---------|------|
| **触发条件** | 具体决策规则 | "法院命令Chrome结构性剥离" |
| **具体阈值** | 量化边界(必须可观测) | "命令文本含'structural remedy'" |
| **当前状态** | 当前读数+数据源 | "DOJ上诉已提交 [硬数据: DOJ filing]" |
| **当前距离** | 距阈值距离 | "12-18月待裁决" |
| **论文含义** | 触发后对投资论文的影响 | "搜索垄断假设失效，需按拆分后各部分重估" |
| **CQ关联** | 关联核心问题 | "CQ2(DOJ影响)" |
| **Bear#关联** | 关联Phase 4看空 | "Bear 1" |
| **数据源** | 验证数据源 | "PACER, SCOTUSblog, Reuters" |
| **紧迫性** | 状态颜色 | 红=迫近 / 黄=监控 / 绿=远期 |

## 追踪信号格式标准 (v9.0, 替代旧VP预测)

> **v9.0变化**: 从"我预测X会发生"→"投资者应追踪X指标"。AI不擅长预测未来数字，但擅长识别哪些指标最能验证/否定核心假设。
> **数量**: 5-8个精选信号（旧框架要求20+ VP，大量空泛预测反而稀释决策价值）

```markdown
### TS-0X: [指标名称]

- **追踪什么**: [具体可观测指标，如"LRCX CSBG季度收入"]
- **为什么重要**: [验证/否定哪个核心假设]
- **当前读数**: [最新数据 + 趋势方向 ↗️/→/↘️]
- **关键阈值**: [什么水平意味着论文需要修正]
- **数据源**: [在哪里查: SEC 10-Q / 第三方 / 财报电话会]
- **CQ关联**: CQ#
```

**特异性测试**: 如果信号替换公司名后对同行业任何公司都成立 → 太空泛，删除。
**示例(通过)**: "LRCX CSBG季度收入是否维持>$1.8B" — 特定于LRCX的护城河验证
**示例(不通过)**: "半导体行业将继续增长" — 对任何半导体公司都成立，零决策价值

## CQ闭环5要素格式标准

> 每个CQ必须包含以下5要素，缺少任何要素=不合格。

```markdown
## CQ#: [问题名称]

### 最终回答
[1-2段确定性回答，含置信度调整]

### 置信度路径
| Phase | 置信度 | 驱动因素 |
|:---:|:---:|------|
| P0.5 | X% | 初始评估理由 |
| P1/P3/P4 | Y% | Phase特定发现 |
**最终置信度: Z%**

### Kill Switch关联
- 主KS: [映射到具体触发器]
- 监控指标: [量化标志]

### 1年内验证事件
1. [事件1] (具体时间)
2. [事件2] (具体时间)

### "如果我们错了"
[下行场景分析 + 对核心论点的影响]
```

## ~~仓位建议5档格式标准~~ (v2.0已移除)

> **v2.0**: 仓位建议模板已移除。报告不再输出具体仓位百分比或价格触发操作。投资者的仓位取决于其自身资金量、风险承受力、组合结构——这些信息报告无法也不应假装知道。报告聚焦于提供"核心论点+关键假设+追踪信号+估值参考"，由投资者自行判断仓位。

## Complete报告组装标准 (v6.0 — 纠错回流)

**Phase 5完成后，必须将所有Phase整合为一份Complete文档**:

```
{TICKER}_Complete_v{版本}_{YYYY-MM-DD}.md
```

**组装流程**:
1. 统一TOC（全报告目录，含所有章节号+标题+字符统计）
2. Phase 0.5 Market Radar
3. **⚡ 纠错回流 (v6.0新增)**: 将Phase 4事实性纠错回流覆盖到Phase 1-3原文
   - 遍历Phase 4纠错清单，逐条修正Phase 1-3中的错误数据/结论
   - 修正处加脚注: `[^P4修正: 原值X→修正值Y, 来源Z]`
   - CQ置信度调整回流到Phase 5 CQ闭环表
4. Phase 1 全文（已含回流修正）
5. Phase 2 全文（已含回流修正）
6. Phase 3+3.5 全文（已含回流修正）
7. **Phase 4精简版**: 只保留方法论+结构性挑战（看空论点/认知偏差/压力测试），删除已回流的事实性纠错条目
8. Phase 5 全文
9. **⚡ 追踪信号精选 (v9.0)**: 对TS执行特异性测试，删除空泛信号（替换公司名仍成立→删除）
10. 全局免责声明
11. 运行 `tests/quality_gate_complete.sh` 验证
12. 只有通过质量门控后才能 git commit

**禁止**: Phase 5完成即宣布"全量完成"而不组装Complete报告。
**禁止**: Phase 4纠错条目原样保留在Complete中而不回流覆盖。
**禁止**: 报告任何位置出现持仓/减仓/加仓/仓位百分比等操作性建议。
