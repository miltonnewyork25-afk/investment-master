# 看空分析等权重方法论 v1.0

> 看空分析不是走形式。最少占总篇幅30%，必须包含具体触发条件和概率。

---

## 设计哲学

**问题**: 看空分析占报告<2%篇幅，沦为"尽管存在风险，但我们仍然看好"的装饰
**方案**: 结构性保证看空分析获得等权重的资源和深度

---

## 等权重规则

### 篇幅要求

| 报告层级 | 看空最低篇幅占比 | 计算方式 |
|---------|:---:|------|
| Tier 3 | ≥30% | (看空相关字符数 / 总字符数) × 100% |
| Tier 2 | ≥25% | 同上 |

**"看空相关"包括**: Bear Case 场景、风险分析、反证挑战、Kill Switch、压力测试、竞争威胁

### 结构要求

每个看空论点必须包含四要素:

```markdown
### 看空论点 #N: [标题]

**触发条件**: [具体可观察的事件/指标变化]
**概率评估**: [X%] [合理推断: 推理链]
**影响量化**: 若触发，对估值的影响为 [X%下行]
**时间窗口**: [何时可验证/何时最危险]
**当前信号**: [目前是否有早期迹象]
```

### 最低看空论点数量

| 报告层级 | 最少看空论点 | 深度要求 |
|---------|:---:|------|
| Tier 3 | ≥8个 | 每个≥500字符，含四要素 |
| Tier 2 | ≥5个 | 每个≥300字符 |

---

## "钢人论证"要求

对看空论点执行钢人论证 (Steelman)：不是找最弱的空头观点来反驳，而是找最强的空头论证来认真回应。

**钢人论证检验**:
1. 这个看空论点是否由聪明的空头投资者提出？
2. 我们的反驳是否足够具体（有数据），还是仅凭"长期来看会好"？
3. 如果我们只看到看空证据，会不会得出相反的投资结论？

---

## 与 Phase 结构的集成

| Phase | 看空内容 |
|-------|---------|
| Phase 1 | 识别潜在看空论点（不需深入） |
| Phase 2 | 量化看空场景对估值的影响（Bear Case场景） |
| Phase 3 | 竞争威胁 + 护城河侵蚀风险的深入分析 |
| Phase 4 | **核心**: 钢人论证 + 反证挑战 + 全量看空分析 |
| Phase 5 | Kill Switch 整合所有看空触发条件 |

## 黑天鹅概率加权表 (v2.0新增)

> **来源**: AMD报告Phase 4识别了5个黑天鹅事件，各有独立概率和影响量化。
> 尾部风险不是"列出来就行"——必须量化其对估值的加权影响。

### 模板

```markdown
### 黑天鹅概率加权分析

| # | 事件 | 独立概率 | 影响幅度 | 加权损失 | 时间窗口 | 早期信号 |
|---|------|:-------:|:-------:|:-------:|:-------:|---------|
| BS-1 | [具体事件] | X% | -Y% | -Z% | N年内 | [可观测信号] |
| BS-2 | ... | | | | | |
| **合计** | | | | **-W%** | | |

加权损失 = 独立概率 × 影响幅度
合计加权损失 = 估值应扣除的尾部风险折价
```

### 执行规则

1. **最少3个黑天鹅事件**，Tier 3报告必须
2. **概率来源**: Polymarket(优先) / 历史类比频率 / 分析师共识
3. **影响量化**: 对股价/估值的百分比影响，不是"很大""很小"
4. **早期信号**: 每个黑天鹅必须有≥1个可监控的早期预警指标
5. **与KS关联**: 每个黑天鹅至少关联1个Kill Switch

### 示例 (AMD)

| # | 事件 | 概率 | 影响 | 加权 | 窗口 | 信号 |
|---|------|:---:|:---:|:---:|:---:|------|
| BS-1 | 台海冲突中断TSMC代工 | 5% | -60% | -3.0% | 3年 | 军事演习频率+TSMC Arizona产能 |
| BS-2 | MI系列AI芯片重大缺陷 | 8% | -35% | -2.8% | 1年 | 客户退货率+良率报告 |
| BS-3 | 美中科技脱钩升级 | 12% | -25% | -3.0% | 2年 | 出口管制扩展范围 |
| **合计** | | | | **-8.8%** | | |
