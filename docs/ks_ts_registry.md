## 联动规则 (Linkage Rules)

### N-out-of-M规则
```yaml
生态围剿模式:
  触发条件: "KS-REG-01 AND KS-COMP-01"
  影响: "论文强度从A级降至C级"

系统性风险:
  触发条件: "2-out-of-3(KS-FIN-01, KS-FIN-02, KS-MACRO-01)"
  影响: "财务安全假设失效"

技术护城河失效:
  触发条件: "KS-TECH-01 OR KS-TECH-02"
  影响: "重新评估技术溢价"
```

### TS校准矩阵
| TS触发数量 | 0-1个L2/L3 | 2-3个L2/L3 | 4+个L2/L3 |
|------------|------------|------------|-----------|
| CQ置信度校准 | 维持 | -5到-10pp | -10到-20pp |
| 论文强度 | 维持 | 中度质疑 | 重新评估 |

## 使用规范

### 强制要求
1. **编码标准化**: 所有KS/TS必须使用注册表编码，禁止重复定义
2. **数据源绑定**: 每个KS/TS必须指定具体数据源和更新频率
3. **特异性测试**: 所有KS/TS必须通过Swap Test(替换公司名后仍特异)
4. **联动检查**: 报告完成时必须检查相关联动规则是否触发

### 更新机制
1. **季度更新**: 根据新报告经验更新注册表
2. **阈值校准**: 基于历史触发准确性调整阈值
3. **新增类别**: 发现新风险模式时扩展注册表

### 质量门控集成
```bash
# quality_gate_complete.sh新增检查
grep -E "KS-[A-Z]+-[0-9]+" $REPORT | sort | uniq -c | awk '$1>1{print "重复KS定义:"$2}' && exit 1
grep -E "TS-[A-Z]+-[0-9]+" $REPORT | sort | uniq -c | awk '$1>1{print "重复TS定义:"$2}' && exit 1
```
