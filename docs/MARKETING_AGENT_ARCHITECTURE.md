## 一、项目背景与定位

### 1.1 整体项目："投资大师"
**投资大师**是一个AGI级买方投资研究系统，由多个专业Agent协作完成任务。

**系统架构**:
```
┌──────────────────────────────────────────────────────────────┐
│                    投资大师 主控制器                           │
│                  (协调所有Agent)                              │
└──────────────────────────┬───────────────────────────────────┘
                           │
       ┌───────────────────┼───────────────────┐
       │                   │                   │
       ▼                   ▼                   ▼
┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│  Company     │   │  Marketing   │   │ Psychology   │
│  Research    │   │  Agent       │   │ Agent        │
│  Agent       │   │  ★本文档★    │   │              │
└──────────────┘   └──────────────┘   └──────────────┘
       │                   │                   │
       ▼                   ▼                   ▼
 产出投资分析报告    将分析转化为      提供行为金融学
 (≥20,000字)        多平台内容         心理洞察
```

### 1.2 Marketing Agent的使命

**核心使命**: 将「投资分析窗口」产出的深度研究报告，转化为5个自媒体平台的高传播内容。

**工作流**:
```
投资分析报告 → Marketing Agent → 5平台适配内容
   (输入)          (处理)            (输出)
                    │
                    ├── X/Twitter: Thread
                    ├── YouTube: 视频脚本
                    ├── Reddit: DD帖子
                    ├── 雪球: 长文
                    └── 小红书: 图文
```

**关键约束**:
- 内容来源固定：另一个对话窗口的投资分析输出
- 不生成原创投资观点，只转化已有分析
- 需同时满足传播效力与合规安全

## 三、Part 0：三条铁律（详细规范）

> ⚠️ **最高优先级约束** — 所有内容必须首先通过铁律检查

### 3.1 铁律1：去身份化 + 合规安全

**目的**: 防止平台因推荐股票/诱导投资封禁账号

**禁止出现**:
| 类别 | 禁止表达示例 |
|------|-------------|
| AI身份 | "AI分析"、"我们的AI"、"机器学习模型显示" |
| 个人身份 | "我"、"我们"、"我的框架"、"我认为" |
| 投资建议 | "看多/看空"、"推荐买入/卖出"、"应该买/卖" |
| 诱导性 | "赶紧上车"、"错过就没了"、"稳赚" |

**允许使用**:
| 类别 | 允许表达示例 |
|------|-------------|
| 客观陈述 | "数据显示"、"历史规律表明"、"指标指向" |
| 中性描述 | "周期评分框架"、"供应链分析" |
| 疑问引导 | "这是否意味着...？"、"值得关注的是..." |

### 3.2 铁律2：忠实原文

**目的**: 确保标题/内容不超出源分析范围

**规则**:
1. 所有标题/内容必须基于源分析报告的实际结论
2. 数据必须与原文一致，不允许夸大或缩小
3. 不允许编造未在原文出现的观点
4. 引用必须可追溯到具体来源

**检验方法**: 删除所有修饰词后，核心信息是否与原文完全一致？

### 3.3 铁律3：洞察驱动 ≠ 炒作驱动

**目的**: 传播力来自认知价值，不是夸张恐吓

| 类型 | 允许/禁止 | 示例 |
|------|----------|------|
| 数据悖论 | ✅ 允许 | "毛利率创新高，但股价在底部——数据错了还是市场错了？" |
| 历史规律 | ✅ 允许 | "过去5次这个信号出现后..." |
| 开放问题 | ✅ 允许 | "如果X发生，Y会怎样？" |
| 夸张断言 | ❌ 禁止 | "即将暴涨"、"史诗级机会" |
| 情绪煽动 | ❌ 禁止 | "不看后悔"、"惊天发现" |
| 虚假紧迫 | ❌ 禁止 | "最后机会"、"窗口即将关闭" |

### 3.4 铁律执行协议

```
内容生产流程:
1. 生成初稿
2. 铁律检查（逐条扫描）
3. 违反 → 标记并重写
4. 全部通过 → 进入后续流程（评分等）

Kill Switch: 任何铁律违反 = 内容不可发布
```

## 四、核心引擎详解

### 4.1 超级选题引擎 (Topic Selection Engine)

**文件**: `marketing/strategy/topic-selection-engine.md`

**五维评分 (5D Score)**:
```
Score = V×0.25 + G×0.25 + T×0.20 + U×0.15 + D×0.15 + 2阶加成(0-3)

V = Virality (传播力)     : 内容的病毒传播潜力
G = Growth (成长贡献)     : 对公司/品牌长期增长的贡献
T = Timing (时机)         : 与当前事件/情绪的匹配度
U = Uniqueness (独特性)   : 市场上是否已有类似内容
D = Debate (辩论潜力)     : 能否引发评论区讨论

阈值:
≥8.5 → 立即执行
7.0-8.4 → 本周发布
5.5-6.9 → 储备
<5.5 → 淘汰
```

**10大超级选题角度**:
| # | 角度 | 典型得分 | 示例 |
|---|------|---------|------|
| 1 | 对抗叙事 | D=10, V=9 | "[权威]说X, 数据说Y" |
| 2 | 数据颠覆 | V=8, U=9 | "你以为X, 数据说Y" |
| 3 | 预测验证 | G=9 | "X天前预测Y, 结果Z" |
| 4 | 教育解密 | G=8 | "底层原理是什么?" |
| 5 | 透明自曝 | G=9 | "我错了/这是完整记录" |
| 6 | 独家数据 | G=10, U=10 | "这个只有我们有" |
| 7 | 时间机器 | V=8 | "如果X年前按策略做..." |
| 8 | 隐藏信号 | V=9, U=8 | "所有人看X, 真正信号是Y" |
| 9 | 排行榜/矩阵 | V=8 | "Top 10/对比矩阵" |
| 10 | 场景模拟 | V=7, G=8 | "如果X发生? 3种结果" |

**选题配比(飞轮最优)**:
```
40% 常青教育 → 复利资产, 搜索长尾
35% 事件驱动 → 即时传播, K-Factor加速
15% 争议辩论 → 算法加权, 品牌差异化
10% 实验探索 → 新增长向量
```

### 4.2 超级标题引擎 (Headline Engine)

**文件**: `marketing/strategy/super-headline-engine.md`
**版本**: v1.2

**7大心理驱动原理**:
| # | 原理 | 效果 | 来源 |
|---|------|------|------|
| 1 | 好奇心缺口 | 3-5x CTR | Loewenstein |
| 2 | 惊讶/打破模式 | 2-3x记忆 | Heath |
| 3 | 具体数字(奇数>偶数) | +20% CTR | 研究统计 |
| 4 | 损失厌恶 | 2x动力 | Kahneman |
| 5 | 高唤醒情绪 | 70%病毒内容 | Berger |
| 6 | 认知流畅 | +信任度 | 心理学 |
| 7 | 身份指认 | +关注度 | 社会认同 |

**Dan Koe RAE框架**:
```
标题效果 = R × A × E
R = Relevance (与读者生活相关)
A = Awareness (认知水平匹配)
E = Effort (低成本高回报感知)
```

**Gene Schwartz 10大标题强化术**:
量化承诺 | 量化速度 | 对比强化 | 隐喻转化 | 感官化 |
案例证明 | 戏剧化 | 悖论表达 | 消除限制 | 身份关联

**八维评分 (8D Score)**:
```
Score = H×0.20 + E×0.15 + A×0.15 + D×0.15 + L×0.10 + I×0.10 + N×0.10 + S×0.05

H = Hook (冲击力)      : 前3秒吸引注意力
E = Emotion (情绪)     : 唤醒高唤醒情绪
A = Accuracy (准确)    : 与内容匹配度
D = Debate (辩论)      : 引发讨论潜力
L = Length (长度)      : 符合平台最优长度
I = Identity (身份)    : 与目标受众身份共鸣
N = Numbers (数字)     : 具体数字使用
S = Shareability (可分享) : 分享后提升分享者形象

阈值:
≥8.5 → 使用
7.0-8.4 → 微调后使用
<5.5 → 重写
```

**5平台字数规范**:
| 平台 | 硬限制 | 最优范围 | 特殊规则 |
|------|--------|---------|---------|
| X/Twitter | 280字符 | 47-73字符 | Hook在首句 |
| YouTube | 100字符 | 60-70字符 | IMPACT格式 |
| Reddit | 300字符 | 80-150字符 | 必须有[DD]等标签 |
| 雪球 | 无限制 | 30-60中文字 | 必须含$股票代码 |
| 小红书 | 20中文字 | 15-20中文字 | 前10字=60%权重 |

### 4.3 合成受众模拟 (Synthetic Audience)

**协议**:
```
1. 定义7类persona × 30个变体 = 210+ 合成读者
2. 对3个内容变体模拟反应:
   - 阅读率
   - 情绪反应
   - 分享意愿
   - 转化意愿
3. 选择加权分享意愿最高的变体
4. 收集合成反对意见 → 在内容中预埋回应
```

**7类Persona**:
| # | 类型 | 特征 | 关注点 |
|---|------|------|--------|
| 1 | 新手投资者 | <1年经验 | 简单易懂、可操作 |
| 2 | 主动投资者 | 1-5年经验 | 独特观点、数据支撑 |
| 3 | 专业投资者 | >5年或从业 | 深度分析、方法论 |
| 4 | 怀疑者 | 对AI/新方法持疑 | 证据、透明度 |
| 5 | 技术爱好者 | 关注技术/AI | 创新性、技术细节 |
| 6 | 价值投资者 | 偏好基本面 | 估值、长期逻辑 |
| 7 | 趋势交易者 | 偏好技术面 | 时机、信号 |

### 4.4 事件驱动营销系统 (Newsjacking)

**核心原理**: 可预测事件 = 零成本注意力杠杆

**事件内容生产协议**:
```
事件前7天: 识别事件群集（同期多个相关事件）
事件前3天: 生产预览内容(Thread+Newsletter+Video)
事件前1天: 预热推文(悬念建设)
事件当天: 即时反应(使用预生成双模板 A/B)
事件后1天: 深度分析Newsletter
事件后7天: 总结+框架更新展示
```

**永赢策略(No-Lose)双模板**:
| 预测结果 | 内容方向 | 触发偏误 |
|----------|---------|---------|
| 正确 | "框架验证+信号回顾" | 权威+从众 |
| 错误 | "透明反思+模型更新" | 互惠+一致性 |

## 六、平台适配规则

### 6.1 X/Twitter

```yaml
format: Thread (多推文串联)
hook_position: 首句
optimal_length: 47-73字符
algorithm_weights:
  - reply: 75x (最高权重)
  - retweet: 20x
  - like: 1x
  - premium_4x: 付费用户4倍曝光
  - video_10x: 视频10倍engagement
best_angles: 对抗叙事, 数据颠覆
posting_time: 美东8-10AM或6-9PM
```

### 6.2 YouTube

```yaml
format: 视频脚本 (60s Shorts 或 10-15min 长视频)
title_format: IMPACT (Interest + Mystery + Promise + Appeal + Curiosity + Tension)
optimal_length: 60-70字符
thumbnail: 高对比+人脸+大字
algorithm_focus: Watch Time > Views
best_angles: 教育解密, 数据颠覆
```

### 6.3 Reddit

```yaml
format: DD帖子 (Due Diligence)
required_elements:
  - [DD] 或 [Research] 标签
  - 仓位披露 (Position: X shares @ $Y)
  - TLDR at top
  - 数据来源标注
optimal_length: 900-1500字
subreddits: r/stocks, r/investing, r/ValueInvesting, r/wallstreetbets
best_angles: 独家数据, 教育解密, 对抗叙事
```

### 6.4 雪球

```yaml
format: 长文分析
required_elements:
  - $股票代码 (可点击)
  - 话题标签 #行业分析# 等
  - 实盘数据披露
optimal_length: 30-60中文字(标题), 2000-5000字(正文)
best_angles: 个股深度, 预测验证, 行业研究
community_rules: 禁止纯推荐, 需有分析过程
```

### 6.5 小红书

```yaml
format: 图文笔记
title_rules:
  - 硬限制: 20中文字
  - 最优: 15-20中文字
  - 前10字 = 60%权重 (搜索截断)
content_style: 场景化, 故事化, 口语化
visual: 必须有图, 最好是数据可视化
best_angles: 教育科普, 结果展示, 投资入门
algorithm_focus: CES评分 (Completion + Engagement + Sharing)
```

## 七、VIRAL-SYSTEM 执行循环（11步）

```
V - Value（价值锚定）     → 定位理论 + JTBD 确定核心价值
I - Insight（洞察挖掘）   → 行为经济学 + 用户研究识别杠杆点
R - Reach（触达策略）     → 扩散理论 + 边际分析选择渠道
A - Amplify（放大效应）   → 网络效应 + STEPPS 设计传播机制
L - Loop（闭环优化）      → AARRR + 边际递减检测

S - Story（故事包装）     → 框架效应 + 叙事经济学包装信息
Y - Yield（产出度量）     → AARRR 漏斗指标度量
S - Scale（规模扩张）     → 飞轮效应 + S曲线识别扩张时机
T - Test（测试迭代）      → 边际分析 + A/B 测试验证假设
E - Evolve（进化升级）    → 淘汰饱和方法，引入新杠杆
M - Multiply（倍增分发）  → 网络效应 + K-Factor > 1 设计
```
