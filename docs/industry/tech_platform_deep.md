## 核心差异 vs 其他行业

科技平台与制造业/消费品/金融公司的根本区别:

- **多边市场与网络效应** — 平台价值随用户增长非线性增加
- **数字商品边际成本趋近于零** — 毛利率结构性高于实体行业
- **数据作为核心资产** — 不仅是财务数据，用户行为数据本身即护城河
- **监管风险来自反垄断** — 而非传统行业监管
- **AI既是机遇也是自我颠覆风险** — 可能侵蚀自身核心产品

### TP01: 平台经济学 (Platform Economics)

**适用**: 全部科技平台
**核心问题**: 网络效应有多强？用户锁定程度如何？

#### 1. 网络效应类型与强度

| 类型 | 定义 | 典型案例 |
|------|------|----------|
| 直接网络效应 | 用户吸引用户 | 即时通讯、社交网络 |
| 间接/跨边网络效应 | 用户吸引开发者/商家 | 应用商店、电商平台 |
| 数据网络效应 | 更多数据 → 更好产品 | 搜索、推荐系统 |

量化指标: 网络效应系数 = Δ用户价值 / Δ用户数量

#### 2. 多边市场健康度

平台各边: 用户 / 广告主 / 开发者 / 内容创作者 / 商家

每一边需分析:
- 增长率
- 流失率
- ARPU（每用户平均收入）
- 满意度指标

跨边弹性: 一边的增长在多大程度上驱动另一边增长？

#### 3. 锁定系数 (Switching Cost Index)

| 锁定维度 | 评估内容 | 评分 |
|----------|----------|------|
| 数据锁定 | 用户数据被平台困住的程度 | 0-10 |
| 社交图谱锁定 | 用户能否带走社交关系 | 0-10 |
| 内容锁定 | UGC内容是否可迁移 | 0-10 |
| 工作流锁定 | 平台嵌入日常工作/生活的深度 | 0-10 |

总分: 0-10（10 = 几乎不可能迁移）

#### 4. 平台生命周期定位

启动 → 增长 → 主导 → 商品化 → 再造

需判断: 每个主要平台产品处于生命周期的哪个阶段？

**输出工件**: 网络效应强度表 + 锁定系数矩阵 + 生命周期定位图

### TP02: 广告经济学 (Advertising Economics)

**适用**: GOOG, META, AMZN（广告依赖型公司）
**核心问题**: 广告收入增长是否可持续？是否存在广告负载天花板风险？

#### 1. ARPU趋势

按地区分解:
- 北美 / 欧洲 / 亚太 / 其他地区

按产品分解:
- 搜索 / YouTube / Instagram / Reels / 其他

要求: 5年趋势 + 增长率分解

#### 2. 广告负载 (Ad Load)

- 当前广告密度（每次用户会话的广告数量）
- 提升空间 vs 用户体验恶化风险
- 与同行对比

#### 3. CPM/CPC趋势

- 千次展示成本 / 单次点击成本趋势
- 广告主ROI指标
- AI对广告定位效率的提升 → 是否推高CPM？

#### 4. 广告格式演进

- 短视频广告（Reels/Shorts）变现差距
- AI生成广告创意的影响
- 搜索广告 vs AI回答整合（Google AI Overview对搜索广告的冲击）

#### 5. 广告主集中度

- Top 10广告主收入占比
- 中小企业 vs 大型企业广告主比例
- 行业分散度

**输出工件**: ARPU趋势表（5年 × 地区） + 广告负载对比 + 格式演进分析

### TP03: 云基础设施 (Cloud Infrastructure)

**适用**: GOOG(GCP), AMZN(AWS), MSFT(Azure)
**核心问题**: 云市场份额走向如何？GenAI贡献多少？

#### 1. 市场份额与增速

- 季度收入 + 同比增长率
- 市场份额 vs AWS/Azure/GCP
- 份额趋势方向（增长/流失）

#### 2. GenAI服务贡献

- GenAI专项收入（如已披露）
- AI模型托管 / 推理收入
- AI工作负载 vs 传统云的增速对比

#### 3. 利润率演进

- 云业务经营利润率趋势
- 盈利路径（尤其对GCP）
- AI基础设施投资对近期利润率的冲击

#### 4. 客户结构

- 企业客户 vs 初创公司比例
- 年合同价值≥100万美元的客户数
- 多云策略采用的影响

#### 5. 竞争威胁

- 专业AI云（CoreWeave, Lambda）
- 本地AI推理趋势
- 主权云需求

**输出工件**: 云市场份额表 + GenAI贡献分析 + 利润率走势

### TP04: 监管与反垄断 (Regulatory & Antitrust)

**适用**: 全部科技平台
**核心问题**: 监管行动的概率和影响有多大？

#### 1. 全球监管矩阵

| 司法管辖区 | 进行中案件 | 潜在影响 | 时间线 |
|-----------|-----------|----------|--------|
| 美国（DOJ, FTC） | — | — | — |
| 欧盟（DMA, DSA, GDPR） | — | — | — |
| 中国（数据安全、算法监管） | — | — | — |
| 其他（印度、日本、韩国、澳大利亚） | — | — | — |

#### 2. 反垄断风险量化

- 拆分/结构性救济概率
- 行为性救济概率
- 罚款概率 + 预估金额
- 每种情景下的受影响收入

#### 3. 合规成本

- 当前合规支出
- 新法规带来的预计增长
- 内容审核成本（Meta, Google）

#### 4. AI监管前沿

- 欧盟AI法案影响
- 美国AI行政令影响
- 算法透明度要求

#### 5. 预测市场数据

- 搜索Polymarket/Kalshi获取监管结果概率
- 映射到财务影响

**监管专用Kill Switch**:

| 触发条件 | 动作 |
|----------|------|
| 反垄断拆分概率 > 35% | 立即清仓 |
| 核心业务禁止概率 > 30% | 立即清仓 |
| 监管限制概率 > 25% | 减仓50% |

**输出工件**: 监管风险矩阵 + 概率×影响表 + 时间线

### TP05: 开发者生态 (Developer Ecosystem)

**适用**: 全部科技平台
**核心问题**: 开发者生态健康度如何？平台是否仍在吸引建设者？

#### 1. 开发者数量与增速

- 注册开发者 / 活跃开发者数量
- 同比增长率
- 开发者满意度调查结果

#### 2. API经济

- API调用量趋势
- 开发者服务收入
- 第三方应用/服务数量

#### 3. 开源战略

- 开源项目贡献（Meta的LLaMA, Google的TensorFlow等）
- 社区采用指标（GitHub stars, 下载量, forks）
- 开源投资的战略价值

#### 4. 生态系统价值

- 生态系统总GMV或交易额
- 开发者分成比例
- 平台抽成率趋势

#### 5. AI开发者工具

- AI API产品（GPT API, Claude API, Gemini API, Bedrock）
- 开发者对AI工具的采用度
- 在AI开发者工具领域的竞争定位

**输出工件**: 开发者生态健康记分卡 + API经济指标 + 开源影响力

### TP06: 数据护城河 (Data Moat)

**适用**: 全部科技平台
**核心问题**: 公司的数据优势有多独特、多可防御？

#### 1. 数据独占性

- 该公司拥有哪些竞争对手**完全没有**的数据？
  - 搜索意图数据（Google）
  - 社交图谱（Meta）
  - 购买行为（Amazon）
- 数据刷新速率: 数据多快会过时？

#### 2. 数据飞轮效率

- 更多用户 → 更多数据 → 更好产品 → 更多用户
- 量化: 多10%的数据是否真的能提升产品X%？
- 收益递减拐点: 公司是否已过数据价值高原期？

#### 3. 隐私法规冲击

- GDPR/CCPA对数据采集的影响
- Apple ATT对跨应用追踪的影响（Meta受冲击最大）
- Cookie废弃的影响（Google受冲击最大）
- 数据本地化要求

#### 4. AI与数据的双向关系

- AI使数据更有价值（从相同数据中提取更好洞察）
- **但**AI也使竞争对手能生成合成数据
- 净影响: 数据护城河是增强了还是削弱了？

#### 5. 数据变现能力

- 每数据点收入（近似值）
- 数据变现渠道: 广告 / 云服务 / 授权
- 未开发的数据资产

**输出工件**: 数据独占性矩阵 + 飞轮效率分析 + 隐私法规冲击评估

## 科技平台专用Kill Switch

| Kill Switch | 触发条件 | 严重度 |
|-------------|----------|--------|
| 用户流失 | DAU/MAU连续2个季度下降 >5% | Hard |
| 广告收入崩塌 | 广告收入连续2个季度增长 <0% | Hard |
| 云份额流失 | 云市场份额年降 >2个百分点 | Soft |
| 反垄断拆分 | 拆分概率 >35% | Hard |
| 开发者流失 | 活跃开发者数年降 >15% | Soft |
| 数据护城河侵蚀 | 关键隐私法规通过 | Soft |
| AI自我颠覆 | 核心产品被AI替代方案蚕食 | Hard |
| 监管禁令 | 核心产品在主要市场被禁 | Hard |
