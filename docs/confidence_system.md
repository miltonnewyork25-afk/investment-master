# 数据可信度系统 v3.0 (DM锚定+脚本验证)

> **v3.0核心变化**: 内联标注从报告正文移除 → 改为DM锚定+自动化脚本验证+文末折叠审计摘要。
> 报告正文=纯净叙事，零标注噪音。AI写作纪律由外部验证保障，不再依赖自报自审。
> **v2.0→v3.0迁移**: 旧报告不改。新报告按v3.0执行。

---

## 设计哲学

**v1.0**: `[置信度: 85%]` — 伪精度装饰
**v2.0**: 内联三层标注 `[硬数据:]` `[合理推断:]` `[主观判断:]` — 有效但严重影响阅读体验
**v3.0**: DM锚定+脚本验证 — 投资者看干净叙事，AI纪律由外部系统保障

### 为什么移除内联标注

| 问题 | 说明 |
|------|------|
| 阅读体验差 | AMD报告45.1标注/万字符，正文变成"穿越注释丛林" |
| 自报自审不可靠 | AI写"[硬数据: X]"不代表X真实存在——标注本身可以被编造 |
| 投资者不看 | 99%读者从不验证标注来源，但100%读者被噪音干扰 |
| 密度竞赛适得其反 | 追求高标注密度导致Agent在显而易见的计算上也加标注 |

## 三层数据分类 (保留概念，移出正文)

三层分类**仍然是数据管理的基础**，但仅体现在DM和审计层，不体现在报告正文：

### H: 硬数据 (Hard Data)
- 直接来自一级/二级来源的可验证事实
- 来源: SEC文件 / 财报 / 官方披露 / 权威数据库 / MCP工具
- DM标注: 锚点必须标记 `type: H`

### R: 合理推断 (Reasoned Inference)
- 基于硬数据的逻辑推导，推理链可审查
- DM标注: 锚点标记 `type: R` + `chain: [推理链简述]`
- **v3.0新增: 证伪条件** — 关键推断必须附带"什么证据能推翻这个推断"

### S: 主观判断 (Subjective Assessment)
- 分析师观点/定性评估，合理人可能不同意
- DM标注: 锚点标记 `type: S` + `basis: [判断依据]`

## v3.0 工作流程

```
Phase 0: DM收集数据 → 每个锚点标记类型(H/R/S)+来源+日期
    ↓
Phase 1-5: 写干净叙事 → 每个数字必须来自DM锚点(铁律14不变)
    ↓
Phase 5.5: 自动验证 → verify_data_sources.sh 交叉核对
    ↓
输出: 干净报告 + 文末折叠审计摘要
```

### Step 1: DM锚点强化 (Phase 0)

shared_context.md 中每个数据锚点增加元数据：

```markdown
### DM-FIN-001 v2.0
- **值**: FY2025 Revenue $27.42B
- **类型**: H
- **来源**: MCP fmp_data income FY2025
- **日期**: 2026-02-11
- **用于**: Phase 1 §1.1, Phase 2 §2.3, Phase 5 §估值
```

关键推断锚点增加证伪条件：

```markdown
### DM-INF-003 v1.0
- **值**: EPYC市场份额FY2027E达45%
- **类型**: R
- **推理链**: 当前41%(Mercury) + 年增2-3pp(Zen5/Turin) + Intel 18A延迟
- **证伪条件**: Intel 18A良率>80%且Granite Rapids下一代提前6月+发布
- **用于**: Phase 2 §Reverse DCF, Phase 5 §承重墙
```

### Step 2: 干净叙事 (Phase 1-5)

报告正文**不包含任何标注**。数字直接写入叙事：

```markdown
<!-- v3.0: 干净叙事 -->
AMD FY2025商誉$25.1B，占总资产$76.9B的32.7%。加上无形资产$16.7B，
无形资产合计$41.8B，占总资产54.4%。有形权益仅$21.2B。

如果仅看有形资产，P/B从5.54x跳升到约16.4x（$348B / $21.2B）。
商誉减值测试的触发条件是分部公允价值低于其账面价值——如果Embedded分部
持续低迷，$25.1B商誉中归属于Xilinx的部分面临减值风险。
```

### Step 3: 章节折叠源表 (可选，推荐)

每个主要章节(##级)末尾可添加折叠审计区：

```html
<details>
<summary>📋 数据源 (§1.4 | H:5 R:2)</summary>

| 数据点 | 类型 | DM锚点 | 来源 |
|--------|:----:|--------|------|
| 商誉$25.1B | H | DM-FIN-012 | MCP balance FY2025 |
| 总资产$76.9B | H | DM-FIN-012 | MCP balance FY2025 |
| 有形权益$21.2B | H | DM-FIN-013 | MCP key-metrics |
| 有形P/B 16.4x | R | — | $348B / $21.2B |
| 减值风险 | R | DM-INF-007 | GAAP减值规则推断 |

</details>
```

### Step 4: 文末审计摘要 (必须)

报告最后一节(免责声明前)放置全局审计摘要：

```html
<details>
<summary>📊 数据审计摘要 | DM覆盖率 94% | 锚点 47个 | 更新 2026-02-11</summary>

**数据来源**: shared_context.md v2.0 (47个数据锚点)
**获取工具**: FMP API / MCP baggers_summary / SEC 10-K / WebSearch
**类型分布**: H 28个(60%) | R 14个(30%) | S 5个(10%)
**关键推断证伪条件**: 见DM-INF系列锚点
**完整审计**: reports/{TICKER}/data/audit_trail.md

</details>
```

### Step 5: 自动化验证 (Phase 5.5)

```bash
bash tests/verify_data_sources.sh \
  reports/{TICKER}/{TICKER}_Complete_v{X}_{date}.md \
  reports/{TICKER}/data/shared_context.md
```

脚本检查三项：
1. **DM覆盖率**: 报告关键数字在DM中有锚点的比例 (目标≥90%)
2. **数值一致性**: 报告数字 vs DM锚点数字匹配率 (目标≥95%)
3. **DM新鲜度**: 所有锚点更新日期≤30天 (目标100%)

## 质量门控适配 (CG8/CG9 v3.0)

| 门控 | v2.0定义 | v3.0定义 |
|------|---------|---------|
| **CG8** | 内联标注密度≥25/万 | **审计覆盖**: 文末审计摘要存在 + 折叠源表≥5个章节 |
| **CG9** | 硬数据占比≥36% | **DM覆盖声明**: 审计摘要中DM覆盖率≥90% + H类型≥50% |

**过渡期兼容**: 旧格式报告(v2.0内联标注)仍按旧CG8/CG9检查。新报告按v3.0检查。脚本通过检测文末 `数据审计摘要` 来自动选择检查模式。

## 推断证伪条件 (v3.0新增)

> **来源**: AMD报告学习——关键推断缺少"什么能推翻它"的显式声明

对每个影响估值结论的关键推断(DM-INF系列)，必须定义：

```markdown
### 证伪条件模板
- **推断**: [推断内容]
- **推理链**: [A → B → C]
- **证伪条件**: [什么可观测证据能推翻这个推断]
- **证伪概率**: [低/中/高 — 基于当前信号]
- **影响**: [若推翻，对估值/论文的影响]
```

**要求**: Complete报告中≥5个关键推断有证伪条件(在DM中定义，审计摘要中汇总数量)

## 与其他机制的集成

| 机制 | v2.0关系 | v3.0关系 |
|------|---------|---------|
| Data Master | 数据来源之一 | **核心锚定层** — 所有数字必须可追溯到DM |
| Fast Gate G2 | 检查内联标注密度 | **检查审计覆盖率** — 审计摘要存在+折叠源表数量 |
| 反幻觉协议 | 5条禁令+自检 | **外部验证** — verify_data_sources.sh |
| Phase 4 R-G | 12项一致性检查 | 不变 — 仍需检查数字一致性 |
| Agent Prompt | 注入标注格式要求 | **注入DM引用要求** — "每个数字必须来自DM" |
