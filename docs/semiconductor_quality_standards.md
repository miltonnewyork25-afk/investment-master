## v20.0深度评分系统（L1-L6）

| 级别 | 描述 | 字数/模块 | v20.0要求 |
|------|------|----------|----------|
| L1 | 表面描述 | <500 | 基础事实陈述 |
| L2 | 数据+趋势 | 500-1000 | 数据源标注+可信度 |
| L3 | 机制分析（为什么） | 1000-2000 | 因果关系论证 |
| L4 | 反直觉洞察+预测 | 2000-4000 | 预测市场概率验证 |
| L5 | 原创框架+可验证 | 4000-8000 | PPDA背离分析+PMSI构建 |
| **L6** | **预测市场驱动创新** | 8000+ | **概率套利+真金白银验证** |

**v20.0目标**：
- **平均深度**：≥L4.5（相比v10.0的L3.5大幅提升）
- **核心模块**：≥L5（预测市场增强分析）
- **创新模块**：≥L6（PPDA+PMSI+五引擎协同等突破性分析）

**L6级别定义**（v20.0独有）：
- 运用预测市场数据发现市场定价错误
- 构建原创的概率-价格背离分析框架
- 整合"真金白银"概率验证传统投资判断
- 创造可复制、可验证的投资策略

**半导体L6专项定义**：
- 地缘风险概率背离套利（如台海冲突概率 vs 市场隐含定价）
- AI需求持续性概率验证（预测市场 vs 分析师主观判断）
- 技术领先地位概率量化（工艺代差转化为概率优势）
- 周期拐点概率预测（6层雷达+预测市场双重验证）

## v20.0数据可信度分级系统（预测市场增强）

| 级别 | 来源 | v20.0标注方式 | 质量要求 |
|------|------|-------------|----------|
| **A级 (95-99%)** | 财报+SEC+**预测市场数据** | `[A:源+概率+流动性]` | 预测市场需≥$500K流动性+200人参与 |
| **B级 (85-94%)** | 第三方数据库+**权威预测平台** | `[B:源+置信度]` | 数据更新频率≤24小时 |
| **C级 (70-84%)** | 分析师共识/市场预期 | `[C:源+分歧度]` | 需标注共识分歧程度 |
| **D级 (50-69%)** | 合理估算 | `[D:方法+概率]` | 需说明估算方法和不确定性 |
| **E级 (<50%)** | 假设/推测 | `[E:假设+风险]` | 仅作为情景分析使用 |

**v20.0特有数据源**：
```yaml
预测市场数据 (A级):
  Polymarket: 地缘政治+经济政策+公司事件概率
  Kalshi: 宏观经济+监管政策+选举结果概率
  Manifold: 技术突破+行业趋势+创新应用概率

半导体特化数据源 (A/B级):
  WSTS: 全球半导体市场统计
  SIA: 半导体产业协会数据
  TrendForce: 晶圆代工/存储价格追踪
  IC Insights: 市场份额+资本支出数据

质量验证标准:
  流动性: ≥$500K (确保价格发现有效)
  参与者: ≥200人 (避免操纵风险)
  历史准确率: ≥80% (预测能力验证)
  数据时效: ≤24小时 (实时性要求)
```

**v20.0证据等级目标**：A+B级占比≥95%

## v20.0七条铁律（预测市场增强+半导体特化）

### 原有五条铁律升级版
1. **数据必有源+概率**：每个关键数字标注来源+可信度+**预测市场概率验证**
2. **判断必有据+验证**：每个核心判断≥2条证据链+**真金白银概率支撑**
3. **预测必可验+追踪**：概率+时间+触发条件+**预测市场实时追踪**
4. **洞察必反证+概率**："但如果___则不成立"+ **反向情景概率评估**
5. **结论必可行+风险**：买卖建议+仓位+时间+**概率驱动Kill Switch**

### v20.0新增两条铁律
6. **概率必背离分析**：重要风险事件必须执行PPDA分析，识别概率-价格背离套利机会
   - 半导体特化：台海冲突、技术制裁、AI需求等核心事件**强制执行PPDA**
7. **情绪必市场验证**：投资情绪判断必须用PMSI等预测市场指标验证，避免主观偏误
   - 半导体特化：使用PMSI-半导体（地缘40%+技术30%+需求20%+供应链10%）**强制验证**

**铁律执行标准**：
- 违反任何一条铁律 → 报告不达标，必须重新分析
- 预测市场数据缺失 → 必须说明原因并提供替代验证
- 概率背离分析遗漏 → 必须补充PPDA分析
- 情绪判断无验证 → 必须构建PMSI或类似指标

**半导体铁律补充**：
- 周期判断必须有6层雷达信号支撑（不可仅凭单一指标）
- 技术评估必须有同业对比（≥2个直接竞争对手）
- 地缘风险必须有情景量化（基准/收紧/极端3情景）

## 质量门控体系（QG-01~QG-12）

### 传统质量检查 (QG-01 to QG-07)
```yaml
QG-01: 技术判断必须有同业对比 (vs≥2个直接竞争对手)
QG-02: 周期定位必须有信号支撑 (≥3层雷达一致确认)
QG-03: AI评估必须区分硬件vs平台 (收入拆解+价值量化)
QG-04: 地缘风险必须情景量化 (3情景数值测算)
QG-05: 技术路线图必须标注不确定性 (概率+替代路径)
QG-06: 估值必须周期调整 (当前位置+正常化调整)
QG-07: Kill Switch必须全维度覆盖 (技术+周期+地缘+竞争)
```

### v20.0预测市场增强检查 (QG-08 to QG-10)
```yaml
QG-08: 预测市场数据必须质量验证
  - 流动性>$500K AND 参与者>200人 AND 历史准确率>80%
  - 关键事件覆盖度≥80% (台海+制裁+AI需求+技术竞争)
  - 数据时效性<24小时，概率变化>5%时必须重新获取

QG-09: PPDA背离分析必须量化可信
  - 概率-价格背离度计算过程可追溯
  - 套利信号必须有预期收益+风险评估+时间窗口
  - 背离置信度≥85%才能作为投资建议依据

QG-10: PMSI情绪指数必须历史验证
  - 四模块权重设定有数据支撑和逻辑依据
  - 指数与股价相关性≥0.75 (基于最近90天数据)
  - 情绪信号必须有历史回测准确率≥80%
```

### v20.0五引擎协同验证 (QG-11 to QG-12)
```yaml
QG-11: 五引擎输出必须逻辑一致
  - Industry Cycle + Smart Money + Signal Monitor结论方向一致
  - Equity Structure溢价与Polymarket概率逻辑匹配
  - 任何引擎间冲突必须明确标注和解释

QG-12: 动态Kill Switch必须实时有效
  - 12重Kill Switch阈值基于当前预测市场概率设定
  - 触发条件明确可执行，无主观模糊判断
  - 风险控制必须有概率支撑和历史验证
```

### 自动化质量检测
- **证据等级检查** — A/B级证据占比≥95%，C/Unknown必须标注
- **逻辑一致性** — 技术判断vs周期判断vs估值结论的内在逻辑
- **预测可验证性** — 每个预测有明确时间点+概率区间+观测指标
- **反证完整性** — 每个核心判断有"什么情况下会错"的反证分析
- **七条铁律验证** — 逐条检查是否全部满足
