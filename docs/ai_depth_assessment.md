# AI深度评估 (AI Depth Assessment) v1.0

> Phase 3.5 详细执行规范。由 `docs/deep_dive_protocol.md` v4.0 引用。
> 引用技能: `skills/ai_assessment/` 目录下10个YAML文件

---

## 设计哲学

本规范解决现有框架的三个核心缺陷:

1. **公司级AI标签掩盖分部差异**。"AI受益者"这类标签过于笼统 — Google Search 可能被AI冲击，而 Google Cloud 却从AI中大幅受益。同一家公司内部，不同分部的AI命运可能截然相反。

2. **现有AI评估技能未集成到报告模板**。`skills/ai_assessment/` 下已有10个YAML技能文件，但从未被纳入标准报告流程，导致深度研究中AI分析缺乏结构化输出。

3. **AI对估值的影响未被量化**。我们说"AI很重要"，但从不调整SOTP数字。分析师的AI判断与最终估值之间存在断层。

**核心原则**: AI评估必须是分部级的、基于证据的、与估值挂钩的。

## 三层递进架构

### Layer 1: 分部级AI冲击矩阵 (Segment AI Impact Matrix)

对每个业务分部（合计覆盖 ≥90% 营收）独立评估5个维度:

#### 5个评估维度

**1. 收入冲击** (-5 到 +5)

| 分值 | 含义 |
|------|------|
| -5 | AI在3年内完全消灭该收入流 |
| 0 | AI对收入影响可忽略 |
| +5 | AI在3年内创造巨大新收入（>50%上行空间） |

**2. 成本冲击** (-5 到 +5)

| 分值 | 含义 |
|------|------|
| -5 | AI大幅增加成本（如AI功能所需算力开支） |
| 0 | 成本影响可忽略 |
| +5 | AI大幅降低成本（>30%削减） |

**3. 护城河变化** (强化 / 中性 / 削弱)

- **强化**: AI强化现有竞争优势（如数据飞轮效应）
- **中性**: 竞争地位无实质变化
- **削弱**: AI使竞争对手能够复制或绕过现有护城河

**4. 竞争格局变化** (利好 / 中性 / 利空)

- **利好**: AI提高该分部进入壁垒
- **中性**: 竞争动态不变
- **利空**: AI降低壁垒，使新进入者出现

**5. 时间窗口** (1-3年 / 3-5年 / 5-10年)

- AI影响何时实质性反映到财务数据？

#### AI角色分类

基于5维度综合评估，将每个分部归入以下角色:

| 角色 | 净分范围 | 定义 | 示例 |
|------|---------|------|------|
| **AI放大器** (Amplifier) | +2 到 +3 | AI强化现有业务 | Google Cloud + GenAI |
| **AI赋能者** (Enabler) | +4 到 +5 | AI创造全新业务线 | Tesla FSD |
| **AI中性** (Neutral) | -1 到 +1 | AI影响极小 | 基础零售运营 |
| **AI易受冲击** (Vulnerable) | -2 到 -3 | AI赋能竞争对手 | 传统内容创作 |
| **AI被颠覆** (Disrupted) | -4 到 -5 | AI威胁商业模式 | 人工数据录入服务 |

#### 聚合计算

```
概率加权AI净分 = Σ(segment_AI_score × revenue_weight × realization_probability)
```

#### 输出表模板

| 分部 | 营收($B) | AI角色 | 收入冲击 | 成本冲击 | 护城河Δ | 竞争格局Δ | 时间窗口 | AI净分 |
|------|---------|--------|---------|---------|---------|----------|---------|--------|
| [分部名] | [数据] | [角色] | [-5~+5] | [-5~+5] | [强化/中性/削弱] | [利好/中性/利空] | [时间段] | [计算值] |

### Layer 2: AI实施深度评级 (AI Implementation Depth)

引用现有技能文件:
- `skills/ai_assessment/ai_implementation_ladder_v1.yaml` — L轴 (L0-L4)
- `skills/ai_assessment/ai_commercial_stage_v1.yaml` — S轴 (S0-S4)
- `skills/ai_assessment/ai_five_invariants_v1.yaml` — 五不变量检验

#### L轴 (实施级别)

| 级别 | 名称 | 定义 |
|------|------|------|
| L0 | 观察级 | AI仅用于报告/洞察，无生产环境使用 |
| L1 | 决策支持级 | AI推荐，人类审批（HITL模式） |
| L2 | 受控自动化级 | AI在规则内执行，异常上报人类 |
| L3 | 自主运营级 | AI独立运营，人类监控 |
| L4 | 完全自主级 | AI自主做出复杂决策 |

#### S轴 (商业兑现阶段)

| 阶段 | 名称 | 定义 |
|------|------|------|
| S0 | 叙事期权期 | 财报电话会提到AI但无收入 |
| S1 | 早期变现期 | 首批AI收入可见（<5%总营收） |
| S2 | 规模化期 | AI收入快速增长（5-20%总营收） |
| S3 | 成熟期 | AI完全融入商业模式（>20%总营收） |
| S4 | 平台化期 | 公司向外部提供AI平台 |

#### 五不变量检验

区分真实AI进展与炒作的5项测试:

1. **生产环境证据** — 不只是演示，必须有实际部署
2. **财务实质性** — AI影响必须在财务数据中可见
3. **竞争差异化** — AI创造独特优势，而非通用能力
4. **规模化验证** — 效果超越试点阶段
5. **组织承诺** — 专属AI团队、预算、路线图

#### 输出要求

- L×S坐标图（标注公司在矩阵中的位置）
- 五不变量通过率 (X/5)
- 同业L×S对比表（至少3家可比公司）

### Layer 3: AI估值影响量化 (AI Valuation Impact)

将Layer 1 + Layer 2的发现连接到Phase 2的SOTP估值:

#### 三类AI估值调整

**1. AI溢价分部**

AI证明更高估值倍数的分部。

- **方法**: 上调 EV/Revenue 或 EV/EBITDA 倍数
- **依据**: AI驱动的增长加速、利润率扩张或护城河强化
- **要求**: 必须引用Layer 1中该分部的AI角色为放大器或赋能者

**2. AI折价分部**

AI威胁证明更低估值倍数的分部。

- **方法**: 下调倍数或对终值施加折扣
- **依据**: AI颠覆风险、竞争侵蚀
- **要求**: 必须引用Layer 1中该分部的AI角色为易受冲击或被颠覆

**3. AI期权分部**

当前营收为$0但依赖AI的未来价值。

- **方法**: 概率加权期权价值 = P(成功) × NPV(未来现金流)
- **要求**: 必须有明确的成功概率及支撑证据

#### 输出要求

- **AI调整后SOTP表**（逐分部列示）
- **对比**: AI调整后SOTP vs 基准SOTP
- **AI隐含溢价/折价** = AI调整值 - 基准值
- **市场定价解读**: "市场定价隐含了$X的AI价值；我们评估认为应为$Y"

## 质量门控 QG-09.5

Phase 3.5完成前必须通过以下检查:

- [ ] 覆盖 ≥90% 营收的所有分部均有AI评分
- [ ] L×S定位有 ≥3 项证据支撑
- [ ] AI调整后SOTP与基准SOTP的差异已量化
- [ ] 至少1个分部被评估为潜在AI颠覆风险（强制分析师寻找威胁，而非只看收益）

## 与其他Phase的数据流

```
Phase 2 ──────────────────────────────── Phase 3
  │                                         │
  │  SOTP基准估值                           │  竞争格局
  │  分部营收数据                           │  护城河评估
  │                                         │
  └──────────► Phase 3.5 ◄────────────────┘
              AI深度评估
                  │
          ┌───────┴───────┐
          ▼               ▼
      Phase 4         Phase 5
    AI极端情景        AI调整后估值
    压力测试         最终评级
                    AI相关Kill Switch
```

- **输入自Phase 2**: SOTP基准估值、分部营收数据
- **输入自Phase 3**: 竞争格局、护城河评估
- **输出到Phase 4**: AI极端情景用于压力测试
- **输出到Phase 5**: AI调整后估值用于最终评级、AI相关Kill Switch
