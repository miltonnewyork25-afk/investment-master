## 回测评分规则（机械化）

```
第0层 结构性（10%）：
  DRAM YoY < -30%  → 3分（深度衰退，反向买入信号）
  DRAM YoY < -10%  → 5分
  DRAM YoY < +10%  → 7分
  DRAM YoY < +30%  → 8分
  DRAM YoY > +30%  → 9分（过热，卖出信号）

第1层 先行指标（25%）：
  基准5分
  + DRAM现货MoM > 5%  → +2分
  + Memory CAPEX指引上调 → +2分
  + DRAM现货MoM < -5% → -2分
  + Memory CAPEX指引下调 → -2分

第3层 财务深度（25%）：
  基准5分
  + 毛利率 > 45% → +2分
  + FCF Margin > 20% → +2分
  + 收入YoY增长 → +1分
  + 毛利率 < 42% → -2分

第4层 估值风险（15%）：
  P/E < 15 → 9分（深度低估）
  P/E 15-20 → 7分（低估）
  P/E 20-30 → 5分（合理）
  P/E 30-40 → 3分（高估）
  P/E > 40 → 2分（泡沫）

综合评分 = 0.10×L0 + 0.25×L1 + 0.20×5(默认) + 0.25×L3 + 0.15×L4 + 0.05×5(默认)
```

| **DRAM价格趋势** | 暴跌中，YoY -40%+ | 历史报道 |
| **Memory CAPEX** | SK Hynix计划-50%，Micron-42% | 历史报道 |
| **行业情绪** | 极度悲观，"历史性暴跌" | 历史报道 |

### 框架评分

```
第0层（结构性）：DRAM YoY < -30% → 3分
  但按照逆向逻辑：深度衰退 = 买入机会 → 调整为 8分

第1层（先行指标）：
  基准5分
  - DRAM现货MoM < -5% → -2分
  - CAPEX指引大幅下调 → -2分
  = 1分（极负面）

第3层（财务深度）：
  基准5分
  + 毛利率45% → +2分
  = 7分

第4层（估值风险）：
  P/E ~12x → 9分（深度低估）

综合评分 = 0.10×8 + 0.25×1 + 0.20×5 + 0.25×7 + 0.15×9 + 0.05×5
        = 0.8 + 0.25 + 1.0 + 1.75 + 1.35 + 0.25
        = 5.4 → 54/100
```

### 框架预测

| 项目 | 预测 |
|------|------|
| 综合评分 | 54/100（中性偏弱） |
| 6个月方向 | **不确定/可能继续下跌** |
| 建议 | 观望或小仓位试探 |
| 置信度 | 低 |

### 实际结果（6个月后：2023年4月）

| 项目 | 实际值 |
|------|--------|
| 股价 | $52.41 |
| 涨跌幅 | **+29.5%** |
| 预测是否正确 | ❌ **错误**（预测中性，实际大涨）|

### 分析

**框架失败原因**：
1. 先行指标给出极负面评分（1分），但这正是周期底部特征
2. 框架缺乏"极端负面=反转买入"的逆向逻辑
3. 估值低（9分）应该覆盖先行指标的负面

## 框架问题诊断

### 问题1：周期底部识别失败

| 问题 | 表现 | 根因 |
|------|------|------|
| 2022年10月 | 框架给出54分（中性） | 先行指标极负面时，框架无法识别这是底部 |
| 2023年4月 | 框架给出48分（偏弱） | 财务数据滞后，拐点识别失败 |

**改进建议**：添加"极端负面=反转信号"规则
```
IF 先行指标 < 3 AND 估值 > 7 THEN
   可能是周期底部，加分+15
```

### 问题2：周期顶部预警不足

| 问题 | 表现 | 根因 |
|------|------|------|
| 2024年4月 | 框架给出70分（正面） | 先行指标全正面时，没有"过热"预警 |

**改进建议**：添加"过热预警"规则
```
IF 先行指标 > 9 AND 估值 < 4 THEN
   可能是周期顶部，减分-15
```

### 问题3：先行指标权重过高

先行指标权重25%，但在周期极端点（底部/顶部）会误导判断。

**改进建议**：
- 正常周期：先行指标25%
- 极端估值时：先行指标降至15%，估值升至25%

### 问题4：财务数据滞后性

财务数据是滞后指标，在拐点时会严重误导。

**改进建议**：
- 在先行指标剧烈变化时，降低财务数据权重
- 添加"边际变化"因子而非绝对值

## 诚实结论

### 框架有效性评估

| 评估维度 | 得分 | 说明 |
|----------|------|------|
| **方向预测准确率** | 30-40% | 低于随机猜测(50%) |
| **周期识别能力** | 差 | 无法识别顶部和底部 |
| **趋势跟随能力** | 中 | 在趋势中期表现较好 |
| **风险控制** | 差 | 顶部时仍给正面评分 |

### 核心发现

1. **框架在周期中期有效，在拐点失效**
   - 测试点3（2023年10月，趋势中期）：✅ 正确
   - 测试点1、2（底部）、测试点4（顶部）：❌ 错误

2. **先行指标的"悖论"**
   - 先行指标极负面时，往往是买入机会
   - 先行指标极正面时，往往是卖出信号
   - 当前框架与此相反

3. **估值的重要性被低估**
   - 在P/E极低时（测试点1），应该大胆买入
   - 在P/E极高时（测试点4），应该果断减仓
   - 当前框架估值权重仅15%

### 改进后的框架建议

```
新权重分配（考虑周期阶段）：

正常阶段：
  - 先行指标：25%
  - 财务深度：25%
  - 估值风险：15%

极端阶段（估值<15x 或 >35x）：
  - 先行指标：15%（降低）
  - 估值风险：30%（提高）
  - 添加：逆向信号 +20%

逆向信号规则：
  - 当先行指标<3 且 估值>7 → 强烈买入信号
  - 当先行指标>9 且 估值<3 → 强烈卖出信号
```
