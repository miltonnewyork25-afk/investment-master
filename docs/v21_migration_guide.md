# v20 → v21 迁移指南

> 框架 v21.0 核心改变: 从"模板填空"到"争议驱动"。

---

## 核心变化总结

| 维度 | v20.0 | v21.0 |
|------|-------|-------|
| **报告组织** | 按 U1-U25 模块编号线性排列 | 按 5-8 个 Core Questions 分章 |
| **模块深度** | 所有模块等权重执行 | 按CQ相关性分配权重(高/中/低) |
| **数据标注** | `[A:]`/`[B:]`/`[P:]`/`[E:]` + `[置信度: XX%]` | `[硬数据:]`/`[合理推断:]`/`[主观判断:]`，废弃置信度 |
| **看空分析** | 无篇幅要求，实际<2% | 强制≥30%篇幅，≥8个看空论点 |
| **质量标准** | 字数+表格数+通过率 | 洞察密度(So What评分)+CQ覆盖率 |
| **五引擎** | 5引擎均算独立验证 | 标注数据依赖，E2+E3一致只算1票 |
| **铁律** | 9条 | 12条(新增三层标注+看空等权+模块So What) |

---

## 迁移步骤

### 对新分析（v21.0原生）

1. Phase 0.5 执行完整8步流程（含3个新增步骤）
2. 报告按CQ分章，模块按相关性分配深度
3. 使用三层标注系统
4. Phase 4 执行看空等权重分析

### 对已有v20报告（回顾性改进）

不建议回溯修改已完成报告。v21.0仅应用于新分析。
