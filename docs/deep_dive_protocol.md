## 启动协议

### Phase 0: 数据基础（自动执行，不占会话）

1. **环境验证** — 确认当前worktree + 公司行业分类
2. **数据预取** — 调用 `data-prefetch` 技能，获取11+数据文件到 `data/research/{TICKER}/`
3. **数据质量检查** — 读取 `prefetch_metadata.json`，确认关键数据可用
4. **预测市场可用性** — 检查 `prediction_market.json` 覆盖度
5. **框架加载** — 读取行业特化配置 + 行业框架文档

**Phase 0 完成标准**: `prefetch_metadata.json` 存在 + Layer 1数据全部OK + ≥8/11文件可用

## Phase详细规范

### Phase 1: 定位与生态

**产出**: 公司定位画像 + 产业链地图 + 预测市场概率矩阵

**必做模块**:
- 公司基本画像（业务模型、历史沿革、管理层评估）
- 产业链上下游映射（供应商、客户、竞争者生态）
- 预测市场环境扫描（宏观+行业+公司特定事件概率）
- 行业周期初步定位（P1-P5）

**字数目标**: ≥20,000字符 (wc -m)

**门控 QG-01~03**:
- QG-01: 公司画像完整（业务模型+管理层+历史≥3000字）
- QG-02: 产业链映射≥10个关键节点
- QG-03: 预测市场数据≥8个相关事件

### Phase 2: 财务与估值

**产出**: 周期精确定位 + 财务深度分析 + SOTP分部估值 + DCF交叉验证

**必做模块**:
- 周期精确定位（6层雷达/行业周期模型）
- 5年财务趋势分析（收入/利润/现金流/资产负债）
- SOTP分部估值（详见 `docs/sotp_methodology.md`）
- DCF估值 + 可比公司估值
- 三情景矩阵（Bull/Base/Bear）

**字数目标**: ≥25,000字符 (wc -m)

**门控 QG-04~06**:
- QG-04: 周期定位有≥4个支撑信号
- QG-05: SOTP覆盖所有主要分部（占营收≥90%）
- QG-06: 三种估值方法交叉验证偏离度<20%

### Phase 3: 护城河深挖

**产出**: 竞争优势量化 + 技术壁垒 + 五引擎分析 + PPDA背离发现

**必做模块**:
- 护城河类型识别与量化（品牌/网络效应/转换成本/成本优势/规模）
- 技术路线图与替代威胁
- 五引擎协同分析（周期+股权+聪明钱+信号+预测市场）
- PPDA概率-价格背离分析（≥3个显著背离）
- PMSI情绪指数构建

**字数目标**: ≥25,000字符 (wc -m)

**门控 QG-07~09**:
- QG-07: 护城河量化（每种护城河有具体数据支撑）
- QG-08: 五引擎每引擎≥3000字分析
- QG-09: PPDA发现≥3个背离 + PMSI构建完成

### Phase 4: 对抗审查（v3.0新增）

**产出**: 行为金融偏差修正 + 事实核查 + 反证挑战 + Smart Money验证

> **设计原因**: TSM v2.0反思发现，估值完成后缺少系统性"第二意见"，导致锚定效应和确认偏误未被纠正。

**必做模块**:
- 行为金融四项偏差检查（详见 `docs/behavioral_finance.md`）
- 关键数据事实核查（抽查≥10个核心数据点的来源真实性）
- 反证挑战（"如果论点完全错误，最可能的原因是什么？"）
- Smart Money立场验证（机构持仓变化 vs 我们的结论是否一致）
- 极端压力测试（SOTP极端场景）
- AI 价值链定位（如适用）

**字数目标**: ≥15,000字符 (wc -m)

**门控 QG-10~11**:
- QG-10: 四项认知偏差检查全部完成 + 量化修正
- QG-11: 事实核查≥10个数据点 + 反证≥3条

### Phase 5: 决策输出

**产出**: 最终评级 + 仓位建议 + Kill Switch + 可验证预测 + 行动清单

**必做模块**:
- 综合评分（0-100分）+ 评级（强烈推荐/推荐/中性/回避）
- 仓位建议（基础仓位 × 周期系数 × 置信度）
- Kill Switch设置（≥8个触发条件 + 阈值 + 动作）
- 可验证预测清单（≥15个，含概率+时间+触发条件）
- 投资日历（未来12个月关键事件+预期影响）
- 90天行动清单（具体的买入/加仓/减仓触发点）

**字数目标**: ≥15,000字符 (wc -m)

**门控 QG-12**:
- Kill Switch≥8个条件
- 可验证预测≥15个
- 铁律七条全部满足

## 五引擎协同分析系统

### 五大核心分析引擎

**Engine 1: Industry Cycle Analyzer** — 多层嵌套周期识别+拐点概率预测
- 输出: Stage定位+拐点概率+6-12月预警+历史验证准确率
- 适配: 半导体(AI 4阶段) | 消费品(6阶段) | 金融(监管周期) | 科技(平台生命周期)

**Engine 2: Equity Structure Analyzer** — 稀缺性溢价量化+控制权解构+流动性深度
- 输出: 股权结构图+稀缺性倍数+控制权价值+流动性约束

**Engine 3: Smart Money Tracker** — 顶级投资者逻辑解码+机构共识度量化
- 输出: 投资大师持仓逻辑+机构共识度评分+资金流向趋势+置信度评级

**Engine 4: Signal Monitoring System** — 五级预警体系+条件概率触发
- 输出: 实时信号状态+预警级别+触发条件+历史胜率统计

**Engine 5: Prediction Market Analyzer** — 预测市场概率+PPDA背离+PMSI情绪
- 输出: 概率矩阵+背离度计算+情绪指数+套利策略+风险控制
- **方法论详见**: `docs/prediction_market_methodology.md`

### 协同调用顺序

```yaml
Step 1: industry-cycle-analyzer → Stage定位+时机判断基础
Step 2: prediction-market-analyzer → 概率矩阵+背离预分析
Step 3: [并行] equity-structure-analyzer + smart-money-tracker → 结构风险+聪明钱双重验证
Step 4: signal-monitoring-system → 综合预警信号+动态阈值
Step 5: 五引擎协同验证 → 统一投资建议+置信度评级
```

## 数据源标注规范

每个关键数据点必须标注来源类型：

| 标签 | 含义 | 示例 |
|------|------|------|
| `[A: 来源, 日期]` | 一级数据（财报/SEC/官方） | `[A: TSM 2025 10-K, 2026-01]` |
| `[B: 来源, 日期]` | 二级数据（第三方数据库/权威平台） | `[B: Yahoo Finance, 2026-02-06]` |
| `[P: 来源, 日期]` | 预测市场数据 | `[P: Polymarket, 2026-02-05]` |
| `[E: 推导说明]` | 分析师估算/推导 | `[E: 基于Q3毛利率趋势线性外推]` |

**密度要求**: ≥15个标注/10,000字符。低于此密度自动触发补充标注。

## 铁律（Deep-Dive强制）

1. **数据必有源**: 每个关键数字标注来源，≥15标注/万字
2. **判断必有据**: 每个核心判断≥2条证据链
3. **预测必可验**: 概率+时间+触发条件
4. **洞察必反证**: "但如果___则不成立"
5. **结论必可行**: 买卖建议+仓位+时间+Kill Switch
6. **概率必背离分析**: 重要风险事件执行PPDA分析
7. **情绪必市场验证**: 投资情绪判断必须用PMSI等指标验证
