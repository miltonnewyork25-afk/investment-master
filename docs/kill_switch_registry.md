# Kill Switch 统一注册表 v2.0

> **解决问题 S4**: Kill Switch权属不清 — 各章节自行描述KS，阈值/状态/动作三重不一致
> **核心思想**: KS在报告中定义为Single Source of Truth，其他章节只引用编号
> **v2.0变化**: 移除所有仓位操作指令(减仓X%/退出/清仓)。KS描述"论文何时失效"，不描述"失效后怎么操作"。操作是投资者自己的事。

---

## 设计哲学

**旧模式**: Phase 2在估值章节写几个KS，Phase 3写几个，Phase 5又写一批，互相矛盾
**新模式**: KS统一在Phase 5的Kill Switch章节定义，所有前序章节只引用KS编号

---

## Kill Switch 注册表结构

每个公司报告中的Kill Switch章节（Phase 5）使用标准注册表：

```markdown
## Kill Switch 注册表

> Single Source of Truth — 所有KS定义仅在此表，其他章节通过[KS-xxx]引用

| KS编号 | 类别 | 触发条件 | 阈值 | 当前值 | 状态 | 论文含义 | 关联KA |
|--------|------|---------|------|--------|------|---------|--------|
| KS-FIN-001 | 财务 | 营收连续2季下滑 | <-5% QoQ | +3% | 🟢安全 | 增长假设失效，需重估核心价值 | KA-GR-001 |
| KS-FIN-002 | 财务 | 毛利率跌破阈值 | <50% | 55% | 🟢安全 | 定价权/成本优势假设被否定 | KA-MG-002 |
| KS-VAL-001 | 估值 | PE超过历史高位 | >40x | 32x | 🟢安全 | 市场定价已超出基本面支撑 | KA-VL-001 |
| KS-CP-001 | 竞争 | 核心市场份额流失 | <25% | 32% | 🟢安全 | 护城河假设需重新审查 | KA-CP-001 |
| KS-AI-001 | AI | AI投入无回报满2年 | ROI<0 2yr | +15% 1yr | 🟢安全 | AI溢价不再成立 | KA-AI-001 |
| KS-REG-001 | 监管 | 反垄断强制拆分 | 裁决生效 | 审理中 | 🟡监控 | 需按拆分后各部分重估 | KA-RK-001 |
```

## KS编号规范

| 前缀 | 类别 | 覆盖范围 |
|------|------|---------|
| `KS-FIN-` | 财务类 | 营收/利润/现金流/负债恶化 |
| `KS-VAL-` | 估值类 | 估值过高/泡沫信号 |
| `KS-CP-` | 竞争类 | 市场份额/竞争格局恶化 |
| `KS-AI-` | AI相关 | AI投入失败/AI被颠覆 |
| `KS-REG-` | 监管类 | 反垄断/政策/合规风险 |
| `KS-MKT-` | 市场类 | 宏观/系统性风险 |
| `KS-MGT-` | 管理类 | 管理层变动/治理恶化 |

## 三级阈值体系

每个Kill Switch定义三级阈值，取代原来的单一二元触发：

```
L1 预警 (🟡黄灯):
  - 触发条件接近但未达到 → 加强监控频率
  - 含义: 假设开始承压，需要更密切追踪相关信号

L2 升级 (🟠橙灯):
  - 触发条件已达到 → 论点需要审查
  - 含义: 核心假设被否定或严重削弱，投资论文可靠性下降

L3 失效 (🔴红灯):
  - 触发条件严重恶化 → 投资论文失效
  - 含义: 原始投资逻辑不再成立，需要根据新现实重新评估
```

### 三级阈值示例

```markdown
### KS-FIN-001: 营收增速恶化

| 级别 | 阈值 | 论文含义 |
|------|------|---------|
| L1 🟡 | 营收增速 < 行业平均 连续1季 | 增长逻辑承压，需月度追踪 |
| L2 🟠 | 营收增速 < 0% 连续1季 | 增长假设被否定，投资论文可靠性下降 |
| L3 🔴 | 营收增速 < -5% 连续2季 | 增长论文失效，原始投资逻辑不再成立 |

当前状态: 🟢安全 (最新增速 +12%)
```

## KS定义规范

每个Kill Switch必须包含以下7个字段（缺一不可）：

| 字段 | 必填 | 说明 |
|------|------|------|
| **KS编号** | ✅ | 唯一标识符 |
| **触发条件** | ✅ | 明确的可观测事件/指标 |
| **三级阈值** | ✅ | L1/L2/L3各自的具体数值 |
| **当前值** | ✅ | 撰写时的实际数据 + 来源 |
| **状态** | ✅ | 🟢安全/🟡预警/🟠升级/🔴失效 |
| **论文含义** | ✅ | 触发后对投资论文的影响（不含仓位操作） |
| **关联KA** | ✅ | 对应的KAL假设编号 |

**禁止事项**:
- 禁止没有具体阈值的KS（如"业绩不佳"→必须量化）
- 禁止KS间阈值冲突
- **禁止仓位操作指令**（如"减仓50%"、"清仓"、"停止加仓"→KS只描述论文何时失效，不描述操作）

## 引用规则

### 其他章节引用KS

报告的非KS章节（Phase 1-4）中提到Kill Switch时，**只引用编号**，不重新定义：

```markdown
✅ 正确: "如果云业务增速持续低于行业平均，将触发KS-FIN-001 [L1预警]"
✅ 正确: "参见KS-AI-001关于AI投入回报的退出条件"
❌ 错误: "如果云增速下滑超过5%应该减仓30%" ← 自行定义了阈值+包含仓位操作
❌ 错误: "Kill Switch: 营收下滑10%减仓50%" ← 包含仓位操作指令（禁止）
```

### Phase 2-4 中的KS预注册

Phase 2-4分析中发现新的风险点时，可以预注册KS（在报告中标注待注册）：

```markdown
"此风险建议注册为Kill Switch [待注册: KS-CP-003, 触发条件: 竞争者份额>30%, 建议L2阈值>35%]"
```

Phase 5统一整理所有预注册KS，赋予正式编号和完整7字段。

## KS与KAL的联动

每个KS必须关联至少一个KAL假设：

```
KS-FIN-001 (营收增速恶化) ← KA-GR-001 (云业务增速25-30%)
  含义: 当KA-GR-001假设被否定(增速<15%)时，KS-FIN-001进入L2

KS-AI-001 (AI投入无回报) ← KA-AI-001 (AI收入贡献$10B)
  含义: 当KA-AI-001假设被否定(AI收入<$2B满2年)时，KS-AI-001进入L2
```

**联动规则**: KAL假设被Phase 4标记为🔴已否定 → 对应KS自动升级至少一级

## 最低KS数量要求

| 类别 | 最低数量 | 说明 |
|------|---------|------|
| 财务类 | ≥3 | 营收/利润/现金流各至少1个 |
| 估值类 | ≥2 | 上行+下行各至少1个 |
| 竞争类 | ≥2 | 份额+格局各至少1个 |
| AI相关 | ≥2 | v5.1要求 |
| 监管类 | ≥1 | 至少覆盖主要监管风险 |
| 其他 | 视情况 | 管理层/宏观等 |
| **总计** | **≥15** | v5.1标准 |

## 质量门控集成

Quality Gate R-G 对KS的检查（详见 `docs/quality_gate_v2.md`）：

1. **完整性**: KS总数≥15，AI相关≥2
2. **一致性**: 报告中所有KS引用 vs 注册表定义 → 无矛盾
3. **可执行性**: 每个KS的阈值可从公开数据源观测
4. **KAL联动**: 每个KS有关联KA编号
5. **状态准确**: 当前值来自最新数据，状态标记正确

## GOOGL案例回顾

| 问题 | 旧行为 | 注册表解决方案 |
|------|--------|---------------|
| Ch39定义KS阈值10%，Ch42引用说15% | 各章节自行定义 | 只在注册表定义，其他章节用[KS-xxx]引用 |
| 状态标记自相矛盾 | 无统一状态管理 | 注册表统一管理状态，7字段缺一不可 |
| 部分KS无具体动作 | 写了触发条件但没说怎么办 | 动作字段强制填写，三级响应体系 |
